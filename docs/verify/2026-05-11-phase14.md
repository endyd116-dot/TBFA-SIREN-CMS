## 3. 발견된 버그 및 즉시 수정

### BUG-14-01: 기관 관리 화면이 가짜 데이터로 동작 — 실제 서버 미연결 (Critical)

**증상**: 기관 목록이 코드에 하드코딩된 가짜 데이터("서울강남경찰서", "서울시교육청")로만 표시되고, 기관 등록/수정/비활성화가 화면에서만 반영되고 DB에 저장되지 않음  
**원인**: `admin-agency-mgmt.js` 내 `USE_MOCK = true` 스위치가 켜진 채 배포  
**수정**: `admin-agency-mgmt.js` — `USE_MOCK = false`로 전환

### BUG-14-02: 기관 목록 필터 파라미터 이름 불일치 (High)

**증상**: 유형·활성 상태 필터 적용 시 필터가 무시되고 전체 목록 반환  
**원인**: 화면에서 `agencyType`, `isActive=true/false` 파라미터 전송 → 서버는 `type`, `active=1/0` 기대  
**수정**: `admin-agency-mgmt.js` — `type=` / `active=1|0` 파라미터로 교정

### BUG-14-03: 인계 이력 화면이 가짜 데이터로 동작 — 실제 서버 미연결 (Critical)

**증상**: 인계 이력 목록이 코드에 하드코딩된 가짜 데이터로만 표시되고, 인계 실행이 DB에 저장되지 않음  
**원인**: `admin-referral.js` 내 `USE_MOCK = true` 스위치가 켜진 채 배포  
**수정**: `admin-referral.js` — `USE_MOCK = false`로 전환

### BUG-14-04: 인계 모달 기관 목록 로드 시 필터 파라미터 불일치 (Medium)

**증상**: 인계 실행 모달에서 기관 드롭다운이 비활성 기관까지 포함해 표시될 수 있음  
**원인**: `isActive=true` 파라미터 전송 → 서버는 `active=1` 기대  
**수정**: `admin-referral.js` — `active=1` 파라미터로 교정

### BUG-14-05: 인계 실행 후 응답 처리 방식 불일치 (Critical)

**증상**: [PDF 생성 & 인계] 클릭 시 즉시 오류 발생 — "인계 실패"  
**원인**: 화면에서 `/api/admin-referral-create` 응답을 `res.blob()`(PDF 바이너리)으로 처리하려 했으나, 서버는 `{ ok, logId, pdfAvailable }` JSON을 반환 후 PDF는 `/api/admin-referral-pdf?referralId=N`으로 별도 제공  
**수정**: `admin-referral.js` — JSON 응답에서 `logId` 추출 후 토스트 표시, 이어서 별도 PDF 엔드포인트에서 다운로드하는 2단계 흐름으로 교정

### BUG-14-06: 인계 상태 목록에 `pending` 상태 누락 (Low)

**증상**: 서버에서 `pending` 상태로 저장된 이력이 있을 경우 레이블·색상 미표시  
**원인**: 화면의 `STATUS_LABELS`/`STATUS_COLORS`에 `pending` 키가 없음 (서버 `VALID_STATUSES`에는 포함)  
**수정**: `admin-referral.js` — `pending: '대기 중'` + 색상 추가, 상태 갱신 셀렉트박스에도 옵션 추가
