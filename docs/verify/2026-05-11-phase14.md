## 3. 발견된 버그 및 즉시 수정

### BUG-14-01: 기관 관리 화면이 하드코딩된 가짜 데이터로만 동작 — 서버 미연결 (Critical)

**증상**: 기관 목록이 코드에 박혀있는 "서울강남경찰서", "서울시교육청" 두 개로만 표시되고, 기관 등록·수정·비활성화가 화면에서만 반영되고 DB에 저장되지 않음  
**원인**: 화면 코드 내 서버 연결 대신 가짜 데이터 사용 스위치가 켜진 채 배포됨  
**수정**: `admin-agency-mgmt.js` — 가짜 데이터 모드 해제, 실제 서버와 연결

### BUG-14-02: 기관 목록 필터 파라미터 이름 불일치 (High)

**증상**: 유형·활성 상태 필터 적용 시 필터가 무시되고 전체 목록 반환  
**원인**: 화면에서 서버가 이해하지 못하는 이름(`agencyType`, `isActive=true/false`)으로 파라미터 전송  
**수정**: `admin-agency-mgmt.js` — 서버가 기대하는 형식(`type=`, `active=1|0`)으로 교정

### BUG-14-03: 인계 이력 화면이 하드코딩된 가짜 데이터로만 동작 — 서버 미연결 (Critical)

**증상**: 인계 이력 목록이 코드에 박혀있는 샘플 2건으로만 표시되고, 인계 실행이 DB에 저장되지 않음  
**원인**: 동일하게 가짜 데이터 사용 스위치가 켜진 채 배포됨  
**수정**: `admin-referral.js` — 가짜 데이터 모드 해제, 실제 서버와 연결

### BUG-14-04: 인계 모달 기관 목록 로드 시 필터 파라미터 불일치 (Medium)

**증상**: 인계 실행 모달 기관 드롭다운에 비활성 기관까지 포함 표시될 수 있음  
**원인**: `isActive=true` 파라미터 전송 → 서버는 `active=1` 기대  
**수정**: `admin-referral.js` — `active=1` 파라미터로 교정

### BUG-14-05: 인계 실행 후 응답 처리 방식 불일치 (Critical)

**증상**: [PDF 생성 & 인계] 클릭 시 즉시 오류 발생 — "인계 실패"  
**원인**: 화면에서 인계 요청 응답을 PDF 바이너리로 처리하려 했으나, 서버는 저장 완료 JSON을 먼저 보내고 PDF는 별도 주소에서 제공  
**수정**: `admin-referral.js` — 저장 완료 확인 후 토스트 표시, 이어서 별도 PDF 주소에서 파일 다운로드하는 2단계 흐름으로 교정

### BUG-14-06: 인계 상태 목록에 `pending(대기 중)` 상태 누락 (Low)

**증상**: 대기 중 상태로 저장된 이력의 레이블·색상 미표시  
**원인**: 서버에서 지원하는 `pending` 상태가 화면 레이블 목록에 없음  
**수정**: `admin-referral.js` — "대기 중" 레이블·색상 추가, 상태 변경 목록에도 추가

### BUG-14-07: 신고 상세 화면에 [외부 기관 인계] 버튼 누락 (High)

**증상**: 사건·악성민원·법률 상담 신고 상세 화면에서 인계 버튼이 없어 Q4 시나리오 자체가 불가능  
**원인**: 프론트 구현 시 신고 상세 화면에 인계 버튼 추가가 누락됨  
**수정**: `admin-siren.js` — 사건·악성민원·법률 상담 상세 화면 하단에 🏛️ 외부 기관 인계 버튼 추가; `admin-referral.js` — 신고 상세 화면에서 신고 유형·ID를 전달받아 인계 모달을 바로 여는 기능 추가

### BUG-14-08: 인계 실행 모달에 양식 미리보기 영역 누락 (Medium)

**증상**: 기관 선택 시 해당 기관의 인계 양식 템플릿 미리보기가 표시되지 않음 (Q5 시나리오 미구현)  
**원인**: 프론트 구현 시 미리보기 텍스트 영역이 모달에 포함되지 않음  
**수정**: `admin-referral.js` — 기관 드롭다운 아래에 양식 미리보기 영역 추가; 기관 선택 변경 시 해당 기관의 템플릿이 미리보기에 자동 표시
