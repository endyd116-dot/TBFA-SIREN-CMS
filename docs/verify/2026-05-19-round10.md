## 3. 주요 확인 사항

### 3.1 댓글 투표 토글 (Q1·Q2)
- `comment-vote.ts`: 기존 동일 투표 타입 → DELETE(removed) / 없으면 INSERT(added)
- 프론트: `c.myVote` 상태 관리 → `renderComments()` 재렌더링으로 `.voted` CSS 파란색 강조 적용/해제
- `.ic-action-btn.voted { background:#dbeafe; border-color:#2563eb; color:#1e40af }` 파란색 계열 확인

### 3.2 댓글 신고 중복 방지 (Q3·Q4)
- API: 409 + `"이미 신고한 항목입니다."` 메시지
- 프론트: 409 상태 분기 → `toast('이미 신고한 항목입니다.')`
- 신고 성공 시 status=`"pending"` INSERT 확인

### 3.3 댓글 숨김 처리 (Q6)
- 어드민 검토 API: `isHidden:true` DB 업데이트
- **API 레벨 필터링**: `incident-comments.ts` line 57 `if (c.is_hidden && !isAdmin) return false` — 일반 사용자는 응답에서 완전 제외
- 어드민 화면: `.hidden-comment { opacity:0.55 }` 시각적 표시

### 3.4 알림 구독 upsert (Q9)
- `ON CONFLICT (member_id, event_type) DO UPDATE SET channels` — 중복 없이 덮어쓰기
- 새로고침 후 GET 재조회 → 저장된 channels 반영 확인

### 3.5 AI 법률 배정 fire-and-forget (Q10)
- `aiLawyerSpecialty` 분석 → `assigned_categories ILIKE %specialty%` 매칭
- 실패 시 `catch(assignErr) console.warn` — throw 없음, 응답 지연 없음
- 배정 성공 시 `LEGAL_ASSIGNED` 알림 dispatch (별도 try-catch)
