## 4. 발견된 버그 및 즉시 수정

### BUG-11-01: 구독 게시글 목록 응답 키 불일치 (High)

- **증상**: 구독 게시글 탭이 항상 빈 화면 표시
- **원인**: 백엔드가 `postSubscriptions` 배열 반환, 프론트가 `rows` 키 탐색
- **수정**: `my-subscriptions.js:64` — `json.postSubscriptions || json.data?.postSubscriptions || json.rows`로 다중 fallback

### BUG-11-02: 멘션 알림 목록 응답 키 불일치 (High)

- **증상**: 새 알림 탭이 항상 빈 화면 표시
- **원인**: 백엔드가 `items` 배열 반환, 프론트가 `rows` 키 탐색
- **수정**: `my-subscriptions.js:157` — `json.items || json.data?.items || json.rows`로 수정

### BUG-11-03: 멘션 알림 항목 필드명 불일치 (High)

- **증상**: 알림 목록에 이름·링크가 표시되지 않음
- **원인**: 백엔드 `sourceType`/`mentionerName`/`sourceId`, 프론트 `type`/`actorName`/`postId` 기대
- **수정**: `my-subscriptions.js:168–178` — 백엔드 필드명으로 전환

### BUG-11-04: 구독 해제 HTTP 메서드 오류 (High)

- **증상**: 구독 해제 버튼 클릭 시 구독 취소가 아닌 "이미 구독 중" 응답
- **원인**: 프론트가 `POST { subscribe: false }` 전송, 백엔드 POST는 항상 구독 추가 처리
- **수정**: `my-subscriptions.js:118` — `DELETE /api/user-post-subscribe?postId=N`으로 변경

### BUG-12-01: 신원 식별 요청 필드명 불일치 (Critical)

- **증상**: 신원 확인 버튼 클릭 시 400 오류 ("revealLevel 필수")
- **원인**: 프론트가 `level` 필드 전송, 백엔드는 `revealLevel` 기대
- **수정**: `admin-anon-reveal.js:193` — `revealLevel: _selectedLevel`로 수정

### BUG-12-02: 신원 정보 응답 키 불일치 (High)

- **증상**: 신원 확인 성공 후 정보 미표시
- **원인**: 백엔드가 `reporter` 반환, 프론트가 `identity` 키 탐색
- **수정**: `admin-anon-reveal.js:203` — `res.data?.reporter || res.data?.identity`로 수정

### BUG-12-03: 감사 로그 응답 키 불일치 (High)

- **증상**: 감사 로그 테이블 빈 화면, 담당자·식별수준 미표시, KPI 0
- **원인**:
  - 목록 키: 백엔드 `items`, 프론트 `rows`
  - 담당자: 백엔드 `revealedByName`, 프론트 `adminName`
  - 수준: 백엔드 `revealLevel`, 프론트 `level`
  - KPI 통계: 백엔드에 `stats` 집계 없음
- **수정**:
  - `admin-anon-audit.js:66` — `items` 우선 fallback
  - `admin-anon-audit.js:87,91` — `r.revealLevel || r.level`, `r.revealedByName || r.adminName`로 수정
  - `admin-anonymous-reveal-logs.ts` — KPI 집계(totalCount/todayCount/level2Count/monthCount) 추가, 필터(level/dateFrom/dateTo) 지원 추가
