## 2. 발견한 BUG 4건 (상세)

### BUG-20A-01 [HIGH · 해결] 자격 심사 탭에 모든 항목이 `-`로 표시

- **증상**: 회원·운영자 그룹의 자격 심사 탭 진입 시 이름·이메일·현재 등급·요청 등급·신청일 모든 열이 `-`로 표시됨
- **원인**: 서버 API는 신청자 정보를 `memberId`, `currentType`, `requestedType`, `createdAt`으로 반환하는데, 화면은 `memberName`, `email`, `currentGrade`, `requestedGrade`, `requestedAt` 필드명으로 읽으려 해서 모두 `undefined` → `-` 표시
- **수정**:
  - 서버 API에서 `memberId`로 신청자 이름·이메일을 별도 조회해 `memberName`, `memberEmail` 필드로 추가 반환
  - 화면에서 `e.email → e.memberEmail`, `e.currentGrade → e.currentType`, `e.requestedGrade → e.requestedType`, `e.requestedAt → e.createdAt` 필드명 수정
- **수정 파일**: `netlify/functions/admin-members-unified.ts`, `public/js/admin-members-group.js`
- **심각도**: HIGH — 자격 심사 업무 자체가 불가

### BUG-20A-02 [MEDIUM · 해결] 회원 목록 등급 열에 `[object Object]` 표시

- **증상**: 회원 탭의 등급 열에 등급명(정회원·후원회원 등) 대신 `[object Object]`가 표시됨
- **원인**: 서버 API가 등급 정보를 `{ id, code, nameKo, icon, colorHex }` 객체로 반환하는데, 화면에서 객체 전체를 문자열로 변환해 출력
- **수정**: 화면에서 `m.grade` → `m.grade?.nameKo`로 수정해 등급 이름만 추출
- **수정 파일**: `public/js/admin-members-group.js`
- **심각도**: MEDIUM — 회원 등급 식별 불가

### BUG-20A-03 [MEDIUM · 해결] '콘텐츠 관리' 메뉴 클릭 시 화면이 표시되지 않음

- **증상**: 사이드바에서 '콘텐츠 관리'를 클릭해도 해당 화면이 나타나지 않고 빈 화면 유지
- **원인**: HTML에 콘텐츠 영역 컨테이너(`id="adm20-content"`)와 콘텐츠 관리 뷰 패널이 같은 ID를 공유. 라우터가 `getElementById("adm20-content")`를 호출하면 항상 컨테이너를 반환해 실제 뷰 패널에 `is-active` 클래스가 적용되지 않음
- **수정**: 콘텐츠 관리 뷰 패널 ID를 `adm20-content-mgmt`로 변경, 사이드바 라우팅 정의에서도 동일하게 수정
- **수정 파일**: `public/admin.html`, `public/js/admin-shell.js`
- **심각도**: MEDIUM — 콘텐츠 관리 화면 접근 불가

### BUG-20A-04 [LOW · 🚚 후속 권고] super_admin 전용 메뉴 접근 제어 미구현

- **증상**: '익명 감사 로그' 메뉴 항목이 모든 어드민에게 표시됨. "(super_admin 전용)" 레이블 텍스트만 있을 뿐 실제 권한 분기 없음
- **영향**: 일반 어드민도 메뉴 진입 가능. 단, 현재 화면 자체는 "20-A 연결 예정" 플레이스홀더여서 실제 데이터 유출 없음
- **권고**: Phase 20-B 또는 별도 작업에서 로그인된 어드민의 역할(role)을 읽어 `super_admin`이 아니면 해당 메뉴 항목을 숨기거나 접근 시 권한 오류 표시

## 5. 후속 권고 사항

### 5.1 BUG-20A-04 — super_admin 전용 메뉴 권한 제어 (Phase 20-B 권고)

현재 어드민 로그인 후 역할 정보(`isSuperAdmin`)가 `admin.js` 내부 변수로만 관리됨. 새 셸(`admin-shell.js`)에서 이 값을 읽으려면 `window.isSuperAdmin` 등으로 전역 노출 후 NAV_GROUPS 빌드 시점에 `anon-audit` 항목을 조건부 제외하는 방식 권고.

### 5.2 가입 승인 탭 데이터 연결 (Phase 20-C 계획)

가입 승인 탭의 승인·거절 버튼이 현재 알림창만 표시. 실제 승인 처리 API 연결은 Phase 20-C에서 계획됨. 현재는 설계상 의도된 플레이스홀더.

### 5.3 Q11 기존 TypeScript 에러 12건

Phase 20-A와 무관한 기존 에러. 주요 항목:
- 어드민 신고 처리 함수들의 schema 컬럼 불일치 (`fromStatus`, `notifiedAt`, `reason` 미등록 컬럼)
- 게시글 구독·멘션 읽음 처리의 `isRead`, `postId` 컬럼 미등록
- 별도 마이그레이션·schema 동기화 작업 필요 (BUG-20A 범위 외)
