## §4 관찰 사항 (BUG 아님)

### 4.1 카카오 주소검색 — 결과 0건 (운영 액션 OP-3 후속 확인 권장)

- 키 등록 확인됨 (200 응답 + 빈 results = 카카오 API 정상 호출됨)
- 단 `query=강남대로`·`query=서울 강남구 테헤란로 152` 모두 0건 반환
- 카카오 주소 API 특성상 정확한 지번/도로명 표기 요구 → 운영자가 첫 사용 시 안내 필요
- 또는 KAKAO_REST_API_KEY가 다른 종류(JavaScript 키 등)일 가능성 — Swain이 developers.kakao.com에서 키 종류 재확인 권장

### 4.2 휴가종류 DELETE — 설계서 vs 구현 불일치 (의도된 정정)

- 설계서 §5.1 A-H4: 휴가종류 DELETE를 `body { id }`로 명시
- 실제 구현: 거점·공휴일과 동일하게 `?id=` 쿼리로 통일
- **A의 판단이 더 합리적** — 3종 자원 CRUD 패턴 일관성 확보. 백엔드(`admin-att-leave-types.ts:214`)도 `?id=` 수령으로 정합. 보고는 PASS, 설계서가 사후 정정될 필요 있을지 메인 판단.

### 4.3 활성 분기 부재 (2026-05-13 C 라이브 검증 이슈-1 재확인)

- `milestone-quarters` 응답: 2025 Q1·Q2 SETTLED / 2026 Q3·Q4 UPCOMING / **2026 Q2 부재 + 현재 ACTIVE 분기 0**
- 운영 액션 OP-1 (Swain 직접 등록) 아직 미수행으로 보임
- 코드는 ACTIVE 없을 때 최신 SETTLED 자동 선택하므로 UI는 동작하나, 실제 매출 입력 대상 분기가 없음
