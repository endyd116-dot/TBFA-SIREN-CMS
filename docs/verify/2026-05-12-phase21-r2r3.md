## 3. 발견·수정한 BUG

### BUG-21R2R3-01 — 알림 시간이 화면에 비어있음 (Q9·Q10 영향)

**증상**: 알림 벨 드롭다운과 알림 전체 페이지에서 각 알림의 "몇 분 전" 시간이 빈 문자열로 표시됨. 사용자는 알림이 언제 왔는지 알 수 없음.

**원인**: 백엔드는 알림 시각을 `sentAt` 필드로 응답하는데, 프론트는 `n.createdAt`를 읽음. 시간을 변환하는 함수가 빈 값을 받으면 빈 문자열 반환.

**fix**: 프론트 두 곳에서 `n.sentAt || n.createdAt`로 fallback 처리 (백 응답 형태 유지, 프론트만 수정).

**검증**: 알림 시간이 "방금 전 / N분 전 / N시간 전 / N일 전 / 날짜" 정상 표시.

## 5. 명세 대비 잔여·주의 사항

- **R&R 응답 키 호환성 (BUG-21R2R3-02 후보, 영향 없음)**: 백 응답이 `data.items`이고 프론트 폴백에 `data.items || data.mappings`가 남아있음. 백이 `items`로 고정됐으니 프론트 폴백은 죽은 코드지만 동작에 영향 없음. 차후 청소 권장.
- **cron 중복 발송 방지**: 24h/72h 알림 중복은 알림 본문 마커로 검사. 정상 동작이지만 알림 양이 많아질 경우 인덱스 활용 검사 권장.
- **카드 done → 서비스 closed**: `closeServiceFromTask`는 origin 잠금(`_activeSync` Set)으로 무한 루프 방지. 코드 흐름은 정상이나 라이브 동작은 사용자 검증 필요.
