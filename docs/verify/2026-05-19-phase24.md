## 3. 주요 확인 사항

### 3.1 라이브 HTTP (Q1~Q11)
- 정적 파일 5개 모두 200 응답
- API 6개 모두 인증 없이 401 응답 (인증 가드 정상 동작)

### 3.2 코드 정합성 (Q12~Q20)

**워크스페이스 패널 통합 (Q12·Q19)**
- `workspace.html` 내에 `wsMilestonePanel`과 `wsDashboard` 두 패널 공존
- `workspace-milestones.js`는 `DOMContentLoaded`에서 `/api/auth-me` 호출 → `state.member` 설정 → `loadDashboard()` 순서로 초기화되므로, `wsSwitchPanel` 호출 전에 이미 데이터가 로드됨
- 비매출 제출 모달(`nrModal`)·반려 모달(`rejectModal`) 모두 `workspace.html` 내에 존재

**성과 관리 링크 정합 (Q13)**
- 6개 워크스페이스 서브페이지 전원 `/workspace.html#milestones` 사용
- `workspace-milestones.html` 파일 자체는 여전히 존재하나, 어떤 네비게이션 링크도 이 파일을 직접 참조하지 않음

**어드민 구조 (Q14·Q15)**
- `admin.html` 성과 관리 서브메뉴 4개 항목 모두 `data-super-only="1"` 그룹에 포함
- 각 항목에 대응하는 iframe 패널 4개 존재
- `admin-milestones.html` 4탭 버튼 정상

**DB 스키마 (Q16)**
- `members` 테이블 `milestoneRole` 컬럼 정의 (line 305)
- 5개 성과 관리 테이블 정의 모두 `schema.ts` 말미에 추가됨 (lines 3416~3510)

**API 설정 (Q17·Q18)**
- 11개 API 파일 전원 `export const config = { path: "..." }` line 6에 존재
- `cron-milestone-quarter`는 `config.schedule` 포함
- `requireAdmin` 패턴: 10개 파일 모두 `auth.res` 사용, 잘못된 키 없음

**CSS 회귀 (Q20)**
- `body` 태그: `class="ws-body ms-body"` — `ms-body`는 CSS 변수 정의만 (`--brand`, `--danger` 등), 기존 `ws-body` 스타일에 간섭 없음
- 기존 대시보드 섹션(일일 브리핑·우선순위·2×2 그리드·피드) 모두 `wsDashboard` 내 유지
