## 3. 발견·수정한 BUG

### BUG-21R4-01 — 활동 피드 항목 모두 클릭 비활성 (Q16 미충족)

**증상**: 워크툴 활동 피드의 모든 항목이 클릭 비활성(회색 텍스트)으로 표시됨. 사용자가 어떤 활동을 눌러도 해당 작업으로 이동할 수 없음.

**원인**: 프론트는 활동 피드 항목에 "이동 링크가 있으면 클릭 가능, 없으면 비활성" 분기를 적용. 그런데 백엔드가 응답에 이동 링크 필드를 넣지 않아 모든 항목에 링크가 비어있는 상태로 전달됨. 활동 로그 테이블에 해당 컬럼이 없는 구조.

**fix**: 백엔드 활동 피드 응답에서 작업 관련 항목(`targetType="task"`)에 한해 이동 링크를 동적으로 만들어 응답에 포함. R1·R2+R3에서 사용하는 일관된 형식 (`/workspace-kanban.html#task=ID`)을 사용해 카드 모달이 자동 오픈되도록 함.

**검증**: 활동 피드의 작업 관련 항목 클릭 시 WBS 페이지로 이동 + 해당 카드 상세 모달 자동 오픈.

## 5. 명세 대비 잔여·주의 사항

- **활동 매핑 캐시버스터 일관성**: `workspace-activity-render.js`가 HTML에서 `?v=1`로 참조되지만 설계서는 `?v=2` 권장. 라이브에서 두 버전 모두 200 OK이고 파일은 같은 내용 — 사용자 영향 없음. 차후 일관성 갱신 권장.
- **AI 검색 로딩 표시**: 설계서 §3.4는 "AI가 검색어를 해석 중이에요..." 로딩 토스트 명시. 코드 검토 시 명시적 로딩 UI 부재 — 현재 버튼 비활성으로 처리. 사용자 영향 작지만 검색 응답이 1~2초 걸릴 수 있어 안내 보강 검토 권장.
- **마감 cron (Phase 21 R2+R3 영역)**: 매일 KST 09:00 발송. 본 라운드 검증 범위 외, 다음 09:00 후 사용자 확인 필요.
