## 3. 발견·수정한 BUG

### BUG-21R1-01 — 워크툴 새 작업 버튼이 WBS 모달을 자동으로 열지 않음 (Q5 미충족)

**증상**: 워크툴(workspace.html)에서 "새 작업" 버튼 클릭 → WBS 페이지로 이동만 발생, 작업 생성 모달은 열리지 않음. 사용자가 WBS 도착 후 다시 "새 작업" 버튼을 눌러야 모달이 보임.

**원인**: `WorkspaceTaskModal.openCreate()`는 현재 페이지가 WBS가 아닐 때 `location.href = '/workspace-kanban.html'`로 단순 이동만 하고, 도착한 WBS 페이지에는 모달을 자동 오픈하는 진입점이 없었음.

**fix**:
- 워크툴 측: WBS 이동 시 `#new-task` hash를 함께 부착
- WBS 측: 페이지 로드 시 `#new-task` hash 감지 → 자동으로 새 작업 모달 오픈 + URL 정리

**검증**: 워크툴 "새 작업" 버튼 → WBS 페이지 도착 후 즉시 모달 오픈, 사용자는 한 번의 클릭으로 입력 시작 가능.

### BUG-21R1-02 — 없는 작업 ID 입력 시 사용자 피드백 누락 (Q8 미충족)

**증상**: URL에 직접 `/workspace-kanban.html#task=999` (없는 ID) 입력 시, WBS 페이지가 열리지만 모달도 안 뜨고 토스트도 없음. 사용자는 무엇이 잘못됐는지 알 수 없음.

**원인**: 페이지 로드 후 hash로 카드를 열려고 하는데, 활성 작업 목록에서 못 찾으면 그냥 조용히 종료.

**fix**: hash로 들어온 작업 ID가 활성 목록에 없을 때 "작업을 찾을 수 없어요. 삭제됐을 수 있습니다." 토스트 표시 + URL hash 정리.

**검증**: 없는 ID 입력 시 빨간 토스트로 즉시 안내 + URL이 깔끔한 `/workspace-kanban.html`로 복원됨.

## 6. 한계·후속 권장
