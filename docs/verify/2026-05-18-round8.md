## 2. 버그 발견 및 수정

### BUG-R8-1 (🔴 Critical) — 프론트 수정/삭제 기능 실 API 미연결

**발견 위치**: `public/js/my-reports.js`, `public/js/mypage-applications.js`

**현상**: A가 B 머지 전 MOCK 데이터로 구현 후 push했으나, B 머지 후 실 API 전환이 누락됨.
- `my-reports.js`: 사건/괴롭힘/법률 수정 3종 + 법률 삭제 1종이 MOCK 응답 반환 (실 API 호출 주석 처리)
- `mypage-applications.js`: 유가족 지원 신청 수정도 동일하게 MOCK 사용
- 추가로 삭제 호출 HTTP 메서드가 `POST`였으나 서버는 `DELETE`만 허용

**수정 내용**:
1. `my-reports.js` — MOCK 상수 4개 제거, 실 API fetch 블록 활성화 (삭제 메서드 POST → DELETE 수정)
2. `mypage-applications.js` — MOCK 상수 제거, api() 헬퍼로 실 API 호출 활성화

### BUG-R8-2 (🔴 Critical) — 동기화 로직이 유효하지 않은 상태값 'resolved' 사용

**발견 위치**: `netlify/functions/admin-workspace-tasks.ts` (Round 8 sync 블록)

**현상**: 워크스페이스 작업 완료 시 연결 보고/신청 상태를 `'resolved'`로 업데이트하도록 구현됐으나, `resolved`는 해당 테이블 enum에 존재하지 않음. PostgreSQL enum 오류 → sync 무음 실패.

**schema 실제 상태값**:
| 테이블 | 완료 상태 |
|---|---|
| `support_requests` | `completed` |
| `incident_reports` | `closed` |
| `harassment_reports` | `closed` |
| `legal_consultations` | `closed` |

**수정 내용**: `tableMap`을 `{ table, status }` 매핑으로 교체하여 각 테이블에 맞는 올바른 완료 상태 사용
