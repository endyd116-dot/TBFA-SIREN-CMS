### Q1 — 마이그 role='operator' → 'admin' 변환

- `6e69a0f chore: migrate-round4-rbac.ts 삭제 (1회용 마이그 완료)` 커밋 확인
- 마이그 SQL: `UPDATE members SET role = 'admin' WHERE role = 'operator' AND type = 'admin';`
- 호출 완료 후 파일 삭제 → 기존 operator 계정 role='admin'으로 전환됨

### Q2 — 로그인 role 하드코딩 제거

`admin-login.ts` 123·141행:
```
role: user.role ?? "operator"   // DB 실제 role 반환, 폴백만 "operator"
```
수정 전 하드코딩 `role: "super_admin"` 제거됨 — 로그인 후 JWT에 실제 role 담김

### Q3·Q4 — 감사 로그 super_admin 전용 접근

`admin-audit.ts` 38-39행:
```typescript
if (!requireRole(member, "super_admin")) return roleForbidden("super_admin");
```
- super_admin(rank=3): 3 >= 3 → 통과 ✅ (Q3)
- admin(rank=2): 2 >= 3 → false → 403 + `"super_admin 권한이 필요합니다"` ✅ (Q4)

### Q5·Q6 — 회원 블랙 처리 admin 이상 필요

`admin-members-blacklist.ts` 27행:
```typescript
if (!requireRole(guard.ctx.member, "admin")) return roleForbidden("admin");
```
- admin(rank=2): 2 >= 2 → 통과 ✅ (Q5)
- operator(rank=1): 1 >= 2 → false → 403 + `"admin 권한이 필요합니다"` ✅ (Q6)

### lib/admin-role.ts 헬퍼 구현

```typescript
const ROLE_RANK: Record<string, number> = {
  super_admin: 3,
  admin: 2,
  operator: 1,
};
requireRole(member, minRole): rank >= ROLE_RANK[minRole]  // 이상(inclusive) 비교
roleForbidden(requiredRole): 403 JSON 응답
```
