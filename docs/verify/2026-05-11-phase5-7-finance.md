## 3. BUG-5·BUG-6·BUG-7 fix 살아있는지 점검 (직전 라운드 회귀 방지)

세 버그 모두 직전 C 검증 라운드에서 발견·수정. Phase 8·9·10 R1·R2 머지 동안 덮어쓰여진 흔적이 없는지 확인.

### 3.1 BUG-5 — 감사 추적 컬럼 NULL 저장 (✅ fix 유지)

| 점검 | 결과 |
|---|---|
| 예산 편성 INSERT 시 어드민 ID 채움 | ✅ — `auth.ctx.admin.uid` 직접 참조 (예전 잘못된 `auth.admin?.id`로 회귀하지 않음) |
| 지출 기안 INSERT 시 기안자 ID 채움 | ✅ — `auth.ctx.admin.uid` 그대로 |
| 지출 승인·반려 UPDATE 시 처리자 ID 채움 | ✅ — `auth.ctx.admin.uid` 그대로 |

→ 누가 작업했는지 DB에 정확히 기록되는 감사 추적 기능 정상 동작.

### 3.2 BUG-6 — 지출 명단 마비 (✅ fix 유지)

| 점검 | 결과 |
|---|---|
| 지출 명단 SELECT의 기안자·승인자 이름 JOIN | ✅ — `members` 테이블로 정상 JOIN (예전 존재하지 않는 `admins` 테이블 참조로 회귀하지 않음) |

→ 어드민이 재정 관리 화면 진입 시 지출 명단이 정상 표시. 500 에러 없이 200 응답 가능.

### 3.3 BUG-7 — 이미 처리된 지출에 거짓 성공 응답 (✅ fix 유지)

| 점검 | 결과 |
|---|---|
| UPDATE에 `RETURNING id` 추가 | ✅ — 영향 받은 행 ID 반환 |
| 영향 행 수가 0이면 409 응답 | ✅ — "대기 상태가 아니거나 존재하지 않는 지출입니다" |

→ 이미 승인·반려된 지출에 다시 처리 호출 시 명확히 거절 + 가짜 성공 메시지 안 나감. 감사 로그 무결성 보장.

## 6. Swain 직접 클릭 검증이 필요한 항목

세션 쿠키 흐름이 필요. 코드 정합성으로는 통과지만 다음은 클릭으로 최종 확인 권장:

1. **예산 편성·집계**:
   - 어드민 → 재정 관리 → 카테고리·연도 선택 → 예산 입력 → 저장 → 집계 화면에 계획 금액 반영
   - 같은 카테고리에 다시 편성 → 갱신 확인 (중복 행 X)
2. **지출 기안 → 승인 → 집행률 반영**:
   - 지출 기안 등록 → 명단에 대기 상태로 표시
   - 승인 처리 → 상태 `approved` + 승인자·승인시각 표시 + 예산 집계 화면의 집행 금액에 반영
3. **반려 흐름**:
   - 다른 지출 기안 → 반려 처리 → 상태 `rejected`
   - 같은 지출에 다시 승인 시도 → "대기 상태가 아니거나 존재하지 않는 지출입니다" (BUG-7 fix 검증)
4. **명단 필터**:
   - 상태(대기/승인/반려) + 카테고리 + 연도 동시 적용 → 결과 정확
5. **수입 요약 화면**:
   - 연간 추이 차트 12개월 표시 + 채널별 집계 표
   - 효성 CMS 후원의 결제일이 시스템 입력 시각이 아니라 실제 결제일 기준으로 집계되는지 (Q12 fix 검증)
6. **재무 보고서**:
   - 종합 재무 현황 화면이 예산·집행·수입 모두 일치하는지
