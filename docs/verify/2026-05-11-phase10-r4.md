## 5. 발견 이슈

### BUG-9 — schema.ts 컬럼 누락 (✅ 즉시 수정)

| 항목 | 내용 |
|---|---|
| 발견 위치 | `db/schema.ts` — `communicationSendRecipients` + `communicationSendJobs` |
| 심각도 | 🟠 High (런타임 정상, `db:push` 시 컬럼 DROP 위험) |
| 원인 | R4 마이그레이션("6항목 성공") 적용 후 신규 테이블 3개만 schema 활성화; 기존 테이블 컬럼 추가분이 schema.ts에 미반영 |
| 영향 | 코드는 모두 raw SQL 사용이라 런타임 정상. 단 `npm run db:push` 실행 시 Drizzle이 DB의 신규 컬럼을 "불필요" 판정하여 DROP 쿼리 발생 가능 |

**수정 내용 (`db/schema.ts`)**

`communicationSendJobs`에 추가:
```typescript
triggeredByAutoId: integer("triggered_by_auto_id"),
```

`communicationSendRecipients`에 추가:
```typescript
trackingToken:    varchar("tracking_token", { length: 64 }),
openedAt:         timestamp("opened_at"),
clickedAt:        timestamp("clicked_at"),
openCount:        integer("open_count").notNull().default(0),
clickCount:       integer("click_count").notNull().default(0),
// 인덱스
trackingTokenIdx: index("send_recipients_tracking_token_idx").on(t.trackingToken),
```

`uniqueIndex` 대신 `index` 사용 이유: DB에는 `WHERE tracking_token IS NOT NULL` 부분 유니크 인덱스가 있으나 Drizzle이 표현 불가 → 일반 `index`로 선언 (컬럼 DROP 방지 목적 달성, 기능 동일)
