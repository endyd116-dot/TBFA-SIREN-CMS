## 3. Q7 — Phase 9-A SMS 실발송 검증

### 3.1 파이프라인 구조 (3계층)

```
이벤트 발생(결제실패·카드만료) → notify-dispatcher → smsAligoAdapter → aligoSend → Aligo API
```

### 3.2 채널 정책 (notify-events.ts)

| 이벤트 | 기본 채널 | 강제 채널 |
|---|---|---|
| `billing.failed` | `[inapp, email, sms, kakao]` | `[inapp, email]` |
| `card.expiring` | `[inapp, email, sms, kakao]` | `[inapp, email]` |

**SMS 전송 조건**: 회원의 `phone_verified_at` 값이 있어야 발송 (미인증 → 필터 제외)

| # | 시나리오 | 코드 경로 | 판정 |

### 3.5 실제 발송 현황

| 항목 | 상태 |
|---|---|
| Aligo 계정·발신번호 등록 | ✅ 완료 (2026-05-10 Swain 확인) |
| ALIGO_API_KEY·ALIGO_USER_ID·ALIGO_SENDER Netlify 등록 | ⚠️ **Swain 직접 확인 필요** (코드에서 검증 불가) |
| 실제 SMS 발송 테스트 | ⏸ **Swain 직접 검증** 필요 — 방법 §3.6 참고 |

### 3.6 Swain 실발송 검증 방법

1. Netlify 콘솔 → Site settings → Environment variables에서 `ALIGO_API_KEY`, `ALIGO_USER_ID`, `ALIGO_SENDER` 3개 모두 등록됐는지 확인
2. 어드민 로그인 → 알림 로그 페이지(`/admin-notifications.html`) 접속
3. 특정 회원에게 결제 실패 이벤트를 수동으로 발생시키거나, 테스트 cron 호출:
   - 또는 `NOTIFICATION_TEST_MODE=true` 상태에서 빌링 cron 실행 → 콘솔 로그 확인
4. 알림 로그에서 `channel=sms`, `status=sent`, `provider_message_id`에 Aligo msgId 기록 확인

## 4. Q8 — Phase 9-B 카카오 알림톡 검증

### 4.1 파이프라인 구조 (3계층)

```
이벤트 발생 → notify-dispatcher → kakaoAligoAdapter → buildAlimtalk() → sendAligoAlimtalk() → Aligo 알림톡 API
                                                      ↓ (템플릿ID 미등록/채널키 미등록/테스트모드/번호없음)
                                              placeholder → console.log + ok=true 반환
```

### 4.2 카카오 발송 조건 (이중 게이트)

1. **dispatcher 레벨**: `kakao_marketing_consent_at` 있어야 kakao 채널 포함
2. **어댑터 레벨**: `ALIGO_KAKAO_CHANNEL_ID` + 템플릿 코드 있어야 실 발송, 없으면 placeholder

### 4.3 Placeholder 폴백 조건 (4가지)

| 조건 | 트리거 | 현재 상태 |
|---|---|---|
| 템플릿 ID 미등록 | `ALIGO_TEMPLATE_BILLING_FAILED` 또는 `ALIGO_TEMPLATE_CARD_EXPIRING` 비어있음 | ⚠️ **현재 해당** — 카카오 심사 대기 |
| 채널 키 미등록 | `ALIGO_KAKAO_CHANNEL_ID` 비어있음 | 미확인 |
| 테스트 모드 | `NOTIFICATION_TEST_MODE=true` | 환경에 따라 다름 |
| 수신자 번호 없음 | 회원 phone 없거나 빈 값 | 회원별 |

**현재 상태**: 템플릿 ID 미등록 → 항상 placeholder 모드 → 실발송 없음, 로그만 기록
