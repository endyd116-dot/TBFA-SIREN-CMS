## 4. 코드 정합성 점검 (정독)

### 4.1 V1 — 보고서 생성·조회

**보고서 생성 흐름**
- ✅ 어드민 인증 → 본문에서 기간 시작·끝 받음 → ISO 날짜 검증
- ✅ 통계 수집(회원·후원·신고·매칭·유족지원 등 도메인별 합계)을 별도 라이브러리로 위임
- ✅ Gemini로 5줄 요약 + 위험 경보 생성. **AI 실패 시 휴리스틱 폴백** (수치 기반 자동 5줄 요약 — "이번 기간 신규 회원 N명 가입" 등)
- ✅ 보고서 행 INSERT → 응답에 새 보고서 ID 즉시 반환
- ✅ 단계별 try/catch + step 라벨(`parse_body`, `collect_stats`, `insert_report`) + detail/stack — CLAUDE §6.2 준수

**보고서 목록·상세 흐름**
- ✅ 목록: 최신순 정렬 + page·limit 페이지네이션 (limit 최대 100)
- ✅ 응답 구조 `data.reports` + 클라이언트 다중 fallback(`payload.reports ?? res.data.reports`)
- ✅ 상세: 행 1건 조회 + 통계·AI 요약·경보 모두 반환

### 4.2 V2 — 이메일 재발송 (BUG-3 fix 확인)

- ✅ 어드민 인증 → 본문에서 보고서 ID 받음 → 정수 검증
- ✅ 보고서 행 조회 (없으면 404)
- ✅ **수신자 자동 구성**: 환경변수 `ADMIN_NOTIFY_EMAIL` + super_admin 등급 회원의 이메일 합집합 (중복 제거)
- ✅ 수신자 0명이면 안내 메시지 반환 (환경변수·super_admin 확인 권유)
- ✅ Resend로 발송 → 부분 실패 허용 (한 명 발송 실패해도 나머지는 진행) + console.warn으로만 기록
- ✅ 발송 성공 후 보고서 행에 `sentEmailAt`·`sentTo` 갱신 (실패해도 응답은 정상)
- ✅ 응답에 발송 건수·수신자 명단 반환

### 4.3 V3 — 인쇄 흐름

- ✅ 보고서 상세 화면의 인쇄 버튼 클릭 → 브라우저 `window.print()` 호출
- ✅ 인쇄 CSS가 `<link media="print">`로 연결돼 있어 인쇄 시에만 활성화
- ✅ @media print 안에서 사이드바·헤더·작용 버튼 숨김 + 본문만 A4 형식 (216줄 인쇄 전용 CSS)

### 4.4 부가 — 자동 cron

- ✅ 일요일 21:00 UTC = KST 월요일 06:00에 자동 실행
- ✅ reportType이 `weekly`로 INSERT (수동 보고서는 `custom`)
- ✅ 자동 cron도 같은 lib로 통계 수집 + AI 요약 + DB 저장 + 이메일 발송

2. **V2 이메일 재발송 (BUG-3 fix 후)**:
   - 보고서 상세 → "📧 이메일 재발송" 버튼 → 확인 → "이메일이 재발송되었습니다" 토스트

3. **V3 인쇄**:
   - 보고서 상세 → "🖨️ 인쇄" 버튼 → 브라우저 인쇄 미리보기
