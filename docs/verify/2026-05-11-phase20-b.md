## 2. 발견한 BUG 6건 (상세)

### BUG-20B-01 [HIGH · 해결] 사이드바 "발송 템플릿/수신자 그룹/발송 분석·로그" 클릭 시 빈 화면

- **증상**: 사이드바에서 발송 그룹 하위 3개 메뉴("발송 템플릿", "수신자 그룹", "발송 분석·로그") 클릭 시 빈 화면 표시
- **원인**: 각 메뉴의 `viewId`가 `adm20-send-templates`, `adm20-send-groups`, `adm20-send-analytics`로 설정되어 있으나, 실제 5탭 UI는 `adm20-send-jobs` 컨테이너 하나에 모두 빌드됨. 라우터가 빈 div를 활성화하고 탭 컨테이너는 숨김
- **수정**:
  - `admin-shell.js` NAV_GROUPS: 3개 메뉴의 `viewId`를 `adm20-send-jobs`로 통일하고 `tab` 필드 추가
  - `admin-shell.js` `buildNav()`: `data-tab` 속성 전달
  - `admin-shell.js` `navigate()`: `tab` 파라미터 추가 → `window.AdminSendGroup.switchTab(tab)` 호출
  - `admin-send-group.js`: `window.AdminSendGroup = { switchTab }` 공개 API 추가
- **수정 파일**: `public/js/admin-shell.js`, `public/js/admin-send-group.js`
- **심각도**: HIGH — 발송 메뉴 4개 중 3개 진입 불가

### BUG-20B-02 [HIGH · 해결] 사이드바 "AI 활동보고서/AI 자동 발송 트리거" 클릭 시 빈 화면

- **증상**: AI 에이전트 그룹 하위 2개 메뉴 클릭 시 빈 화면 표시 (BUG-20B-01과 동일 구조)
- **원인**: `adm20-ai-activity`, `adm20-ai-triggers` 빈 div로 라우팅됨. 실제 UI는 `adm20-ai-recommend` 하나에 3탭으로 빌드
- **수정**:
  - `admin-shell.js` NAV_GROUPS: 2개 메뉴의 `viewId`를 `adm20-ai-recommend`로 통일, `tab` 필드 추가
  - `admin-shell.js` `navigate()`: `window.AdminAiGroup.switchTab(tab)` 호출
  - `admin-ai-group.js`: `window.AdminAiGroup = { switchTab }` 공개 API 추가
- **수정 파일**: `public/js/admin-shell.js`, `public/js/admin-ai-group.js`
- **심각도**: HIGH — AI 에이전트 메뉴 3개 중 2개 진입 불가

### BUG-20B-03 [MEDIUM · 해결] 재정 화면에 실 API 데이터 미표시

- **증상**: 재정 그룹 진입 시 모든 항목이 0 또는 비어있음
- **원인**: 서버 API는 `income`(지출 배열), `budget`(예산 배열), `report`(집계 요약 객체)를 평탄한 구조로 반환하는데, 화면은 `income.total`, `income.byCategory`, `budget.categories` 등 중첩 객체 형식으로 기대. 구조 불일치로 모든 값이 `undefined` → 0
- **수정**: `admin-finance-group.js`에 `transformFinanceData()` 변환 함수 추가. 평탄한 배열을 화면이 기대하는 중첩 객체 형식으로 변환 후 렌더
- **수정 파일**: `public/js/admin-finance-group.js`
- **심각도**: MEDIUM — 재정 화면 데이터 전체 미표시

### BUG-20B-04 [HIGH · 해결] 발송 작업 목록 화면에 필드명 불일치로 빈 값 표시

- **증상**: 발송 작업 탭에 "발송 작업 목록"은 표시되나 모든 행의 제목·채널·발송 수·실패 수가 `-` 또는 0으로 표시
- **원인**: 실 API 응답 필드명과 화면 렌더 함수의 필드명이 다름
  - `title` → 실제 `name`
  - `type` → 실제 `channel`
  - `sentCount` → 실제 `successCount`
  - `failCount` → 실제 `failureCount`
  - 로그: `recipient` → `memberId`, `subject` → 미제공
- **수정**: `admin-send-group.js` 렌더 함수 내 필드명을 실 API 응답 기준으로 수정 (fallback 포함)
- **수정 파일**: `public/js/admin-send-group.js`
- **심각도**: HIGH — 발송 작업 목록·알림 로그 데이터 전체 미표시

### BUG-20B-05 [HIGH · 해결] AI 에이전트 화면 응답 구조 완전 불일치

- **증상**: AI 추천 센터·활동보고서·자동 트리거 탭 모두 오류 또는 빈 화면
- **원인**: mock 설계 시 사용한 응답 구조와 실 API 응답 구조가 완전히 다름
  - **추천**: mock `{ title, summary, priority, score, actions }` vs 실 `{ triggerName, successRate, totalRuns, suggestion }`
  - **활동보고서**: mock `{ period, summary, highlights, generatedAt }` vs 실 `{ date, total, ok, skipped, error }`
  - **트리거**: mock `{ condition, templateName, lastFiredAt, firedCount }` vs 실 `{ triggerType, description, delayHours, cooldownDays }`
- **수정**: 세 렌더 함수(renderRecommend, renderActivity, renderTriggers)를 실 API 응답 형식에 맞게 전면 재작성
- **수정 파일**: `public/js/admin-ai-group.js`
- **심각도**: HIGH — AI 에이전트 그룹 전체 표시 불가

### BUG-20B-06 [HIGH · 해결] `/api/admin/me` 응답에서 role 하드코딩 → super_admin 접근 제어 무력화

- **증상**: 일반 admin 계정으로 로그인해도 익명 감사 로그 메뉴가 표시됨 (BUG-20A-04 fix 의도와 반대)
- **원인**: `netlify/functions/admin-me.ts`에서 `role: "super_admin"`으로 하드코딩됨. `admin-shell.js`가 이 값을 읽어 역할 판별하므로, 모든 어드민이 super_admin으로 처리됨
- **수정**: `role: "super_admin"` → `role: guard.ctx.member.role` (DB에서 읽은 실제 역할 반환)
- **수정 파일**: `netlify/functions/admin-me.ts`
- **심각도**: HIGH — 권한 분리 기능 전체 무력화

## 5. 후속 권고 사항

### 5.1 발송 수신자 그룹 — memberCount·channels 필드 미제공
현재 `recipientGroups` 테이블에 `memberCount`(수신자 수)와 `channels`(허용 채널 배열)이 없어 해당 값이 `-`로 표시됨. 20-C에서 memberCount 집계 쿼리 추가 또는 테이블 컬럼 추가 권고.

### 5.2 발송 로그 — recipient 이메일·전화 미노출
`communicationSendRecipients`에 수신자 식별자는 `memberId`만 있고 이메일/전화가 없어 회원 번호로만 표시됨. 필요 시 members 테이블과 별도 조회로 개인정보 표시 (마스킹 정책 적용 필요).

### 5.3 재정 수입 — donations 테이블 연계 미구현
현재 재정 API의 `income`은 `expenditures`(지출) 테이블을 반환. 실제 "수입"에 해당하는 donations 통계는 별도 연결 필요. 20-C에서 `admin-finance-unified.ts` 확장 권고.

### 5.4 기존 TypeScript 에러 12건 (Phase 20-A 보고와 동일)
`admin-anonymous-reveal.ts`, `admin-report-status-update.ts` 등의 schema 컬럼 불일치. Phase 20-B 범위 외.
