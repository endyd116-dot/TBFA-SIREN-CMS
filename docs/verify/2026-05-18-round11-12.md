### 🚨 BUG-R11-2 (Critical) — 미수정 지속

`admin-comment-reports.js submitReview()`에서 `status` 필드 누락.
백엔드 400 "유효하지 않은 status" → BUG-R11-1 MOCK 폴백이 가려 UI 성공처럼 보임 → 신고 DB 미갱신 + 알림 미발송.

**예상 1줄 fix**:
```javascript
var status = action === 'none' ? 'dismissed'
           : action === 'delete_comment' ? 'resolved' : 'reviewed';
body: { reportId: parseInt(currentReportId, 10), action, status }
```

### 🚨 BUG-R11-3 (Major) — 미수정 지속

리마인더 cron(`cron-workspace-task-reminder.ts`)이 참조하는 `reminder_config->>'remindAt'` 타임스탬프가 PATCH API·프론트 어디서도 계산·저장되지 않음 → cron 조건 `IS NOT NULL` 미충족 → 발송 불가.

### 🚨 BUG-R12-1 (Critical) — 신규

**발견 위치**: `public/js/mypage-notifications.js`

**현상**: 마이페이지 알림 탭이 기존 헤더 벨(`notification-bell.js`)과 **다른** API 경로·메서드를 사용해 항상 404/405 오류 → MOCK 폴백 → 가짜 알림 표시 + 실제 DB 읽음 갱신 불가.

| | 프론트 `mypage-notifications.js` | 헤더 벨 `notification-bell.js` (정상) |
|---|---|---|
| 목록 조회 | `GET /api/notifications-mine` → **404** | `GET /api/notifications/mine` → 401 ✅ |
| 단건 읽음 | `PATCH /api/notifications-read {id}` → **404/405** | `POST /api/notifications/read {id}` → 401 ✅ |
| 전체 읽음 | `PATCH /api/notifications-read {all:true}` → **404/405** | `POST /api/notifications/read {all:true}` → 401 ✅ |

**구체 영향**:
1. `loadNotifications()`: 목록 조회 → 404 → `MOCK_NOTIFICATIONS` (가짜 2건: 태스크 멘션·법률 배정) 표시
2. `handleItemClick()`: 읽음 처리 → 404 → `is_read` DB 미갱신 → 페이지 재진입 시 미읽음 복원
3. `handleReadAll()`: 전체 읽음 → 404 → UI만 업데이트, DB 갱신 안 됨 → 재진입 시 복원
4. `MOCK_NOTIFICATIONS` 상수: 동일한 MOCK 폴백 패턴 (BUG-R11-1 동류)

**예상 fix**: 경로·메서드를 기존 API와 통일
```javascript
// loadNotifications()
fetch('/api/notifications/mine', ...)  // /api/notifications-mine → /api/notifications/mine

// handleItemClick() / handleReadAll()
fetch('/api/notifications/read', { method: 'POST', ... })  // PATCH → POST, -read → /read
```
