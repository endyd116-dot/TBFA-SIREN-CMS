## 3. 발견·수정된 버그

### BUG-15-01 (Critical) — AI 추천 시 법률 상담 테이블명 오류
- **증상**: 법률 신고(`sourceType=legal`)로 AI 추천 요청 시 쿼리가 존재하지 않는 테이블 `legal_reports`를 조회해 빈 내용 전송
- **원인**: 백엔드 tableMap에 `legal: "legal_reports"` 오기입, 실제 테이블명은 `legal_consultations`
- **수정**: tableMap을 `{ table, contentCol }` 구조로 변경, `legal` → `"legal_consultations"`
- **파일**: `netlify/functions/admin-ai-expert-recommend.ts`

### BUG-15-02 (High) — AI 추천 시 content 컬럼명 오류
- **증상**: 사건 신고·괴롭힘 신고·법률 상담 내용을 빈 문자열로 Gemini에 전달 → AI 적합도 판단 불가
- **원인**: `SELECT title, content, ai_summary` 쿼리 사용, 해당 3개 테이블의 실제 컬럼명은 `content_html` (support_requests만 `content`)
- **수정**: tableMap에 `contentCol` 필드 추가, 테이블별로 올바른 컬럼명 사용
- **파일**: `netlify/functions/admin-ai-expert-recommend.ts`

### BUG-15-03 (Critical) — 프로필 편집 저장 시 없는 API 경로 호출
- **증상**: 전문가 프로필 편집 후 [저장] 클릭 시 404 오류 (`/api/admin-expert-profile-update` 없음)
- **원인**: 프론트가 `/api/admin-expert-profile-update`로 요청, 실제 API 경로는 `/api/admin-expert-profile-upsert`
- **수정**: API 경로 수정
- **파일**: `public/js/admin-expert-profiles.js`

### BUG-15-04 (Medium) — 프로필 목록 유형 필터 동작 안 함
- **증상**: 유형 필터 선택해도 목록이 변경되지 않음
- **원인**: 프론트에서 `?type=lawyer` 파라미터를 백엔드에 전송했으나, 백엔드는 `?all=true` 파라미터만 인식하고 type 필터 처리 없음
- **수정**: 항상 `?all=true`로 전체 조회 후 프론트에서 JS 레벨 필터링
- **파일**: `public/js/admin-expert-profiles.js`

### BUG-15-05 (Critical) — 사용자 별점 UI가 표시되지 않음
- **증상**: 마이페이지에서 종결된 매칭을 봐도 별점 입력 화면이 표시되지 않음
- **원인**: `checkFeedbackStatus()`가 `?matchId` 파라미터 없이 호출되어 서버에서 400 오류, `init()` 함수가 어느 matchId를 체크할지 알 수 없음
- **수정**: `init(matchId)` 형태로 변경, 종결된 매칭 카드에 [⭐ 후기 작성] 버튼 추가 (`mypage-expert-match.js`), 클릭 시 해당 matchId로 init 호출
- **파일**: `public/js/mypage-match-feedback.js`, `public/js/mypage-expert-match.js`

### BUG-15-06 (High) — AI 추천 API 요청 파라미터 불일치
- **증상**: AI 추천 버튼 클릭 시 백엔드에서 `sourceType이 필요합니다` 오류
- **원인**: 프론트가 `{ matchId, matchType }` 전송, 백엔드는 `{ sourceType, sourceId, matchType }` 요구
- **수정**: 백엔드에서 `matchId`를 받으면 `expert_matches` 테이블에서 `source_domain`/`source_id`를 자동 조회하는 로직 추가
- **파일**: `netlify/functions/admin-ai-expert-recommend.ts`
