## 3. 발견된 버그 및 즉시 수정

### BUG-13-01: 차트 레이블 키 불일치 — 상태·심각도 모두 `undefined` 표시 (High)

**증상**: 상태별 막대 차트와 심각도/긴급도 도넛 차트의 레이블이 모두 빈칸  
**원인**:
- 백엔드 `byStatus` 배열: `[{ status: "submitted", count: 5 }]` — `status` 키 사용
- 백엔드 `bySeverity`/`byUrgency` 배열: `[{ level: "high", count: 3 }]` — `level` 키 사용
- 프론트엔드: `x.label`로 접근 → 항상 `undefined`

**수정**: `admin-siren-stats.js`
- 상태 레이블: `statusLabel(x.status || x.label)` — 한국어 매핑 함수 추가 (`STATUS_LABEL_MAP`)
- 심각도 레이블: `severityLabel(x.level || x.label)` — 한국어 매핑 함수 추가 (`SEVERITY_LABEL_MAP`)
- `SEVERITY_COLORS`에 `unknown` 키 추가 (백엔드가 `COALESCE(..., 'unknown')` 사용)

### BUG-13-02: 전체 탭 막대 차트 레이블-값 순서 불일치 (Medium)

**증상**: 전체 탭에서 사건·악성민원·법률지원 상태별 막대 차트의 레이블과 값이 어긋남  
**원인**: 고정 레이블 5개(`접수·검토 중·처리 중·완료·종료`)를 사용하면서 `byStatus` 배열 순서가 DB 반환 순서에 의존 → 레이블-값 불일치 가능

**수정**: `admin-siren-stats.js`
- 실제 반환된 `status` 키 기반으로 동적 레이블 생성
- `STATUS_ORDER` 배열로 정렬 보장
- 각 유형별 `byStatusToMap` 함수로 status→count 매핑
