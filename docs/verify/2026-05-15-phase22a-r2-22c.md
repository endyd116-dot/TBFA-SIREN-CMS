## 4. 발견 신규 BUG (2건)

### BUG-013 (Critical, 회귀) — `pl-summary.ts` `donationRefund` const 잘못 선언

**위치**: `netlify/functions/admin-finance-pl-summary.ts:21,70`

```typescript
let donationGross = 0;
const donationRefund = 0;      // ❌ const로 잘못 선언 (R1 fix는 let이었음)
...
for (const row of refundRows) {
  ...
  donationRefund += t;          // ❌ const에 += → TS2588 컴파일 에러
}
```

**tsc 결과**: `netlify/functions/admin-finance-pl-summary.ts(70,7): error TS2588: Cannot assign to 'donationRefund' because it is a constant.`

**영향**: 22-C 머지(commit `e46f69f`/`a616772`) 시점에 누군가 `let donationRefund` → `const donationRefund`로 변경. Netlify가 esbuild로 빌드하면 tsc 에러를 무시하고 통과할 수 있어 운영 배포 가능성 있음. 그 경우 런타임에서 `TypeError: Assignment to constant variable.` 발생 → P&L API 500. 결과적으로 BUG-002 fix(후원 환불 차감) 동작 무효화.

**fix**: line 21 `const` → `let` 한 글자 변경.

### BUG-015 (High, 22-C 신규) — 지출 환불도 누적이 아닌 덮어쓰기

**위치**: `netlify/functions/admin-expense-refund.ts:64` + `lib/ai-agent-tools.ts:3636-3641` (`tool_expenseRefund`)

```typescript
// API
.set({ refundAmount: Number(refundAmount), updatedAt: new Date() })  // 덮어쓰기

// AI 도구
UPDATE expenses SET refund_amount = ${Number(refundAmount)}, ...
```

22-A BUG-001 fix(C가 5월 14일 작업)와 정확히 같은 패턴이 22-C 환불에 다시 등장. B 백엔드 채팅이 22-C 작업 시 22-A fix 사례를 참고하지 않은 것으로 추정.

**영향**: 동일 지출에 환불 2회 등록 시 1차 환불액 사라짐 → 손익 산식 누적 오차. BUG-001과 동일 심각도 BUT 지출은 상대적으로 등록 빈도가 낮아 High로 분류.

**fix**: `admin-expense-refund.ts` + `tool_expenseRefund` 둘 다 BUG-001 fix 패턴 적용 (`currentRefund + incremental = newTotalRefund` + 누적 초과 검증).
