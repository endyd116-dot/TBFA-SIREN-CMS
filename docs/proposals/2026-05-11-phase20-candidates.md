## 후보 1: 운영 안정성 (외부 에러 모니터링 + 백업·복구)

- **무엇을**: 운영 중 발생하는 서버 에러를 외부 도구에 자동 수집해 어드민이 한 화면에서 통계·발생 빈도·스택을 확인. 동시에 데이터베이스·파일함 백업 정책을 정리하고 정기적으로 "복구 시뮬레이션"을 돌려 실제 복구 가능 여부를 측정.
- **왜**: 현재는 운영 중 에러가 어드민 알림으로만 도착하고 통계·재현이 어려움. 또한 데이터베이스는 호스팅 자동 백업에 의존하고 파일함은 별도 정책이 없어 장애 시 복구 시간 측정이 안 됨. 협회 운영이 커질수록 장애 대응·데이터 유실 사전 차단이 필수.
- **범위**: 신규 테이블 0~1개(에러 통계 캐시) / 외부 모니터링 도구 1종 연동(예: Sentry 또는 Logtail) / 어드민 화면 1개(에러 통계·검색·해소 처리) / 백업 점검 자동화 cron 1개 / 복구 절차 문서 1개
- **예상 기간**: 7~10일 (B 큰 비중, A 작음)
- **B+A 분리 가능 여부**: 가능 — B가 외부 도구 연동·cron·통계 API, A가 어드민 화면 1개. 다만 A 작업량이 작아 다른 후보와 병행 가능
- **회귀 위험**: 낮음 — 운영 중 코드는 거의 건드리지 않고 새로운 모니터링 계층을 위에 얹는 방식. 백업·복구는 비파괴 검증

## 후보 2: 장학금 프로그램 운영 (신규 서비스 도메인)

- **무엇을**: 유가족 자녀 대상 장학금을 신청서 접수 → 어드민 심사·선정 → 지급 처리 → 분기·연간 결과 보고까지 한 흐름으로 운영. 사용자(유가족 회원)는 마이페이지에서 신청 → 진행 단계 확인 → 결과 통보까지 자동 안내.
- **왜**: 협회 핵심 서비스 영역(`CLAUDE.md` §1: "유가족 지원 — 심리상담·법률·**장학**")에 명시되어 있으나 현재까지 미구현. 협회 사명상 핵심 사업이라 외부 후원자에게도 가시성 있는 활동 영역. 다른 협회와 차별화 포인트.
- **범위**: 신규 테이블 3~4개(장학금 신청서·심사 이력·지급 기록·서류 첨부) / 신규 어드민 화면 3개(접수함·심사·지급 관리) / 신규 사용자 화면 2개(신청서 작성·진행 현황) / 첨부 파일은 기존 R2 인프라 재사용
- **예상 기간**: 10~14일 (신규 도메인이라 B·A 모두 큰 비중)
- **B+A 분리 가능 여부**: 가능 — B가 신청·심사·지급 흐름과 첨부 처리, A가 사용자 신청서 + 어드민 심사 화면. 영역이 명확히 분리됨
- **회귀 위험**: 낮음 — 완전 신규 도메인. 기존 후원·재정·신고 영역과 데이터 분리. 회원·인증 인프라만 재사용

## 후보 3: 후원자·회원 자가 서비스 마이페이지 강화

- **무엇을**: 후원자가 본인의 누적 후원 내역·정기 후원 카드 변경·해지 신청·기부금 영수증 즉시 다운로드를 어드민 도움 없이 마이페이지에서 직접 처리. 회원은 자신의 자격 변경·블랙 사유 확인·로그인 이력·알림 수신 동의 변경까지 셀프 관리.
- **왜**: 현재는 "후원 변경하고 싶다", "영수증 다시 받고 싶다", "어떤 정보 동의했나" 같은 단순 문의가 어드민 수동 처리 대상. 사용자 입장에서는 즉시 안 됨이 답답하고, 어드민 입장에서는 단순 반복 업무가 운영 시간 소진. 자가 서비스 제공으로 양쪽 동시 개선.
- **범위**: 신규 테이블 0~1개(알림 수신 동의 이력) / 기존 후원·결제 API 일부 재사용 + 마이페이지 전용 신규 API 5~7개 / 마이페이지 화면 4~5개 신규 또는 확장(누적 내역·정기 관리·영수증·동의 관리·로그인 이력)
- **예상 기간**: 7~10일
- **B+A 분리 가능 여부**: 가능 — B가 마이페이지 전용 조회·변경 API, A가 마이페이지 5개 화면. 단 결제·후원 영역과 인접해 데이터 일관성 확인 필요
- **회귀 위험**: 중간 — 정기 후원 변경·해지는 토스 빌링·효성 CMS+ 결제 시스템과 직접 맞물림. 카드 변경 시점·해지 시점이 어드민 처리와 충돌하지 않도록 잠금 로직 검토 필요

## 후보 4: 연말정산·기부금 영수증 일괄 발행 + 회계 자동화

- **무엇을**: 어드민이 연말 1회 또는 분기별로 "영수증 일괄 발행" 버튼을 누르면 해당 기간 모든 후원자에게 기부금 영수증 PDF를 자동 생성·이메일 일괄 발송. 동시에 국세청 신고용 기부자별 합계 자료를 엑셀로 자동 생성. 결산 보고서(수입·지출·잔액)도 같은 화면에서 분기·연간 단위로 자동 산출.
- **왜**: 현재 영수증은 후원 건별 발행만 가능하고 연말 일괄 처리 기능이 없음. 협회 어드민이 매년 1월에 1주일 이상 단순 발행·발송에 시간 소진. 국세청 신고 자료도 수동 집계. 연 1회 큰 운영 부담을 1버튼으로 압축하면 운영 효율 큰 폭 개선.
- **범위**: 신규 테이블 1개(일괄 발행 작업 로그) / 일괄 발행 cron 1개 또는 어드민 트리거 API 1개 / 기존 영수증 PDF·이메일 발송 인프라 재사용 / 어드민 화면 1개(일괄 발행 + 결산 보고서 통합) / 국세청 양식 엑셀 변환 1개
- **예상 기간**: 5~8일 (기존 인프라 재사용 비중 큼)
- **B+A 분리 가능 여부**: 가능 — B가 일괄 발행·결산 집계·엑셀 생성, A가 어드민 화면 1개. A 작업량 작아 다른 작업과 병행 가능
- **회귀 위험**: 중간 — 기부금 영수증은 세무 관련이라 1건이라도 누락·오류 시 협회 신뢰도 영향. 발행 멱등성(중복 발행 방지)·금액 정확도 검증 필요. 발송 시 Phase 8 알림 인프라 부하 점검
