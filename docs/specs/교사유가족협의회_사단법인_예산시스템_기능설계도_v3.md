## 1. 법적 근거 — 사단법인 운영에 적용되는 법령 체계

```
민법 제32조 (비영리법인 설립·허가)
  └─ 「교육부 소관 비영리법인의 설립 및 감독에 관한 규칙」
       (교육부령 제329호, 2024.6.18.)
        └─ 시도교육감 위임 규정 (시도별 조례·규칙)
             └─ 시도교육청 비영리법인 결산 보고 양식

「지방보조금 관리에 관한 법률」(보조금 받을 경우만 적용)
  └─ 시·도 지방보조금 관리 조례
       └─ 보조금 정산보고 양식

회계기준 (강제 적용 아님, 권장)
  └─ 비영리조직회계기준 (한국회계기준원, 2017)
```

### 1.1 사단법인이 시도교육청에 반드시 제출해야 하는 것

- **회계연도 시작 전**: 사업계획서 + 예산서
- **회계연도 종료 후 2개월 내**: 사업실적서 + **수입·지출 결산서** + 임원 변동 사항
- **수시**: 정관 변경, 임원 변경, 자산 처분, 차입 등 — **주무관청 사전 허가**

### 1.2 강제되지 **않는** 것 (참고)

- 공익법인회계기준 (지정 시에만)
- 외부 공인회계사 감사 (보조금 일정 규모 이상이면 별도 의무)
- 국세청 공시 (공익법인 지정 시에만)
- 운영성과표·현금흐름표 (시도교육청에 의무 아님, 비영리조직회계기준 권장 사항)
- 부문별(공익/기타) 구분 표시
- 일반관리비 비율 법정 상한

### 1.3 사단법인의 거버넌스 특징 (★ 시스템 설계에 직접 반영)

| 의결 기관 | 권한 |
|---|---|
| **총회 (Members' Assembly)** | 정관 변경, 예산·결산 승인, 임원 선임·해임, 사업계획 의결, 해산 — **최고 의사결정기관** |
| **이사회 (Board of Directors)** | 일상 운영, 총회 안건 사전 심의, 사무국장 임명, 자산 취득·처분 (정관 한도 내), 추경 발의 |
| **감사 (Auditor)** | 회계감사, 업무감사, 부정 시 총회 소집 요구 |

> v2까지는 이사회 의결만 강조했는데, 사단법인은 **총회가 최종 승인 권한**을 갖습니다. 시스템도 두 트랙을 모두 모델링해야 합니다.

## 2. 시스템 목표

1. 시도교육청에 즉시 제출 가능한 **사업계획서·예산서·사업실적서·수입·지출 결산서** 자동 생성.
2. **총회 의결 → 이사회 사전심의** 두 단계 거버넌스를 한 시스템에서 추적.
3. 4축 태깅으로 **사업별·재원별·기능별 집행 현황**을 실시간 노출.
4. 교육청 보조금 수령 시 **별도 회계·증빙·정산보고서** 자동 분리 처리.
5. 비영리조직회계기준에 따라 **운영성과표** 등 확장 보고서를 옵션으로 생성.
6. **AI 어시스턴트**가 4단계 전반에서 실무자 업무를 80% 줄여 줌.

### 2.1 시스템 비목표

- 복식부기 분개 자체 (회계 패키지에 push만)
- 급여·세무 신고
- 회원·후원자 CRM

### 2.2 핵심 가정

| 항목 | 값 |
|---|---|
| 법인 형태 | 비영리 사단법인 (민법 제32조) |
| 주무관청 | 시도교육청 |
| 회계연도 | FY2027부터 시작 예정 (정관 확정 후 lock) |
| 사업 분류 | P01 법률·행정지원 / P02 의료지원 / P03 인과관계조사 / P04 재발방지캠페인 / P05 연계사업 + G00 일반관리 |
| 재원 | 회비, 개인·법인 기부금, 시도교육청 보조금(예정), 사업수익 |
| 회계기준 | 비영리조직회계기준(권장) + 시도교육청 양식(필수) |
| 외부감사 | 의무 아님 (보조금 일정 규모 시만) |
| 공익법인 지정 | 미추진 |

## 4. 4축 태깅 모델 — 사단법인 버전 (★ 시스템의 심장)

모든 예산·집행·결의 레코드는 다음 4축으로 태깅된다.

| 축 | 값 예시 | 의미 / 근거 |
|---|---|---|
| **① 회계구분 (Accounting Segment)** | `GEN` 일반회계 / `GRANT` 보조금회계 | 보조금은 「지방보조금법」상 별도 회계 강제. 일반회계 안에서는 추가 구분 없음 (공익법인이면 PUBLIC/OTHER 추가 가능, 지금은 불필요) |
| **② 사업 (Program)** | `P01~P05`, `G00` | 협의회 5대 사업 + 일반관리 |
| **③ 기능 (Function)** | `EXEC` 사업비 / `ADMIN` 일반관리비 / `FUND` 모금비 | 비영리조직회계기준 권장 분류 |
| **④ 재원 (Funding)** | `FEE` 회비 / `DON-IND` 개인기부 / `DON-CORP` 법인기부 / `GRANT-EDU` 교육청보조금 / `OWN` 사업수익 / `INT` 금융수익 | 재원별 사용제약 추적 |

> **v2.0과 차이**: 1축 의미만 바뀜. 공익/기타 구분 → 일반/보조금 회계구분. 이게 사단법인 + 보조금 받는 단체에 더 정확함.

### 4.1 4축 태깅 실전 예시

```
2027년 본예산 라인 #142
  account     = 5-200-030 법률비용지원
  program     = P01 법률·행정지원사업
  segment     = GEN          ← 일반회계 (자체 기부금 재원)
  function    = EXEC          ← 사업비
  funding     = DON-IND       ← 개인기부금
  amount      = 12,000,000

2027년 본예산 라인 #205 (보조금 사업)
  account     = 5-300-010 교육콘텐츠제작비
  program     = P04 재발방지캠페인사업
  segment     = GRANT         ← 보조금회계
  function    = EXEC
  funding     = GRANT-EDU
  grant_id    = G-2027-SEN-001 (서울시교육청 보조사업)
  amount      = 30,000,000
```

## 6. 라이프사이클 워크플로우 4단계 (사단법인 버전)

각 단계의 AI 통합 지점은 **[AI-n]** 으로 표시.

### 6.1 단계 ① 편성 (Budget Formulation)

```
[사무국] 사업계획 초안
   │ [AI-1] 사업계획서 초안 자동 생성 (전년 실적+키워드)
   ▼
[사업담당(PM)] 사업별 예산 추계
   │ [AI-2] 예산 추계 (작년 패턴+CPI+벤치마크)
   ▼
[회계담당] 계정과목 매핑·4축 검증
   │ [AI-3] 4축 자동 태깅 추론 (신뢰도 < 0.7 시 수동 확인)
   ▼

   │ [AI-4] 비율·이상 highlight, 작년 ±30% 변동분 사유 요청
```

| R-005 | 모금비 ÷ 모금수익 ≤ 30% | WARN |

## 7. ★ 핵심 AI 기능 — 통장거래 자동분류 (AI-12) — 사단법인 버전

v2.0 §5 설계 그대로 유지. 사단법인 특화 변경점만 정리.

### 7.1 변경점

- 4축 추론 출력에서 **segment**는 `PUBLIC/OTHER`가 아니라 **`GEN/GRANT`** 으로 산출.
- 보조금 통장(`funding_source.bank_account`와 매칭) 입출금 거래는 자동으로 `segment=GRANT` + `grant_id` 후보 매칭.
- 자체사업 거래(`segment=GEN`)는 증빙 미첨부도 ER 생성 허용(WARN), 보조금 거래는 BLOCK.
- 세목 자동 추가 규칙은 그대로 (같은 거래처 3회+ 시 신규 세목 제안).
