# 📋 [개발 명세서] 마일스톤·성과급 관리 시스템

## 0. 시작 전 안내

> 권한 체계(RBAC), 데이터 모델/ERD, 화면 와이어프레임, 비기능 요구사항(보안·성능·인증 등)은 **현재 우리가 개발 중인 종합 플랫폼의 기존 구조에 맞춰** 구현해주세요. 본 명세서는 **비즈니스 워크플로우와 계산 로직만** 정의합니다.

> **AI 통합은 기존에 보유한 Google Gemini API 키를 사용합니다.**

---

## 1. 시스템 목적

교사유가족협의회·함께워크 ON·플레오(SI) 3개 사업체의 직원 성과를 **분기 단위**로 추적하고, **매출 연동 + 비매출 마일스톤** 달성도에 따라 **인센티브를 자동 계산·정산**하는 시스템.

3개 사용자 계층:
- **슈퍼어드민** — 마일스톤 정의·임계점·인센티브 공식 설정, 최종 결산 승인
- **어드민(국장)** — 본인 영역 성과 추적, 비매출 성과 2개 우선순위 선택, 분기 결산 제출
- **운영자(계약직·알바)** — 매출·실적 데이터 일상 입력

## 2. 핵심 도메인 개념

### 2.1 마일스톤(Milestone)
성과 측정 단위. 2가지 카테고리.

| 카테고리 | 설명 | 인센티브 정산 방식 |
|---|---|---|
| **매출 연동(REVENUE_LINKED)** | 매출 데이터 누적치로 자동 계산 | 분기 자동 정산 |
| **비매출(NON_REVENUE)** | 이벤트성 성과 (인가·계약·낙찰 등) | 어드민이 분기당 2개 선택 |

### 2.2 임계점(Threshold)
**"본업 기대치까지는 인센티브 없음"**의 핵심 개념.

- 매출 연동 마일스톤에는 대부분 임계점 존재
- 실적 ≤ 임계점 → 인센티브 0원
- 실적 > 임계점 → 초과분에만 공식 적용
- 일부 이벤트성 매출(외주용역·정부과제 등)은 임계점 없음

### 2.3 분기당 비매출 2개 한도
어드민은 분기마다 달성한 비매출 성과 중 **우선순위 2개만 선택**해 인센티브 받음. 나머지는 카운트되지 않음.

### 2.4 인센티브 공식 4가지 타입

| 타입 | 설명 | 예시 |
|---|---|---|
| `FLAT` | 단위당 정액 | 1명당 6만원, 1건당 50만원 |
| `PERCENT` | 초과분 또는 매출의 % | 초과분의 2%, 매출의 5% |
| `BRACKET` | 구간별 정액 | 200만원↑ 20만원 / 500만원↑ 70만원 |
| `EVENT_RANGE` | 사건 규모에 따라 범위 내 수동 결정 | 1건당 20~110만원 |

## 3. 사용자 워크플로우

### 3.1 [운영자] 매출·실적 입력 워크플로우

1. 운영자가 입력 폼 접근
2. 선택: 사업체 (협의회 / 함께워크 ON / 플레오)
3. 선택: 매출 원천 (드롭다운, 17개 매출 원천 중 1개)
4. 입력: 날짜, 금액(또는 수량), 비고, 증빙 파일
5. 저장 → 상태: PENDING
6. 시스템: 해당 사업체 책임자(어드민)에게 검증 요청 알림

### 3.2 [어드민] 매출 검증 워크플로우

1. 어드민이 본인 영역 PENDING 항목 조회
2. 각 항목 검토 (증빙 확인)
3. 두 가지 액션
   - 검증 승인 → 상태: VERIFIED → 인센티브 계산에 반영
   - 반려 → 상태: REJECTED → 운영자에게 사유 통보
4. VERIFIED 데이터만 분기 결산에 반영됨

### 3.3 [어드민] 비매출 성과 제출 워크플로우

1. 어드민이 본인 비매출 마일스톤 목록 조회
2. 달성한 성과에 대해 "성과 제출"
3. 입력: 달성일, 설명, 증빙 (인가증·계약서·보도 캡처 등)
4. 저장 → 상태: PENDING
5. 슈퍼어드민에게 검증 요청
6. 슈퍼어드민 승인 → 상태: VERIFIED
7. 어드민이 분기 종료 전 본인 VERIFIED 성과 중 우선순위 2개 선택 (isSelectedForQuarter = true)

### 3.4 [어드민] 분기 결산 제출 워크플로우

1. 분기 종료 1주 전, 시스템이 어드민에게 결산 작성 알림
2. 어드민이 결산 페이지 진입
3. 시스템이 자동 표시
   - 본인 영역 VERIFIED 매출 데이터 → 매출 연동 인센티브 자동 계산
   - 본인 VERIFIED 비매출 성과 목록 (최대 2개 선택 필요)
4. 어드민이 비매출 성과 2개 우선순위 선택
5. 자가평가 의견 작성
6. "제출" 클릭 → 상태: DRAFT → SUBMITTED
7. 슈퍼어드민에게 검토 요청

### 3.5 [슈퍼어드민] 분기 결산 승인 워크플로우

1. 슈퍼어드민이 SUBMITTED 상태 결산 목록 조회
2. 각 직원별 결산 상세 검토
   - 매출 데이터 검증 상태 확인
   - 비매출 성과 검증·선택 적정성 확인
   - 자동 계산된 인센티브 금액 확인
3. 세 가지 액션
   - 승인 → 상태: SUBMITTED → REVIEWED → APPROVED
   - 반려 → 상태: REJECTED, 사유 기재, 어드민이 재제출
   - 보류 → 추가 자료 요청
4. APPROVED 후 지급 처리 → 상태: PAID
5. 다음 분기 자동 시작

### 3.6 [슈퍼어드민] 마일스톤 관리 워크플로우

1. 슈퍼어드민이 마일스톤 마스터 관리 페이지 진입
2. CRUD 작업
   - 신규 마일스톤 추가 (이름, 카테고리, 담당 직책, 임계점, 공식)
   - 기존 마일스톤 수정 (변경 이력 추적 필요)
   - 비활성화 (삭제 대신 isActive=false)
3. 적용 시점 설정
   - effectiveFrom, effectiveTo로 기간 관리
   - 분기 중간에 마일스톤 변경 시 해당 분기는 기존 공식 유지
4. 변경 시 모든 어드민에게 알림

## 4. 핵심 비즈니스 로직 (워크플로우 설명)

### 4.1 매출 연동 인센티브 계산

- 임계점이 활성화된 경우
  - 실제 누적치 - 임계점 = 초과분
  - 초과분 ≤ 0이면 인센티브 0원
  - 초과분 > 0이면 공식 적용
- 임계점이 없는 경우 (이벤트성 매출)
  - 전체 매출에 공식 적용

### 4.2 공식 타입별 계산 로직

- **FLAT (단위당 정액)**: 단위당 금액 × 수량 (예: 6만원 × 25명 = 150만원)
- **PERCENT (퍼센트)**: 매출(또는 초과분) × 비율 (예: 2,000만원 × 1.5% = 30만원)
- **BRACKET (구간형)**: 가장 높은 적용 가능 구간의 정액 (예: 600만원 → 600↑ 100만원 구간 적용)
- **EVENT_RANGE (이벤트성 범위)**: 사건 규모에 따라 범위 내 수동 결정 (슈퍼어드민이 결정)

### 4.3 비매출 분기 2개 한도 처리

1. 본인의 VERIFIED 비매출 성과 중 isSelectedForQuarter = true인 것만 조회
2. 한도 검증 — 선택 항목이 2개 초과면 에러
3. 선택된 항목들의 보너스 합산

### 4.4 분기 결산 자동 계산

1. 본인 담당 마일스톤 조회
2. 매출 연동 인센티브
   - 각 매출 연동 마일스톤별로 VERIFIED 매출 합산
   - 임계점 적용 + 공식 적용으로 인센티브 계산
   - 모든 매출 연동 인센티브 합계
3. 비매출 보너스
   - 선택된 2개 성과의 보너스 합산
4. 총 변동급 = 매출 연동 합계 + 비매출 합계
5. 결산 DRAFT 생성

### 4.5 임계점 적용 예시 (실제 케이스)

**예시 1: 사무국장 - 신규 정기후원자 유치**
- 임계점: 분기 20명
- 공식: FLAT 1명당 6만원
- 분기 25명 달성 시: (25 - 20) × 60,000 = **30만원**

**예시 2: 정책국장 - 함께워크 ON 매출 1.5%**
- 임계점: 분기 1,500만원
- 공식: PERCENT 초과분의 1.5%
- 분기 매출 3,500만원 달성 시: (35,000,000 - 15,000,000) × 0.015 = **30만원**

**예시 3: 사무국장 - 정기후원 누적액 구간형**
- 임계점: 월 300만원
- 공식: BRACKET (400↑ 50만 / 600↑ 100만 / 800↑ 200만)
- 월 평균 600만원 달성 시: 600만원 구간 → **100만원**

**예시 4: 정책국장 - 유족 순직 컨설팅 (이벤트성 범위)**
- 임계점: 없음
- 공식: EVENT_RANGE 1건당 20~110만원
- 사건 발생 시 슈퍼어드민이 사건 규모에 따라 20~110만원 사이 수동 결정

## 6. 중복 지급 방지 룰

### 6.1 사무국장 - 정기후원자 카테고리 분리

> 한 후원자는 둘 중 한 카테고리에만 카운트됩니다.

- 신규 등록자가 **캠페인·일시후원 참여 이력 없음** → "신규 정기후원자 유치"
- 신규 등록자가 **캠페인·일시후원 참여 이력 있음** → "장기후원 전환"

운영자가 정기후원자 등록 시 **"캠페인 경유 여부"** 체크박스를 통해 분류. 이 분류는 추후 슈퍼어드민이 변경 가능.

### 6.2 SI 수주 채널 임계점 공유

si-001 (중개플랫폼), si-002 (직접 영업), si-003 (NPO 영업)은 **합산 임계점 3,000만원**을 공유합니다.

- 3채널 매출 합산
- 합산이 3,000만원 초과 시 초과분 발생
- 초과분을 채널별 매출 비율로 배분
- 각 채널별 인센티브율(중개 4% / 직접 5% / NPO 5%) 적용하여 합산

## 7. 분기 자동 관리 로직

1. 시스템이 매일 0시에 분기 상태 점검
2. 상태 전환 룰
   - 분기 시작일 도래 → status: UPCOMING → ACTIVE
   - 분기 종료일 도래 → status: ACTIVE → ENDED
   - 결산일 도래 + 모든 결산 PAID → status: ENDED → SETTLED
3. 분기 종료 7일 전
   - 모든 어드민에게 결산 작성 알림
   - PENDING 매출 데이터 검증 알림
4. 결산일 도래 시 미제출 결산은 슈퍼어드민에게 에스컬레이션

## 13. 비기능·운영 가이드

### 13.1 분기 결산 프로세스 (운영)

- 분기 종료 1주 전: 직원이 자체 평가서 제출 (달성 마일스톤, 매출 기록, 우선순위 2개 선정)
- 분기 종료 직후 ~ 2주 내: 대표 검토·승인
- 분기 종료 후 45일 내: 인센티브 지급 (결산일 기준)
- 분기 결산 직후: 다음 분기 목표·우선순위 협의

### 13.2 비매출 보너스 한도 룰

| 룰 | 내용 |
|---|---|
| 한도 카운트 | 분기 내 달성 마일스톤이 2개 초과면 직원이 우선순위 2개 선택 |
| 이월 가능? | 미달성 마일스톤은 다음 분기로 이월 가능 (한도는 별도 카운트) |
| 중복 적용 | 매출 연동 + 비매출은 별도 카테고리 (각각 카운트) |
| 팀 보너스 | 분기 한도와 별개로 조직 전체 목표 달성 시 균등 분배 트랙 가능 |

### 13.3 성과 검증 기준

| 성과 항목 | 검증 자료 |
|---|---|
| 사단법인 인가 | 인가증 사본 |
| 함께워크 ON 입주율 | 분기말 기준 좌석 점유율 표 |
| 회원 등록 수 | 회원관리 시스템 기록 |
| 후원 모금액 | 은행 입금 내역 + 회계 장부 |
| 순직 인정 승소 | 결정문 |
| MOU·법안 발의 | 협약서·발의문 |
| 언론 보도 | 보도 캡처 (1면·메인뉴스 명시) |
| SI 수주·계약 | 계약서 (수주처·금액·일자) |
| 공공입찰 낙찰 | 낙찰 통지서 |
| 인증 취득 | 인증서 |

### 13.4 임계점(Threshold) 운영 원칙

- Base salary는 본업 영역(임계점까지)에 대한 보상입니다.
- 매출 인센티브는 임계점을 초과한 "기대 이상의 성과"에 대해서만 지급됩니다.
- 임계점 미달은 base salary 감액 사유가 아닙니다. 단순히 인센티브 0원입니다.
- 임계점은 매년 회계연도 시작 전 재협의하여 조정합니다.
- 정책 영역 외주용역·정부과제·공공입찰 낙찰 등 이벤트성 매출은 임계점 없이 그대로 적용됩니다.

### 13.5 매출 연동 정산 방식

| 직무 | 정산 주기 | 비고 |
|---|---|---|
| 사무국장 | 분기 결산 | 후원 모금이 정기적이므로 분기 누적 |
| 정책국장 | 분기 결산 | 외주용역 계약 단위로 정산 |
| SI 영업관리자 | 월 정산 (선택 가능) | 영업 직무 특성상 빠른 피드백 권장 |
