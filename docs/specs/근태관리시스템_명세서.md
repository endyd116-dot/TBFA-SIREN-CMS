# 📋 [개발 명세서] 근태관리 시스템

## 0. 시작 전 안내

> 권한 체계(RBAC), 데이터 모델/ERD, 비기능 요구사항(보안·인증·푸시 인프라 등)은 **현재 운영 중인 종합 플랫폼의 기존 구조에 맞춰** 구현해주세요. 본 명세서는 **비즈니스 워크플로우와 로직**, **AI 통합 흐름**, 그리고 **핵심 UI 흐름**을 정의합니다.

> **AI 통합은 기존에 보유한 Google Gemini API 키를 사용합니다.**

---

## 1. 시스템 목적

직원의 출퇴근을 위치 기반으로 기록하고, 슈퍼어드민이 근무 형태(사무실·재택·외근)·휴가 정책·근태 현황을 통합 관리하며, **WBS 시스템과 Gemini AI 기반 보고서 생성기를 연동**해 재택근무·전체 업무 진행 상황을 자동 분석하는 모듈.

## 2. 핵심 도메인 개념

### 2.1 사용자 그룹

| 역할 | 출퇴근 | 정책 설정 | 모니터링 |
|---|---|---|---|
| 슈퍼어드민 | ✅ (선택) | ✅ 전체 | 전체 |
| 어드민 | ✅ | ❌ | 본인 |
| 운영자 | ✅ | ❌ | 본인 |

### 2.2 핵심 개념

| 개념 | 정의 |
|---|---|
| **출퇴근 거점(Workplace)** | 슈퍼어드민이 지정한 위치 (위도·경도) + 허용 반경. 본사·지사·외근지 다중 등록 가능 |
| **근무 형태(WorkMode)** | OFFICE(사무실) / REMOTE(재택) / FIELD(외근) / BUSINESS_TRIP(출장) 4가지 |
| **근무형태 스케줄(WorkSchedule)** | 직원별·날짜별 근무 형태 지정. 단발 또는 규칙 기반 반복 가능 |
| **반복 규칙(RecurringRule)** | 매주 특정 요일에 재택·외근 자동 설정 (예: 매주 월·수·금 재택) |
| **근무 정책(WorkPolicy)** | 기준 시간, 유연근무, 코어타임, 휴게시간 등 |
| **휴가 종류(LeaveType)** | 유급/무급. 슈퍼어드민 CRUD |
| **유급휴가 부여 규칙(LeavePolicy)** | 만근 시 +1, 입사 1년 후 15일 등 자동 부여 조건 |
| **출퇴근 기록(AttendanceRecord)** | 일별 1건. 출근·퇴근 시각·위치·근무시간·상태·근무형태 |
| **재택근무 보고서(RemoteWorkReport)** | 재택근무일 직원이 제출하는 일일 업무 보고. Gemini AI 초안 생성 가능 |
| **WBS 카드(WbsCard)** | 외부 WBS 시스템과 연동되는 업무 단위. 작업 로그 포함 |
| **AI 흐름 파악(AI Insight)** | Gemini AI가 카드·로그·보고서를 종합 분석해 업무 진행도·보고서를 생성 |

### 2.3 근무 형태별 검증 방식

| 근무 형태 | 위치 검증 | 비고 |
|---|---|---|
| OFFICE | 등록된 사무실 거점 50m 이내 | 기본 모드 |
| REMOTE | **위치 검증 면제** | 재택근무 보고서 의무 제출 |
| FIELD | 슈퍼어드민이 사전 등록한 **외근지 주소 50m 이내** | 외근지별 별도 거점 등록 |
| BUSINESS_TRIP | 위치 검증 면제 (출장 신청 승인 시) | 일별 출장 보고서 권장 |

### 2.4 출퇴근 상태(Status)

| 상태 | 정의 |
|---|---|
| NORMAL | 정상 출근·퇴근 |
| LATE | 지각 |
| EARLY_LEAVE | 조퇴 |
| ABSENT | 결근 |
| LEAVE | 휴가 (유급/무급) |
| HOLIDAY | 공휴일·회사 휴무일 |
| PARTIAL_LEAVE | 반차 |

## 3. 사용자 워크플로우

### 3.1 [슈퍼어드민] 근무 형태·스케줄 지정 워크플로우

**(1) 직원별 정규 근무 형태 설정**

1. 슈퍼어드민이 직원 목록에서 특정 직원 선택
2. "근무 형태 관리" 진입
3. 기본 근무 형태 선택: OFFICE / REMOTE / FIELD / HYBRID
4. HYBRID 선택 시 반복 규칙 추가
   - 요일 선택 (월·화·수·목·금·토·일)
   - 각 요일별 근무 형태 지정 (예: 월=재택, 화=사무실, 수=재택, 목=사무실, 금=재택)
   - 적용 기간 (시작일~종료일, 무기한 가능)
5. 외근 모드 포함 시 외근지 사전 등록 (주소 입력 → 자동 좌표 변환)
6. 저장 시 미래 날짜에 대해 자동으로 AttendanceRecord 예약 생성 (실제 출퇴근 시 해당 형태 검증 적용)

**(2) 단발성 재택·외근 지정**

1. 슈퍼어드민이 특정 직원·특정 날짜 클릭
2. 해당 날짜만 근무 형태 변경 (반복 규칙보다 우선)
3. 외근 시 외근지 주소 입력 (저장된 외근지 목록에서 선택 또는 신규 입력)
4. 사유 기재 (선택)
5. 저장 → 직원에게 알림

**(3) 직원의 재택·외근 신청 (역방향 워크플로우)**

1. 직원이 본인 캘린더에서 특정 날짜 선택
2. "근무 형태 변경 요청" 클릭
3. 신청 양식 작성
   - 변경할 근무 형태
   - 외근 시 외근지 주소
   - 사유
4. 제출 → 상태: PENDING
5. 슈퍼어드민 알림 → 승인/반려
6. 승인 시 해당 날짜 근무 형태 적용

### 3.2 [직원] 출근 워크플로우 (근무 형태별 분기)

**공통 시작**:
1. 직원이 "출근하기" 버튼 클릭
2. 시스템이 오늘 날짜의 예정 근무 형태 조회
3. 근무 형태에 따라 분기:

**OFFICE 모드**:
- GPS 위치 획득
- 등록된 사무실 거점들과 거리 계산
- 가장 가까운 거점이 허용 반경 내인지 검증
- 반경 외 → 출근 불가 안내
- 반경 내 → 출근 기록 저장

**REMOTE 모드**:
- 위치 검증 면제
- "재택근무 시작 확인" 메시지
- 출근 기록 저장 (location = "REMOTE")
- WBS에 자동으로 "재택근무 일일 보고서" 카드 생성
- 보고서 카드는 사용자의 오늘자 To-Do 리스트 최상단에 표시

**FIELD 모드**:
- GPS 위치 획득
- 오늘 예정된 외근지 좌표와 거리 계산
- 외근지 허용 반경 내인지 검증
- 반경 외 → "외근지 50m 내에서 출근 가능합니다" 안내
- 반경 내 → 출근 기록 저장

**BUSINESS_TRIP 모드**:
- 위치 검증 면제 (사전 승인 출장)
- 출장지·일정 자동 매핑
- 출근 기록 저장

**공통 마무리**:
4. 시각 검증 (지각·정상)
5. 응답: "출근 완료. 09:23. 재택근무 모드. 오늘 업무를 시작하세요"

### 3.3 [직원] 퇴근 워크플로우

1. "퇴근하기" 버튼 클릭
2. 근무 형태별 위치 검증 (출근과 동일 로직)
3. 출근 기록 존재 검증
4. 자동 근무시간 계산 (휴게시간 차감)
5. 재택근무 모드인 경우:
   - "오늘 재택근무 보고서를 작성하셨나요?" 알림
   - 보고서 미작성 시 → "지금 작성" / "내일 보강" 옵션
6. 퇴근 기록 저장
7. 응답: "퇴근 완료. 오늘 근무시간 7시간 45분"

### 3.4 [직원] 재택근무 보고서 작성 워크플로우 (Gemini AI 초안 생성)

1. 재택근무일 출근 시 WBS에 자동 생성된 "재택근무 일일 보고서" 카드 진입
2. 보고서 화면 구성
   - 오늘 작업한 WBS 카드 목록 (자동 수집)
   - 카드별 진행 로그 (자동 수집)
   - 보고서 본문 (작성 영역, 500자 미만)
3. 직원이 "AI 초안 생성" 버튼 클릭
4. 시스템이 다음 데이터를 **Gemini API**에 전달
   - 오늘 작업한 카드 제목·설명·상태 변경 이력
   - 작성한 코멘트·로그
   - 완료된 체크리스트 항목
   - 이전 며칠 보고서 (맥락 파악용)
5. Gemini가 500자 미만 보고서 초안 생성
   - 오늘 한 일 (3~5개 항목)
   - 진행률·완성도
   - 내일 계획
   - 이슈/막힌 부분 (있다면)
6. 초안이 보고서 영역에 자동 채워짐
7. 직원이 초안 검토 → 수정·추가 → "제출"
8. 제출 시간 기록, 상태: SUBMITTED
9. 슈퍼어드민·해당 부서장에게 알림

### 3.5 [슈퍼어드민] 재택근무 모니터링 워크플로우

1. "근태 관리" → "재택근무 보고서" 메뉴 진입
2. 화면 구성
   - 좌측: 직원 목록 (재택근무자만 또는 전체)
   - 우측: 선택된 직원의 보고서 목록 (날짜별)
3. 필터: 부서·기간·근무형태·제출 상태
4. 보고서 클릭 → 상세
   - 보고서 본문
   - 첨부된 WBS 카드 링크
   - 작업 로그 요약
5. 슈퍼어드민 액션
   - 댓글 작성 (피드백)
   - 카드 직접 이동
   - 별표 표시 (우수 보고서)

### 3.6 [슈퍼어드민] AI 흐름 파악 워크플로우

**(1) 재택근무 흐름 파악 (특정 직원)**

1. 직원 보고서 목록 상단에 "AI 흐름 파악 (재택)" 버튼
2. 슈퍼어드민이 클릭 → 기간 선택 (최근 1주/2주/한달/사용자 지정)
3. 시스템이 **Gemini API**에 전달
   - 해당 기간 재택근무 보고서 전체
   - 관련 WBS 카드 진행 이력
   - 보고서 간 일관성 데이터
4. Gemini가 다음을 종합 분석
   - 진행된 주요 작업 카테고리
   - 작업별 완성도 추정 (%)
   - 업무 진행 패턴 (집중 시기, 정체 시기)
   - 이상 신호 (보고서 누락, 진행 정체 등)
   - 다음 단계 제안
5. 결과를 시각화
   - 작업 분류 도넛 차트
   - 일자별 작업량 막대 그래프
   - 완성도 진행 라인 그래프
   - 텍스트 요약 (500자 내)
6. 결과를 보고서·PDF로 export 가능

**(2) 전체 업무 흐름 파악**

1. 직원 페이지 또는 대시보드에서 "AI 흐름 파악 (전체 업무)" 버튼
2. 시스템이 다음 데이터를 종합
   - WBS의 모든 할당 카드 (재택·사무실 무관)
   - 카드 상태 변경 이력
   - 코멘트·로그
   - 분기 마일스톤 진행도
3. **Gemini API**가 분석하여 500자 미만 업무 보고서 생성
   - 현재 진행 중인 업무 3~5개
   - 각 업무 진행률
   - 이번 주·이번 분기 핵심 성과
   - 막힌 부분·지원 필요 사항
   - 종합 평가 (한 문장)
4. 슈퍼어드민이 직원별로 한눈에 파악 가능
5. 전체 직원 일괄 생성: "이번 주 전직원 업무 흐름" 페이지

### 3.7 [직원] 출퇴근 누락 수정 요청 워크플로우

1. 직원이 본인 캘린더에서 누락 일자 확인
2. "수정 요청" 클릭
3. 양식 작성
   - 대상 일자
   - 누락 항목 (출근/퇴근/둘다)
   - 요청 시각
   - 사유
   - 증빙 (선택)
4. 제출 → 상태: PENDING
5. 슈퍼어드민에게 알림
6. 승인 → 기록 갱신 / 반려 → 사유 통보

### 3.8 [직원] 휴가 신청 워크플로우

1. 직원이 본인 휴가 잔여 확인
2. "휴가 신청" 클릭
3. 양식 작성
   - 휴가 종류 (드롭다운, 슈퍼어드민이 정의한 목록)
   - 시작일·종료일 또는 반차 선택
   - 사유
   - 증빙 (선택)
4. 시스템 자동 검증
   - 잔여 일수 확인
   - 일정 충돌 검사
   - 회사 휴무일·공휴일 제외 계산
5. 제출 → 상태: PENDING → 슈퍼어드민 승인/반려
6. 승인 시 해당 일자 LEAVE 상태로 변경 + 잔여 차감

### 3.9 [슈퍼어드민] 거점·정책 설정 워크플로우

1. "근태 설정" 진입
2. 거점 관리
   - 사무실 거점 등록 (지도 클릭 또는 주소 검색)
   - 외근지 거점 등록 (주소 검색 → 좌표 자동 변환)
   - 거점별 허용 반경 설정 (기본 50m)
   - 거점별 적용 직원 지정 가능
3. 근무 정책 관리
   - 기준 출근·퇴근 시각
   - 유연근무·코어타임 설정
   - 허용 지각·조퇴 시간
   - 휴게시간
   - 주당 근로시간 한도
4. 휴가 종류 CRUD
   - 휴가명, 유급/무급, 단위(일/시간), 결재 필요 여부
5. 유급휴가 부여 규칙
   - 입사일 기반 자동 부여
   - 만근 인센티브
   - 연차 자동 발생 주기
6. 공휴일·회사 휴무일 등록

## 4. 비즈니스 로직 (워크플로우 설명)

### 4.1 위치 검증 로직

- 직원의 GPS 좌표와 등록 거점의 좌표 사이 거리를 Haversine 공식으로 계산
- 거리가 거점의 허용 반경 이하인 경우에만 출퇴근 허용
- 거점이 여러 개인 경우 가장 가까운 거점 자동 선택
- 재택·출장 모드는 위치 검증 면제 (단, 디바이스·IP 로그 기록은 유지)
- 외근 모드는 사전 등록된 외근지 좌표 기준 동일하게 검증
- **퇴근 위치 검증 (2026-05 추가)**: 퇴근도 OFFICE(일반근무)는 거점 반경 강제(출근 기록 거점 우선, 없으면 가장 가까운 OFFICE 거점). REMOTE·FIELD·BUSINESS_TRIP 퇴근은 위치 미검증.

### 4.2 근무시간 계산 로직

- 총 시간 = 퇴근 시각 - 출근 시각
- 휴게시간 차감 조건: 총 시간이 일정 기준(예: 4시간) 이상일 때만 차감
- 최종 근무시간 = 총 시간 - 휴게시간
- 재택근무 시에도 동일 적용 (단, 코어타임 정책 별도 가능)
- **다중 세션 합산 (2026-05 재출근)**: 같은 날 퇴근 후 재출근하면 세션을 `att_records.sessions`(JSONB)에 누적하고 모든 세션의 근무시간을 합산. 업무시간 내에서는 퇴근취소·시각 셀프수정 가능, 업무시간 외에는 재출근만(기존 퇴근 기록 보존).

### 4.3 지각·조퇴·결근 판정 로직

- 휴일·휴가 우선 판정 → 해당 시 상태 확정
- 출근 기록 없음 + 휴가 없음 → 결근
- 기준 출근 시각 + 허용 지각 시간 초과 출근 → 지각
- 기준 퇴근 시각 - 허용 조퇴 시간 미달 퇴근 → 조퇴
- 지각 + 조퇴 동시 발생 시 별도 상태 표기

### 4.4 유연근무 검증 로직

- 유연근무 모드 직원은 코어타임 내 근무 필수
- 일일 최소 근무시간(예: 6시간) 충족 검증
- 코어타임 도달 못 한 경우 지각·조퇴 처리

### 4.5 유급휴가 자동 부여 로직 (연차 산정 정책 — 2026-05 모드 A/B)

슈퍼어드민이 근태 설정 '근무 정책' 탭에서 **연차 산정 방식을 택일·편집**(`att_policies` 연차 6컬럼). 두 모드는 동시 운영하지 않음.

- **모드 A (5인 이하·현재 협회 기본)**: 매월 만근 시 +`perfect_bonus_per_month`일(기본 1일) 부여. 근속 기반 일괄부여 미실행.
- **모드 B (5인 이상)**: 입사 N주년 도래 시 `annual_base_days + floor((근속년수-1)/annual_increment_years) × annual_increment_days`일 부여(상한 `annual_cap_days`). 기본값 기준 1주년 12일·3주년 13일·5주년 14일. 월 만근 보너스 미실행.
- **근속 기준일**: `members.hire_date`(입사일). NULL이면 가입일(`created_at`) 폴백.
- **기존 잔여 보존**: 부여 시 ON CONFLICT GREATEST로 기존 잔여를 덮어쓰지 않음.
- **자동 실행**: `cron-att-leave-auto`가 매월 정책을 읽어 모드 분기 실행. (옛 "입사 1년 도래 15일 일괄" 로직은 모드 B 파라미터로 대체됨)
- **운영자 수동 조정**: 슈퍼어드민이 잔여 휴가 화면에서 직원별 부여(잔여 기록 없는 직원 첫 부여 포함)·회수(음수)·조정 가능. 잔여 0·미부여 직원도 전원 표시.

### 4.6 만근 판정 로직

- 한 달 동안 결근·무단 지각·무단 조퇴 0건이면 만근
- 휴가는 만근에 영향 없음
- 만근 시 자동으로 유급휴가 +1 부여

### 4.7 야근(연장근로) 자동 계산 로직

- 일일 표준 근무시간(8시간) 초과 시 야근으로 자동 집계
- 주당 누적 근무시간이 48시간 도달 시 임계 경고
- 주당 52시간 초과 시 슈퍼어드민에게 즉시 알림

### 4.8 재택근무 보고서 AI 초안 생성 로직 (Gemini API)

- 입력 데이터: 오늘 작업한 WBS 카드 + 코멘트 + 상태 변경 로그 + 이전 며칠 보고서 맥락
- Gemini API 호출 시 시스템 프롬프트로 다음 지시
  - 500자 미만 한국어
  - 4가지 섹션: 오늘 한 일 / 진행률 / 내일 계획 / 이슈
  - 객관적·간결한 문체
  - 카드 ID·제목 인용
- 생성된 초안은 편집 가능한 상태로 직원에게 제공
- 직원이 수정·추가 후 최종 제출

### 4.9 AI 흐름 파악 로직 - 재택 전용 (Gemini API)

- 기간 내 재택근무 보고서 전부 수집
- 관련 WBS 카드 진행 이력 결합
- Gemini API에 분석 요청
  - 주요 작업 카테고리 분류
  - 카드별 완성도 추정
  - 진행 패턴 분석 (집중도·정체)
  - 이상 신호 탐지
  - 500자 미만 종합 평가
- 결과를 차트·텍스트로 시각화

### 4.10 AI 흐름 파악 로직 - 전체 업무 (Gemini API)

- 직원의 모든 할당 WBS 카드 + 마일스톤 데이터 수집
- 카드 상태·코멘트·완료율 종합
- Gemini API에 분석 요청
  - 현재 진행 중인 업무 추출
  - 각 업무 진행률
  - 핵심 성과
  - 막힌 부분
  - 종합 평가 500자 미만
- 슈퍼어드민 대시보드에 직원별 카드 형태로 노출

## 6. AI가 자동 추가해야 할 워크플로우 (모순·누락 분석)

### 6.1 모순점 자동 보완

| 모순 | 해결 |
|---|---|
| 재택 모드인데 위치 검증 없음 → 부정 출근 가능 | **디바이스·IP 로그 추가 기록**, 평소 패턴 이탈 시 슈퍼어드민 알림 |
| 외근지 등록은 했는데 직원이 다른 외근지로 갔다면? | **출근 시 외근지 선택 옵션** 추가 (사전 등록된 목록에서) |
| 휴가 신청과 근무형태 변경이 같은 날 충돌 | 휴가가 우선 적용. 근무형태 자동 무효화 |
| AI 보고서 생성 시 WBS 데이터 없으면? | "오늘 작업 카드가 없습니다. 직접 작성해주세요" 안내 + 빈 템플릿 제공 |
| 재택근무 보고서 미제출 채로 다음 출근 | 출근 시 "어제 재택근무 보고서가 미제출입니다. 지금 작성하시겠습니까?" 알림 |
| 출장 중 재택 + 외근 혼재 | 출장 모드 우선 적용, 일별 보고서는 자유 형식 |

### 6.2 필수 추가 기능 (AI 판단)

| 기능 | 설명 |
|---|---|
| **재택근무 빈도 한도 관리** | 월 X일 재택 한도 설정 (회사 정책상). 초과 시 슈퍼어드민 승인 필요 |
| **외근지 자동 등록 & 즐겨찾기** | 자주 가는 외근지 즐겨찾기. 첫 방문 시 자동 등록 제안 |
| **재택근무 보고서 품질 점수** | Gemini AI가 제출된 보고서 완성도 평가 (0~100점). 우수 보고서 별표 |
| **연속 재택 일수 알림** | 같은 직원이 연속 N일 재택 시 슈퍼어드민에게 알림 (편향 방지) |
| **재택 일별 작업 로그 최소 기준** | 재택일에 WBS 카드 활동 0개면 "활동 없음" 경고 |
| **외근지 출근 사진 옵션** | GPS 검증 신뢰도 낮을 때 셀카·현장 사진 첨부 |
| **출장 신청 워크플로우** | 출장 사전 신청 → 슈퍼어드민 승인 → 자동 일정 등록 |
| **재택근무 신청 워크플로우** | 정규 외 재택 요청 시 사유·승인 |
| **AI 보고서 다국어** | 영어·일본어 등 (확장 시) |
| **재택 근무자용 단순 출퇴근 위젯** | 매번 앱 진입 불필요. 푸시 알림 또는 위젯에서 1탭 출퇴근 |
| **재택 시 의무 사항 체크리스트** | "오늘 슬랙 응답률 X%", "정기 미팅 참여" 등 (선택 정책) |
| **출장비·교통비 자동 청구** | 외근·출장 시 영수증 첨부 → 자동 청구서 생성 |
| **재택근무 보고서 비교 뷰** | 같은 직원의 이전 보고서들과 비교 (성장 추적) |
| **팀 캘린더 통합 뷰** | 누가 오늘 사무실/재택/외근/휴가인지 한눈에 |
| **AI 일일 요약 (슈퍼어드민용)** | 매일 저녁 자동 발송. "오늘 전직원 근태 요약 + 이상 신호" |
| **재택근무 IP 로깅 (선택)** | 재택 시 접속 IP 기록. 평소와 다른 IP면 확인 알림 |

### 6.3 있으면 좋은 기능

- 재택근무 만족도 조사 (분기별)

## 9. 급여·성과급 연동 워크플로우

- 매월 말 또는 슈퍼어드민이 지정한 정산일에 자동 집계
- 집계 항목: 총 근무시간, 휴가 사용(유급/무급), 지각·결근, 야근, 만근 여부
- 무급 휴가·결근 → 급여 차감 계산
- 야근 → 1.5배 가산 수당 (법정)
- 만근 → 유급휴가 자동 +1 부여
- 성과급 시스템에 데이터 전달: 지각·결근 누적이 일정 수준 초과 시 비매출 마일스톤 자격 박탈 검토 알림
- 슈퍼어드민 검토·승인 후 외부 회계·급여 시스템으로 export

## 10. AI 통합 (Gemini API) 사용 포인트 요약

본 시스템에서 Gemini API를 활용하는 모든 지점입니다.

| 기능 | 입력 데이터 | 출력 |
|---|---|---|
| 재택근무 보고서 초안 생성 | 오늘 WBS 카드 + 로그 + 코멘트 + 이전 보고서 | 500자 미만 한국어 보고서 초안 |
| AI 흐름 파악 (재택 전용) | 기간 내 재택 보고서 + WBS 카드 이력 | 작업 분류 + 진행 패턴 + 500자 종합 평가 |
| AI 흐름 파악 (전체 업무) | 전체 WBS 카드 + 마일스톤 데이터 | 500자 미만 업무 보고서 |
| 재택근무 보고서 품질 점수 | 제출된 보고서 본문 | 0~100점 평가 |
| AI 일일 요약 (슈퍼어드민용) | 오늘 전직원 근태·재택 데이터 | 일일 요약 + 이상 신호 |
| 출퇴근 패턴 이상 감지 | 직원별 누적 출퇴근 데이터 | 패턴 분석 + 알림 사유 |

**Gemini API 모델 권장**:
- 보고서·요약 생성: `gemini-2.5-flash` (속도·비용 효율)
- 분석·인사이트: `gemini-2.5-pro` (품질 우선)
