# 교사유가족협의회 온라인 추모관 — 기능 명세 및 워크플로우

> 작성 목적: 협의회 플랫폼 내 신규 온라인 추모관의 기능 정의와 사용자 동선을 설계하기 위한 기획 문서. UI 디자인이 아닌 **기능 + 워크플로우** 중심으로 작성됨.

---

## 0. 한눈에 보는 컨셉

> "잔잔하게 흐르는, 그러나 결코 잊히지 않는 공간"

- **톤앤매너**: 차분한 자연광 톤, 손글씨/세리프 타이포, 절제된 모션 (페이드·슬로우 패럴럭스).
- **사운드**: 진입 시 BGM 자동 재생 + 우상단에 명확한 음소거 토글 (접근성 고려, 사용자 설정 기억).
- **시각**: 메인 키비주얼은 유가족의 마음을 담은 영상(루프 또는 1회 재생). 영상 위에 짧은 헌사 카피.
- **흐름**: 입장 모달이나 인트로 영상 없이, **자연스러운 스크롤**로 추모 공간으로 이어진다.

---

## 1. 정보 구조 (Information Architecture)

메인은 **통합 추모 공간**, 그 아래로 **개별 선생님 페이지**가 연결되는 하이브리드 구조.

```
온라인 추모관 (메인)
├── 히어로 (유족 영상 + 키 메시지)
├── 실시간 참여 카운터
├── 통합 헌화/촛불 공간
├── 개별 선생님 추모 페이지 그리드  ─┐
├── 통합 추모 메시지 / 방명록         │
├── 자료실 (보고서 · 뉴스 · 법안)      │
├── 함께하는 행동 (공유 카드)          │
└── 협의회 소개 · 연락                │
                                    │
개별 선생님 페이지 (n개) ◀───────────┘
├── 프로필 (사진 · 짧은 약력)
├── 타임라인 (사망일 · 순직인정일 등)
├── 촛불 · 헌화 (개별 카운트)
├── 추모 메시지 (방명록형)
├── 기억의 편지 (선생님께 직접 쓰는 글)
└── 공유 (개인화 추모 카드)
```

## 2. 기능 명세

### 2.1 메인 페이지 — 자연스러운 스크롤 흐름

진입한 사용자가 어떤 클릭도 없이 스크롤만으로 추모를 경험하도록 구성한다.

**섹션 ①. 히어로 (Hero)**

- 유가족의 마음을 표현하는 **유족 영상** 자동 재생(무음 시작 가능, 자막 포함).
- 영상 위 짧은 헌사 카피 (예: "우리는 당신들을 기억합니다").
- 우하단 또는 우상단에 **BGM 음소거/재생 토글**. 기본은 재생(잔잔한 추모곡), 사용자 선택은 `localStorage`에 저장.
- 스크롤 힌트(아래 화살표) — 다음 섹션으로 자연 유도.

**섹션 ②. 실시간 참여 카운터**

| 지표 | 표기 예 |
|---|---|
| 함께 기억한 사람 | "지금까지 **OO,OOO명**이 함께 기억했습니다" |
| 켜진 촛불 | "**OO,OOO개**의 촛불이 켜졌습니다" |
| 남겨진 메시지 | "**OO,OOO개**의 마음이 모였습니다" |

- 숫자 카운트업 애니메이션(스크롤 시 1회).
- 클릭 시 각 섹션으로 부드러운 스크롤 이동.

**섹션 ③. 통합 헌화 / 촛불**

- 모든 순직 선생님을 **한 공간에서 함께 추모**하는 인터랙티브 영역.
- 클릭 한 번으로 가상의 촛불이 켜지고, 화면에 천천히 추가됨(파티클/플로팅 효과).
- 헌화 시 **닉네임(선택)**만 입력하여 비회원도 가능.
- 24시간 / 7일 / 누적 토글로 시점 전환 가능.

**섹션 ④. 개별 선생님 추모 페이지 그리드**

- 카드형 그리드. 카드 1장당 영정 사진(또는 실루엣), 성함, 짧은 한 줄 헌사, 헌화/촛불/메시지 수.
- 카드 클릭 → 개별 선생님 페이지로 이동.
- **검색 / 필터 UI는 기존 개발된 컴포넌트 재사용** (이름, 지역, 연도 등).

**섹션 ⑤. 통합 추모 메시지 / 방명록**

- 선생님 한 분이 아니라 **모든 선생님께 드리는 글**을 남기는 공간.
- 최근 글 무한 스크롤. 좋아요/공감 카운트.
- 글쓰기는 **회원 로그인 필수** (열람은 누구나).

**섹션 ⑥. 자료실 진입 카드**

- 카테고리 미리보기: ① 진상규명/조사보고서 ② 교권보호 법안 진행상황 ③ 언론 보도 아카이브 ④ 협의회 성명·기자회견.
- 각각 자료실 상세 페이지로 연결.
- **순직 인정 타임라인(연표)** 위젯을 자료실 입구에 노출 — 사건/입법/판결의 흐름을 한눈에.

**섹션 ⑦. 함께하는 행동 (공유 카드)**

- "내가 켠 촛불을 카드로 받아보기" CTA.
- 카드 생성 → 닉네임/메시지/촛불 이미지가 합성된 PNG → 카카오톡·인스타그램 스토리·링크 복사·X 공유.
- 카드 하단에 짧은 협의회 URL/QR 자동 삽입 → **자연스러운 외부 확산**.

**섹션 ⑧. 풋터 / 협의회 소개**

- 협의회 소개, 연락처, 후원/연대 안내(필요 시).
- 뉴스레터 구독으로 마무리.

### 2.2 개별 선생님 추모 페이지

각 선생님마다 동일한 템플릿을 따르되, 콘텐츠는 유가족 협조하에 운영진이 채운다.

**① 프로필 영역**
- 영정 사진(또는 실루엣 + 이름).
- 성함, 학교/지역(선택), 출생–사망일.
- 한 줄 헌사(유가족 또는 동료 작성).
- 페이지 BGM은 메인과 동일 정책(전체 사이트 1개 BGM, 음소거 상태 공유).

**② 짧은 약력 / 기억의 사진**
- 2~5문단의 짧은 약력 (강요하지 않으며, 유가족 의사에 따름).
- 사진 슬라이드 (선택, 운영진이 큐레이션).

**③ 타임라인 (사망일 · 순직 인정 관련 주요 일자)**
- 세로형 또는 가로형 간단한 타임라인.
- 예: 사망일 → 진상조사 청원 → 순직 신청 → 순직 인정일 → 추모식 등.
- 각 노드 클릭 시 자료실의 관련 문서로 점프.

**④ 헌화 / 촛불 (개별)**
- 이 선생님께 직접 촛불을 켜는 버튼. 메인의 통합 카운트와는 별도로 개별 누적.
- 헌화 종류 선택 가능(국화 / 촛불 / 종이학 등 1~3종 정도 권장).

**⑤ 추모 메시지 (방명록)**
- 짧은 메시지(50자 / 1000자 이원화) — 4·16 기억관 패턴 차용.
- 닉네임 + 한 줄 메시지 카드 형태로 누적 표시.
- 작성은 **회원 로그인 필수**, 열람은 전면 공개.

**⑥ 기억의 편지**
- "선생님께"로 시작하는 편지 형식 입력 폼.
- 메시지보다 호흡이 긴 글, 별도 탭으로 분리해 정독 환경 제공.
- 발행 시 작성자 명의/익명 선택, 발행 후 수정·삭제 가능.
- **편지함**처럼 펼쳤다 접는 인터랙션.

**⑦ 공유**
- 이 선생님 페이지 전용 추모 카드 생성.
- "OOO 선생님을 기억합니다" 카피 + 영정/실루엣 + 내 메시지가 들어간 1080×1080 카드.

---

### 2.3 자료실

- 카테고리: 조사보고서 / 법안·정책 / 언론보도 / 협의회 활동.
- 각 자료에 발행일, 출처, 다운로드 또는 외부 링크.
- 첨부 PDF는 **인앱 뷰어**로 새 창 없이 열리도록.
- 진상규명·입법 흐름을 시계열로 보여주는 **공통 타임라인 위젯**을 자료실 상단에 고정.

---

### 2.4 개인화 추모 카드 (Shareable Card)

- 진입점: 메시지 작성 직후 / 촛불 켠 직후 / 개별 선생님 페이지의 "공유" 버튼.
- 카드 구성: 닉네임 + 메시지(또는 촛불 아이콘) + 추모관 URL + 협의회 마크.
- 출력: PNG 다운로드, 카카오톡·인스타·X 공유 버튼, 링크 복사.
- 카드 자동 생성에 동의했을 때만 발행되며, 본인 메시지만 카드화 가능.

---

### 2.5 사운드 / 음악 정책

- 사이트 진입 시 **잔잔한 BGM 자동 재생(브라우저 정책상 첫 클릭 시 발동될 수 있음)**.
- 우상단 고정 음소거 버튼 — 모든 페이지 공통.
- 사용자 선택은 세션을 넘어 기억(localStorage).
- 영상 재생 중에는 BGM 자동 페이드아웃 → 영상 종료 시 페이드인.
- 모바일 데이터 절약 모드 감지 시 자동 음소거.

## 3. 사용자 워크플로우

### 3.1 일반 방문자 (비로그인)

```
입장 (URL 클릭)
   ↓
메인 진입 — BGM 자동 시작, 유족 영상 노출
   ↓
스크롤 — 카운터 → 통합 헌화 (촛불 켜기 1회 가능)
   ↓
선생님 카드 그리드 → 특정 카드 선택
   ↓
개별 선생님 페이지에서 헌화 가능 (메시지/편지는 로그인 안내)
   ↓
[옵션] 회원가입 안내 모달 → 가입 후 글 작성
   ↓
[옵션] 공유 카드 생성 → SNS 공유
```

**핵심 원칙**: 헌화·열람은 진입 장벽 0. 글쓰기만 회원 인증.

### 3.2 가입 회원

```
로그인
   ↓
메시지/편지 작성 가능
   ↓
내 활동 보기 — 내가 켠 촛불 / 남긴 메시지 / 받은 공감 모음
   ↓
내 메시지를 공유 카드로 발행
   ↓
뉴스레터/추모일 알림 수신 동의(선택)
```

### 3.3 유가족 (식별된 회원)

```
운영진이 사전 식별 → 유가족 권한 부여
   ↓
해당 선생님 페이지의 헌사 / 사진 / 약력 수정 요청 폼 사용
   ↓
운영진 검토 후 반영 (실제 편집은 운영진이 수행)
```

> 유가족이 직접 편집하는 권한을 부여할지는 후속 결정. 현재 안은 **요청 → 운영진 반영**.

### 3.4 운영진 / 관리자 (기존 어드민 활용)

```
신규 선생님 추모 페이지 등록
   ├─ 유가족 사전 동의 / 자료 제공 확인
   ├─ 영정·약력·타임라인 입력
   └─ 공개 전 검수 → 게시
   ↓
일상 운영
   ├─ 메시지/편지 신고 처리 (기존 신고 기능 연동)
   ├─ 자료실 신규 자료 등록
   ├─ 추모일/입법 진척 타임라인 업데이트
   └─ 카운터·통계 대시보드 확인
```

## 5. 비기능 요구사항

- **잔잔한 분위기**: 채도 낮은 컬러 팔레트(웜그레이·아이보리·딥블루), 손글씨 또는 명조체 헤더, 모션은 모두 250~400ms 이내, ease-out.
- **접근성(A11y)**: BGM 음소거 명확, 영상 자막, 이미지 alt, 키보드 탐색, 색상 대비 AA 이상.
- **모바일 우선**: 추모 트래픽 다수가 모바일. 카드 그리드는 1열 → 2열, 카운터/촛불은 터치 최적화.
- **성능**: 영상/이미지 lazy load, BGM은 짧은 길이로 loop, 카운터는 폴링이 아니라 캐시 + 증분 갱신.
- **개인정보**: 비회원 닉네임은 평문 저장하지 않거나 임시 토큰 처리. 영정·약력은 유가족 사전 동의 기록 보관.

## 6. 외부 사례 참고 (Benchmark 요약)

설계에 차용한 주요 레퍼런스. 자세한 분석은 별첨.

| 사례 | 차용 포인트 |
|---|---|
| **4·16 세월호 기억관** | 누적 참여 카운터, 단문/장문 이원화 메시지 |
| **9/11 Memorial** | 이름 기반 진입(개별 페이지 우선), 구술/육성 영상 아카이브, 교육 자료 통합 |
| **국립현충원 사이버 참배** | 개인 묘역 페이지 모델, 클릭 헌화 인터랙션 |
| **ForeverMissed** | 가상 촛불·음악·기념일 자동 알림 |

## 7. 향후 협의 필요 사항

설계를 진행하며 추가로 결정할 항목들.

1. **유가족 페이지의 콘텐츠 편집권** — 운영진 일괄 vs 유가족 직접 편집 + 운영진 검수.
2. **추모일 자동 알림** — 회원이 특정 선생님 페이지를 "구독"하면 기일/순직 인정일에 메일·푸시 수신할지.
3. **오프라인 행사 연계** — 매년 추모식 라이브 스트리밍을 동일 페이지에서 송출할지.
4. **다국어 지원** — 영문 페이지 필요 여부(국제 연대 차원).
5. **후원 / 연대 액션** — 자료실 외에 모금 또는 청원 페이지 별도 구성 여부.
6. **데이터 공개 범위** — 카운터·메시지를 외부 API로 노출해 자매 단체 사이트와 연동할지.

## 8. 결론

이 추모관은 단순한 기념 페이지가 아니라, **유가족의 마음과 시민의 연대가 만나는 잔잔한 공간**을 지향합니다. 통합 헌화의 위로감, 개별 선생님을 기억하는 구체성, 자료실을 통한 진상규명·제도 변화의 지속성 — 이 세 가지 축이 한 흐름에서 자연스럽게 이어지도록 설계했습니다.
