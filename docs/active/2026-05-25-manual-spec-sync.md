## §0 배경·목표

최근 대형 라운드(온라인 추모관·성과관리 v4·교유협 뉴스/여론 분석·연차 산정 정책·근태 신규·결제 KICC 전환·발송 솔라피 전환)가 줄줄이 운영에 들어갔으나 **사용자/운영자 메뉴얼·AI 비서 학습자료·명세 마스터가 뒤처짐**. 특히 옛 업체명(토스페이먼츠·알리고)이 라이브 메뉴얼·AI 답변·명세에 그대로 노출 중 → 운영 혼란·신뢰 문제.

목표: 3개 문서(메뉴얼·AI 학습자료·명세)를 신규 기능에 맞춰 동기화해 ① 운영자가 화면 없이도 기능을 알고 ② AI 비서가 정확히 답하고 ③ 명세가 현행과 일치하게 한다. (CLAUDE.md §6.18 운영 자립)

**성격**: 콘텐츠/문서 작업. DB·API·외부 연동 변경 0. 스키마 0.

## §4 메인 작업 — 명세 (docs/specs/)

### 4.1 근태관리시스템_명세서.md — 절 추가/보완
- **연차 산정 정책 절 신설**: 모드 A(5인 이하·월 만근 시 +1일)/모드 B(5인 이상·근속 기반 1주년 12일, base+floor((근속-1)/incYears)*incDays, 상한 cap). 슈퍼어드민이 근무 정책 탭에서 택일·CRUD. `att_policies` 6컬럼.
- **퇴근 후 재출근·다중 세션 절**: `att_records.sessions`(JSONB)에 세션 누적, 다중 세션 근무시간 합산. 업무시간 내 퇴근취소/시각 셀프수정, 업무시간 외 재출근.
- **휴가 부여/회수 절**: 잔여 기록 없는 직원 첫 부여 가능·회수는 음수·기존 행 ±1 조정 유지. 전 직원(잔여 0 포함) 표시.
- **퇴근 위치 검증 보완**: 출근(OFFICE·FIELD)은 거점 반경 내, 퇴근도 OFFICE(일반근무)는 거점 반경 강제. REMOTE/FIELD/BUSINESS_TRIP 퇴근 미검증.
- **근태 메뉴 2분리 단서**: '근태 현황'(조회 8탭)/'근태 설정'(설정 4탭).

### 4.2 성과관리시스템_명세서.md — v4 절 보강
- 기존 v4 배너 1줄 → 절로 확장: 마일스톤 정의 71개 전면 교체·매출/비매출 5:5 영역 캡(역할별 cap DB화)·비매출 5카테고리(분기 7개·카테고리당 2개)·매트릭스 텍스트 AI 매핑(`admin-milestone-matrix-parse`).

### 4.3 교유협 뉴스·여론 분석 단서 (기존 명세에 절 추가)
- 독립 명세 신설 안 함(Swain 확정). 가장 근접한 명세(성과/운영 관련) 또는 명세 인덱스에 운영 도구 절로: 네이버 수집+Gemini 분석(요약·워드클라우드·여론·추천·변경점·사건사고)·매일 09:00 cron·수동 재조사·히스토리. featureKey `org_news_analysis`.
  - ※ 마땅한 마스터가 없으면 메인 판단으로 위치 확정(명세 정독 후).

### 4.4 업체명 정정 (명세 전반)
- 결제: 토스페이먼츠 → **KICC**(효성 CMS+ 계좌이체는 유지). 단 정기 자동청구는 효성 CMS+ 유지 표기.
- 발송: 알리고+프록시 → **솔라피(SOLAPI)**. SMS·휴대폰 인증 완료, 알림톡 6종 카카오 승인 대기.
- grep으로 `토스`·`알리고`·`네이버 클라우드` 명세 전수 점검 후 일괄 정정(operational_standards §5 자체 grep 의무).
