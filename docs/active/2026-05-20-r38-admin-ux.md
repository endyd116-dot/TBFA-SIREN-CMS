## §0 목표

R29~R37 시리즈 종결로 명세 100% 동작 보장됐지만 어드민 실 사용 흐름에서 UX 격차 3건 발견:
1. **사람별 마일스톤 CRUD 부재** — 직원 클릭으로 본인 마일스톤 일람·관리 불가 (칼럼별만 가능)
2. **직원 출퇴근 월/리스트 부재** — 어드민이 일별 현황만 가능·월 누적·필터 검색 불가
3. **회원 추가·운영자 권한 부여 안내 부재** — Swain이 진입점 못 찾음

- [ ] 신규 어드민 화면 — admin-workspace-management.html "근태 현황" 탭 옆에 "📅 출퇴근 기록" 탭 추가
  - 직원 선택·기간 선택 (월 단위 기본·일자 범위 옵션)
  - 결과: 일자·근무형태·상태·출퇴근 시각·근무시간·야근·휴가 일자 표

- [ ] 슈퍼어드민 전용

┌─ 마일스톤 정의 ─ 직원 역할 배정 ─ 분기 관리 ─ 👥 직원별 마일스톤 (신규) ─┐

### cms-tbfa.html 통합 일반 회원 페이지
```
상단 안내 박스 추가:
💡 직원 추가·운영자 권한 부여
  • 신규 회원 추가: [+ 회원 추가] 버튼
  • 운영자 권한: 회원 상세 모달 → "운영자 활성" 토글
  • 어드민 권한: 회원 상세 모달 → "역할" 변경 (regular → admin → super_admin)
  • 성과 담당 역할: 운영 관리 → 성과관리 설정 → 직원 역할 배정 탭
```

## §4 검증 시나리오 (C 라이브 검증·R38 종결 후)

- Q1. 직원별 마일스톤 탭 진입 → 직원 일람 표시
- Q2. 직원 행 클릭 → 모달 열림·해당 직원 milestoneRole 마일스톤만 일람
- Q3. 모달에서 마일스톤 정의 신규 추가·편집·비활성화 → 즉시 반영
- Q4. admin-att-records dateFrom·dateTo 쿼리 → 기간 내 출퇴근 일자 일람
- Q5. 어드민 출퇴근 기록 탭 → 직원 선택 + 월 → 일자별 표 표시
- Q6. CSV 다운로드 (옵션) → 한글 헤더·UTF-8 BOM
- Q7. 통합 회원 페이지 안내 박스 → 진입점 4종 명확 표시
- Q8. 회귀 — R37 회귀·E2E 12 시나리오 정합 유지
