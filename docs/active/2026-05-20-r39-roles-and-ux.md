## §0 목표

운영 시작 직후 발견된 어드민 UX 10가지 부족 + 역할 시스템 동적 전환을 한 라운드로 통합·각 단계 끝마다 회귀 확인.

### 끝나면 가능해지는 일

**역할 시스템**
1. 슈퍼어드민이 **역할 자체를 자유롭게 추가·이름 변경·없애기** (예: "마케팅실장" 신설·기존 SI 이름 변경)

**성과관리 UX**
2. 직원 이름 클릭으로 **그 직원의 마일스톤을 한 번에 일람·수정·추가**
3. **운영자가 제출한 비매출 성과를 슈퍼어드민이 검토·승인·반려**하는 화면

**근태관리 UX**
4. 직원 + 월 고르면 **그 달 출퇴근 기록 일자별 표·CSV 내려받기**
5. **지금 어떤 직원이 출근·퇴근했는지 실시간 현황** (근태 설정 → 근태현황 보강)
6. 직원이 찍은 **출근·퇴근 위치를 카카오 지도로 시각화** (재택·사무실·외근 모두)
7. 어드민이 **직원 잔여 휴가를 직접 부여·차감** (자동 부여 외 수동 조정)
8. 어드민이 **직원 출퇴근 기록을 직접 수정** + 직원에게 "잘못 찍었으니 확인하라" 알림
9. **PC·노트북에서도 위치정보 강제 수집** (모바일만 아니라 PC도 navigator.geolocation으로 좌표 기록)
10. **워크툴 메인 화면 상단에 출퇴근 버튼** — 워크스페이스 제목·상태 메시지("늦은 밤이에요") 옆·한 번 누르면 출근·두 번째 누르면 퇴근·알림 발송

**기타**
11. 회원 페이지에 **직원 추가·운영자 권한 부여 흐름 안내**
12. **워크스페이스 진입 시 로딩 지연 fix** (성과·근태 메뉴 진입 속도)

### 단계 분할 원칙
- 각 단계 끝마다 B push → 메인 머지 → 동작 확인 → 다음 단계 트리거
- 회귀 위험(결산·진행률·임계점 알림·AI 매칭 등) 사전에 차단
- 도중 막히는 영역 발견 시 즉시 메인 보고·범위 조정

## §1 사전 정독 결과

### 역할(SM/PM/SI) 하드코딩 위치 — 백엔드 5곳·프론트 4파일·총 11~15곳
- 역할 배정 검증 valid 배열
- 결산 CSV 한글 라벨 매핑
- 워크스페이스·어드민 화면 라벨 객체·드롭다운

### 비매출 검토 UI 부재
- 서버 코드는 검토(REVIEW)·승인(VERIFIED)·반려(REJECTED)·EVENT_RANGE 금액 결정 4가지 액션 완비
- 사람이 클릭할 화면이 없음 → 알림 받아도 처리 경로 막힘

### 부분 존재 영역
- 출퇴근 수정 요청: 직원 신청·어드민 결재 흐름은 완비
- 잔여 휴가 부여·차감 API: 완비·UI는 일부 (어드민이 수동 조정 UI 부재)
- 출퇴근 위치 lat·lng: DB에 저장 완비·시각화 부재

## §2 7단계 작업 흐름

### Stage 1 — DB 기반 신설 (1일)
**목표**: 역할 정보를 코드 박힌 상수 → DB 테이블로 옮김. 단 기존 동작은 그대로 유지.

작업:
- 신규 테이블 — 역할 카탈로그 (코드·이름·설명·정렬순서·활성여부)
- 1회용 마이그 함수 작성·Swain 호출
- 시드: SM/PM/SI 3건 자동 적재 (호환성)
- schema 정의 활성화·마이그 파일 삭제

머지 전 확인: 기존 매출 입력·결산·진행률 등 모두 정상 동작.

---

### Stage 2 — 역할 카탈로그 API + 백엔드 검증 동적화 (1일)
**목표**: 백엔드가 역할 정보를 DB에서 읽어 검증. 신규 역할 추가해도 자동 인식.

작업:
- 신규 API: 역할 카탈로그 일람(전 계층)·등록·수정·비활성화(슈퍼어드민만)
- 역할 배정 검증 동적화 (valid 배열 제거)
- 결산 CSV 라벨 동적화
- 영향 함수 전수 점검 (역할 참조하는 5곳)

머지 전 확인: 기존 SM/PM/SI 매출·비매출·결산 회귀 없음.

---

### Stage 3 — 프론트 라벨 동적화 + 역할 관리 화면 (1.5일)
**목표**: 모든 화면이 역할 카탈로그 API에서 라벨·드롭다운. 슈퍼어드민이 화면에서 역할 추가·수정·비활성화.

작업:
- 4개 JS 파일 라벨 객체 → API 호출·캐싱 패턴 전환
- 라벨 헬퍼 함수 도입
- 성과관리 설정에 신규 탭 "🎯 대상 역할 관리"
  - 일람·신규 등록·수정·비활성화
- 신규 역할 코드 영문 대문자 2~10자 검증

머지 전 확인: 기존 SM/PM/SI 표시 정상·신규 역할 추가 → 모든 화면 즉시 반영.

---

### Stage 4 — 사람별 마일스톤 + 회원 안내 (1일)
**목표**: 직원 클릭으로 마일스톤 관리·회원 페이지 안내 박스.

작업:
- 성과관리 설정에 신규 탭 "👥 직원별 마일스톤"
  - 직원 일람·직원 클릭 → 모달로 마일스톤 정의 CRUD
- 통합 회원 페이지 상단 안내 박스 (회원 추가·운영자 토글·역할 변경·성과 역할 배정 4종)

머지 전 확인: 기존 마일스톤 정의 탭 동작 회귀 없음.

---

### Stage 5 — 출퇴근 월/리스트 + 실시간 현황 + 카카오 지도 + PC 위치정보 (2일)
**목표**: 출퇴근 시각화 + 모든 디바이스 위치 기록.

작업:
- 출퇴근 기록 API 확장 (기간 쿼리)
- 워크스페이스 관리 화면 신규 탭 "📅 출퇴근 기록"
  - 직원 + 월 선택·일자별 표·CSV 다운로드
- 워크스페이스 관리 근태 현황 탭 보강
  - 지금 출근한 직원·지금 퇴근한 직원·미출근 직원 실시간 표
  - 자동 새로고침 (예: 30초)
- 출근·퇴근 시각 옆에 "📍 위치 보기" 버튼
  - 카카오 지도 모달 (Kakao Maps JS SDK)
  - 사무실·외근·재택 모두 동일 — 직원이 찍은 좌표·거점 지정 좌표 동시 표시
  - 거리 표시 (Haversine 활용·lib/att-utils 활용)
- **PC·노트북 위치정보 강제 수집** (추가 #9)
  - 직원 출퇴근 화면(workspace-attendance·신규 워크툴 상단 버튼)에서 navigator.geolocation 호출
  - 모바일·PC 무관·위치 권한 거부 시 안내 + 출퇴근 차단
  - att-checkin·att-checkout 응답이 이미 lat/lng 받음·서버 변경 0
  - 클라이언트만 위치 권한 요청 흐름 통일

머지 전 확인: 기존 근태 현황 탭 회귀 없음·카카오 지도 정상·PC에서 위치 권한 거부 시 안내.

---

### Stage 6 — 워크툴 상단 출퇴근 버튼 + 알림 (1일)
**목표**: 워크스페이스 메인(워크툴) 진입 시 한눈에 출퇴근 가능.

작업:
- 워크툴 화면(`workspace.html`) 상단 헤더 영역
  - 워크스페이스 제목 + 상태 메시지("늦은 밤이에요" 등) 옆
  - 출퇴근 버튼 1개·상태에 따라 라벨 자동 변경
    - 미출근: "🟢 출근" (클릭 시 출근)
    - 출근 중: "🔴 퇴근" (클릭 시 퇴근)
    - 퇴근 완료: "✅ 퇴근 완료 (재출근하려면 새로고침)"
- 클릭 시 navigator.geolocation 위치 받아서 att-checkin·att-checkout API 호출
- 출퇴근 성공 시 인앱 알림 + 토스트 + 슈퍼어드민에게도 (선택) 출퇴근 활동 알림
- 위치 권한 거부 시 안내 + 차단

머지 전 확인: 기존 workspace-attendance 출퇴근 흐름 회귀 없음·이중 출퇴근 방지.

---

### Stage 7 — 휴가 수동 CRUD + 출퇴근 양방향 수정 (1.5일)
**목표**: 어드민이 직원 휴가·출퇴근을 직접 조정·역방향 알림.

작업:
- 휴가 잔여 수동 부여·차감 UI
  - 워크스페이스 관리 화면 "잔여 휴가" 탭 보강
  - 직원 일람·각 직원 옆에 [+1] [-1] [숫자 직접 입력] 버튼
  - 수동 조정 시 사유 입력 필수·이력 적재 (감사용)
- 어드민 직접 출퇴근 수정 (어드민 → 직원)
  - 워크스페이스 관리 "출퇴근 기록" 탭에서 일자 클릭 → 어드민 수정 모달
  - 출근 시각·퇴근 시각·근무형태·메모 편집·`is_manually_adjusted=true` 마킹
- 어드민이 직원에게 "잘못 찍었으니 확인하라" 알림
  - 위 모달에 "📨 직원에게 확인 요청" 버튼
  - 직원에게 인앱 알림 + 워크스페이스 알림 패널 표시

머지 전 확인: 기존 휴가 자동 부여·결재·att_records 흐름 회귀 없음.

---

### Stage 8 — 비매출 검토 화면 + 로딩 지연 fix + 통합 검증 (1.5일)
**목표**: 마지막 큰 격차 + 성능 + 회귀 종결.

작업:
- 비매출 검토 화면 (어드민·슈퍼어드민)
  - 어드민 마일스톤 화면 신규 탭 "비매출 검토"
  - 상태 필터·일람·상세 모달·4가지 액션
- 로딩 지연 진단·fix
  - 워크스페이스 진입 시 성과·근태 메뉴 진입 속도 측정
  - 원인 추정: 다중 API 호출 직렬·드리즐 leftJoin·캐시 없음
  - 병렬 호출·캐싱·필요한 컬럼만 SELECT 등
- 전체 회귀 시나리오 (B 자체 검증):
  - SM/PM/SI 매출·비매출·결산·진행률 정상
  - 신규 역할 추가 → 마일스톤 부여 → 매출 입력 → 결산 정상
  - 비매출 제출 → 슈퍼어드민 검토·승인 → 운영자 알림
  - R37 급여 자동 집계 회귀
  - E2E 12 시나리오 회귀
- push 후 메인 머지 → C 라이브 검증 트리거

산출: 모든 격차 해소·운영 가능 상태 진입.

## §4 회귀 위험 영역 (단계마다 점검)

| 영역 | 회귀 위험 |
|---|---|
| 매출 입력·검증 | 역할 분기 동적화 시 응답 키 변경 가능 |
| 비매출 N/2 선택 | 역할별 한도 검증 동적화 시 누락 가능 |
| 결산 자동계산 | 역할별 마일스톤 합산 |
| 분기 cron | 역할 필터 동적화 |
| AI 자동 매칭 | 신뢰도 + 역할 분기 |
| 임계점 알림 | 역할별 담당 그룹 발송 |
| 진행률 대시보드 | 역할별 게이지·집계 |
| CSV·PDF 라벨 | 라벨 매핑 동적화 |
| 휴가 자동 부여 cron | 수동 CRUD와 충돌 가능 |
| att_records UPSERT | 어드민 직접 수정 시 R35 H-G2 보존 정책 유지 |

각 단계 머지 전 B 자체 점검·메인 머지 후 라이브 확인.
