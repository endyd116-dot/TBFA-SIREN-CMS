## ③ 마일스톤 매트릭스 AI 매핑 (성과분석)

### Swain 요구 (원문 요지)
> 마일스톤 셋팅할 때, **마일스톤 매트릭스(워드/표)를 새 분기에 업데이트하면** AI든 내부 시스템이든 **매핑하여 새 마일스톤이 셋업**되도록. 기존 마일스톤 중 **겹치는 게 있으면 삭제할지 수정할지 등을 물어봐** 줄 것. 언급 안 된 워크플로우는 판단해서 보완.

### 접근·판단 (이전 메인)
- **입력 방식**: 워드 파일 직접 파싱은 불안정 → **표/텍스트 붙여넣기** 권장(분기 성과 기준표를 텍스트로 입력).
- **AI 파싱**: Gemini로 매트릭스 텍스트 → 마일스톤 항목(이름·설명·가중치·역할 등) 구조화 추출(`callGeminiJSON`).
- **기존 매핑·충돌**: 새 분기 마일스톤 후보 ↔ 기존 마일스톤 정의 유사도 비교 → 겹치면 **"수정/삭제/유지" 선택 모달**로 어드민이 결정.
- **소프트삭제**: 결산 참조 보존을 위해 기존 정의는 소프트삭제(하드삭제 금지).

### 정독·관련 코드 (설계 전)
- 마일스톤 정의 단일 API: `milestone-definitions`(성과관리 통합 시 신설·소프트삭제). `netlify/functions/` 에서 grep.
- 성과관리 화면: `public/admin-milestones.html` / `public/js/admin-milestones.js` (6탭 통합)
- 분기 구조: `quarters`·`quarterly_settlements`·`milestone_roles`(SM/PM/SI) 테이블 schema 정독
- 역할 카탈로그: R39에서 동적 CRUD화 (`milestone_roles`)

### 미결정 (Swain 확인 필요)
- 매트릭스 입력 형식 최종(표 입력 UI vs 텍스트 vs CSV 업로드)
- 매핑 단위(마일스톤 정의 자체 vs 사람별 마일스톤)
- AI 추출 실패 시 수동 폴백 범위

## ④ 교사유가족협의회 최신 뉴스 분석

### Swain 요구 (원문 요지)
> 최근 1주일간 **교사유가족협의회 / 박두용 대표** 관련 모든 뉴스·블로그·방송 등 콘텐츠를 조사 → 최근 어떤 활동/이슈인지 보여주고, **연관 키워드를 워드클라우드**로. **AI가 추천한 기능**도 추가.
> 분석은 AI가 하되, **버튼 누르면 최신 재조사 / 기본은 매일 아침 1회 자동 조사**해 채움. **이전과 비교해 바뀐 점**도 AI가 판단해 같은 탭에 신설. 보고서가 새로 바뀌면 **히스토리**를 남겨 이전 기록도 조회. 언급 안 된 워크플로우는 판단해서 보완.

### 결정 (Swain 2026-05-24)
- **수집 = 네이버 검색 API + 분석 = Gemini**. (네이버 개발자 키 발급 필요·무료 쿼터 / Swain env 등록)

### 접근·판단 (이전 메인)
- **수집**: 네이버 검색 API(뉴스·블로그·웹) 로 "교사유가족협의회", "박두용" 등 키워드 최근 1주 결과 수집(중복 제거·날짜 필터).
- **분석(Gemini)**: 수집 본문/스니펫 → ① 활동·이슈 요약 ② 연관 키워드(워드클라우드용 빈도·가중치) ③ 여론 경향(긍/부정) ④ AI 추천 액션. `callGemini`/`callGeminiJSON` + featureKey 신규 등록(예: `org_news_analysis`).
- **자동/수동**: 매일 아침 cron(예 KST 07:00) 1회 자동 생성 + 어드민 "최신 재조사" 버튼(즉시).
- **변경점 diff**: 직전 보고서와 비교 → 신규 이슈·사라진 이슈·키워드 변화 AI 요약(같은 탭).
- **히스토리**: 보고서 갱신 시 이전 버전 보존(테이블 행 누적) → 과거 보고서 조회.
- **워드클라우드**: 클라이언트 라이브러리(예 wordcloud2.js CDN) 또는 CSS 태그 클라우드.

### 필요 인프라
- **스키마(마이그)**: 뉴스 분석 보고서 테이블(요약·키워드 JSONB·여론·추천·생성일·소스 링크) + 히스토리(같은 테이블 행 누적 또는 버전 컬럼).
- **cron**: `cron-org-news.ts` (매일 아침) + `netlify.toml` 등록.
- **env(Swain)**: `NAVER_SEARCH_CLIENT_ID`·`NAVER_SEARCH_CLIENT_SECRET` (네이버 개발자센터 검색 API).
- **화면**: 통합 CMS 새 메뉴/탭(예 "여론·뉴스 분석") — 요약·워드클라우드·변경점·히스토리 목록.
- **featureKey**: `lib/ai-feature.ts`에 `org_news_analysis` 등록.

### 미결정 (Swain 확인 필요)
- 화면 위치(독립 페이지 vs 기존 CMS 탭) / 메뉴명
- 검색 키워드 세트(대표명·협회명·약칭 등) 확정
- cron 시각 / 수동 재조사 비용 가드(AI 호출 한도)
- 워드클라우드 라이브러리 선택
