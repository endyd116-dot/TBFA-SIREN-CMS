- 출근 도장이 아니라 **실제 근무시간**으로 그날 지급일수: 8h↑ 1.0 / 6~8h 0.75 / 4~6h 0.5 / 2~4h 0.25 (`payDayFraction`, 유예 `PAY_DAY_GRACE_MINS=10`분).

### 신규 기능
- **전자서명·증빙보관**: 직원이 서류 모달로 계산근거 확인 → 손글씨/성명 수령확인 서명 → 교부 PDF를 R2에 고정(sha256) → 관리자 문서함·서명증적. 이의제기·정정재발행·연간내역서.

- **조정 항목 과세/비과세 구분** + 4대보험 자동 재계산.
- **법정 신고 서류** (급여관리 → 신고·법정 서류, super 전용): 임금대장(PDF/엑셀)·원천징수(홈택스 복사버튼)·**간이지급명세서**(국세청 양식 .xls 브라우저에서 채움)·연간집계·4대보험. **주민번호 미저장**(브라우저에서만 엑셀 생성). → 상세 메모리 `project_payroll_statutory`.
- **[지급] 워크플로우 가드**: 서명하면 초록·미서명이면 흐리게 + 실제 이체일 입력(원천징수는 지급일 기준).
- **재택보고서 미제출 = 근무 불인정**(7월부터·재택일+3일 마감, `lib/att-remote-policy.ts`).
- **근태 수정요청 파일 첨부**(본인 파일함 경유, `lib/att-evidence.ts`).

### PDF 함정 (재발 주의)

## 2. KST 전면 정비 — 무엇을 왜 (7/12~13)

Swain 지시: "우리 다 KST 기준으로 쓰니까 KST로 통일". 서버(Netlify)는 UTC로 도는데 곳곳이 UTC를 그대로 써서 **한국 새벽에 하루 밀리거나 시각이 9시간 어긋나던** 것을 전부 잡음.

### 🚫 절대 금지 (실측 확인 후 배제)
- **서버/DB 세션 TZ를 `Asia/Seoul`로 바꾸기** → postgres-js가 timestamp 문자열을 `new Date`로 파싱하는데 프로세스가 서울이면 **읽는 모든 시각이 9시간 밀린다**. 재앙.

### 올바른 구조 (단일 출처 = `lib/kst.ts` / 화면 `public/js/lib-kst.js`)
- 저장·전송은 **UTC 그대로**. 변환은 사람이 보거나 날짜로 판정할 때만.
- 서버 '오늘/지금': `todayKST()`·`nowKST()`·`yearKST()`·`hourKST()` (전부 getUTC 기반 → 프로세스 TZ 무관).
- SQL: `CURRENT_DATE` 금지 → `(NOW() AT TIME ZONE 'Asia/Seoul')::date`. 저장 timestamp는 **2단계** `AT TIME ZONE 'UTC' AT TIME ZONE 'Asia/Seoul'`.
- **API 응답: 반드시 `jsonKST()`(=JSON.stringify 대체)·`jsonRes()`(=Response.json 대체)로만 만든다.** raw SQL 시각이 시간대 표시 없이 새어 브라우저가 9시간 오해하던 것을 이 관문이 막음(`isoUTC`로 Z 부착).
- 화면 표시: `lib-kst.js`가 전 페이지 head에서 `Date.prototype.toLocale*` 기본 TZ를 서울로 못박음.
- **입력**(datetime-local): `kstLocalToISO()`로 벽시계+09:00 고정(브라우저 TZ 무관).
- **엑셀 날짜 컬럼**: `dateKST()`.

### ⚠️ 핵심 교훈 (다음 메인 필독)
1. **'날짜' vs '시각(instant)' 구분**: `next_retry_at`처럼 `<= 지금` 비교하는 값은 시각이라 KST로 옮기면 안 됨(9시간 늦어짐). 실제로 그렇게 고쳤다 되돌린 회귀 있음.
2. **패턴을 하나씩 쫓지 말고 나가는 길목에서 막아라**: 시각이 새는 경로가 필드별→raw행→SQL별칭→Response.json 순으로 **4번** 나왔다. 관문(`jsonKST`/`jsonRes`)으로 통일해서 잡음.
3. 상세 = 메모리 `project_kst_policy`(사무실 노트북엔 없으니 이 문서로 갈음).

### 덤으로 잡은 것 (원래 깨져 있던 API)
- 휴가 신청 내역 조회(500·`.rows` undefined), 전표 목록(500·NULL 파라미터 타입), 신고 게시판(500·enum UNION), 워크스페이스 7개 화면 사용자정보 빈 껍데기.

### 검증 방식 (재현용)
- 진짜 Postgres = **PGlite**(`@electric-sql/pglite`, 스크래치패드에 설치)로 SQL 문법·시간대 경계 테스트(로컬은 Netlify DB 외부접속 차단).
- 라이브 = 관리자 로그인 후 전 GET API 훑어 `"YYYY-MM-DD HH:MM:SS"`(Z 없음) 패턴 탐색.
- **라이브 관리자 계정 = `admin` / `admin12345`** (⚠️ Swain에게 비번 변경 요청함 — 아래 미결).
