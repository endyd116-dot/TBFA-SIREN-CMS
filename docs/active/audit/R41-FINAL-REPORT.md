## 1. 보안 — 가장 시급했던 문제들 (수정 완료)

### 1-1. 비공개 첨부파일을 아무 회원이나 내려받을 수 있었음 〔P0·개인정보〕
- **문제**: SIREN 신고 증거(괴롭힘 캡처·진단서)와 딥릴리프 유족 자료(사망현장 사진·심리부검·가족관계증명서)는 비공개로 저장되는데, 이 파일을 내려주는 통로가 **"관리자거나, 그냥 로그인만 되어 있으면"** 통과시켰다. 파일 번호가 1·2·3… 연속이라, 일반 후원자 계정으로 로그인해 주소의 번호만 바꾸면 남의 가장 민감한 자료를 그대로 받을 수 있었다.
- **수정**: 비공개 파일은 **관리자이거나, 본인이 직접 올린 파일만** 내려받게 막았다. 남의 번호로 접근하면 거부(403).

### 1-2. 워크스페이스 파일 공유에 소유자 확인이 없었음 〔P1〕
- **문제**: 로그인한 운영자면 누구나 남의 파일을 임의의 사람에게 공유하거나, 기존 공유를 끊거나, 누구에게 공유됐는지 볼 수 있었다.
- **수정**: 소유자·권한을 확인하는 보호된 경로로 일원화하고, 검증 없던 옛 경로는 폐기.

### 1-3. 작업(체크리스트·리마인더·서브태스크·반복)에 권한 확인 누락 〔P1〕
- **문제**: 작업 번호만 알면 남의 작업에 체크리스트를 덮어쓰거나 리마인더·하위작업을 만들 수 있었다.
- **수정**: 네 경로 모두 "내 작업이거나, 내가 담당/지시한 작업"인지 확인하도록 보강.

### 1-4. 게시판 구독 알림으로 누구나 사칭·스팸 발송 가능 〔P1〕
- **문제**: 로그인만 하면 발신자 이름·내용을 마음대로 넣어 **구독자 전원에게** 알림을 쏠 수 있었다.
- **수정**: 발신자·내용을 사용자 입력에서 안 받고 **실제 댓글 기록(작성자 본인 확인)**으로만 생성. 숨김 댓글은 발송 안 함.

### 1-5. 만료된 공유 링크로 계속 다운로드 〔P1〕
- **문제**: "○월○일까지"로 공유해도 기한이 지난 뒤에도 계속 받아졌다.
- **수정**: 다운로드·조회 모든 경로에 만료시각 검사 추가.

## 2. 결제·후원 — 돈이 잘못 흐르던 문제 (수정 완료)

### 2-1. 정기 후원 결제가 실패하면 매일 무한 재청구 + 자동해지가 영원히 안 됨 〔P1〕
- **문제**: 잔액부족 등으로 정기결제가 한 번 실패하면, 다음 날 "정상 결제"로 다시 잡혀 **매일 재시도**됐다. 시도 횟수가 항상 1회로 고정돼 의도된 *1일→3일→3회 실패 자동해지*가 전혀 작동하지 않았다(죽은 카드가 영구히 매일 청구 시도 + 실패 알림).
- **수정**: 실패 시 다음 결제일을 재시도 날짜로 덮어쓰던 코드를 제거. 이제 재시도는 별도 경로로만 처리돼 1→2→3차로 정상 증가하고, 3회째 자동해지된다.

### 2-2. 정기 후원 첫 결제가 이중청구될 수 있었음 〔P2〕
- **문제**: 카드 등록 후 첫 결제의 주문번호가 매번 랜덤이라, 결제창 복귀가 중복 수신되면 **두 번 청구**될 수 있었다.
- **수정**: 첫 결제 주문번호를 등록 건 기준 고정값으로 → 중복 복귀해도 같은 거래로 처리(1회만 청구).

### 2-3. 위조된 취소/환불 알림으로 완료 후원이 둔갑 가능 〔P2〕
- **문제**: 결제사 알림에 서명 검증이 없어, 내부 주문번호를 아는 사람이 **가짜 취소/환불 알림**을 보내면 완료된 후원이 장부상 취소·환불로 바뀔 수 있었다.
- **수정**: 완료건의 취소/환불 전환은 **서명 검증 통과 시에만** 자동 반영. 미검증이면 보류하고 슈퍼어드민에게 통지(실제 취소 여부 수동 확인).

### 2-4. "정기 해지" 눌러도 계속 청구될 수 있었음 〔P2〕
- **문제**: 일시 후원 해지 화면이 후원 기록만 "해지됨"으로 바꾸고 **카드(빌키)는 안 끊어서** 다음 달 자동청구가 이어질 수 있었다.
- **수정**: 해지 시 카드까지 실제 비활성화 + 결제사 빌키 삭제 + 다음 결제일 해제.

## 3. 익명성·개인정보 보호 (수정 완료)

### 3-1. 익명 신고인데 신고자 실명·연락처가 그대로 노출 〔P1〕
- **문제**: "익명"으로 신고해도 운영자 화면에 회원 정보에서 가져온 **실명(상세는 이메일·전화까지)**이 보였다. 신원을 단계적으로·사유와 기록을 남기며 확인하는 별도 절차가 있는데 일반 화면이 이를 건너뛰어, "누가·왜 익명 신고자를 봤는지" 기록이 전혀 없었다.
- **수정**: 익명이면 이름·연락처를 "익명"으로 가리고, 신원 확인은 **사유·감사기록을 남기는 전용 절차로만** 가능하게.

### 3-2. 익명 게시글의 작성자 회원번호가 응답에 노출 〔P2〕
- **문제**: "익명으로 작성"해도 상세 응답에 작성자 회원번호가 들어가 개발자도구로 추적 가능했다.
- **수정**: 익명이고 본인이 아니면 회원번호를 응답에서 제거.

## 4. 신고·유가족 지원·딥릴리프 (수정 완료)

- **사건 댓글 좋아요·신고가 항상 실패**하던 것(없는 주소 호출 + 미정의 값 참조 오류) → 올바른 통합 경로로 연결, 좋아요 수 갱신 포함.
- **AI 분석을 거친 신고는 작성 직후부터 수정 불가**(주 사용자가 자기 신고를 영영 못 고침) → 운영자 검토 전까지 수정 허용.
- **댓글 신고 "숨김/삭제"가 무조건 거부**되던 것(화면↔서버 상태값 불일치) → 처리 가능하게 정합.
- **딥릴리프 발간물이 검수 없이 곧장 발간** 가능하던 것 → 검수 단계를 거쳐야만 발간되도록 서버에서 강제.
- **지원 상세의 "AI 추천 전문가"가 가상 인물**이라 배정해도 실제 매칭·채팅이 안 생기던 것 → "참고용·실제 배정은 별도" 명시.
- **AI를 건너뛴 신고는 운영자에게 알림이 0건**이라 방치되던 것 → 접수 즉시 운영자 알림.
- (그 외 통합 신고 목록 필터·집계, 발간 실패 가시화, 검토 멱등, 인계 중복방지 등 다수)

### 4-1. 추모관 부적절 콘텐츠 — AI 사전검토 도입 (확정 정책대로)
- **문제**: 추모 메시지·편지가 작성 즉시 공개돼, 부적절 글도 누군가 신고한 뒤에야 숨길 수 있었다.
- **수정**: 작성 시 **AI가 먼저 검토 → 부적절하면 우선 비공개로 보류 + 운영자·어드민·슈퍼어드민에게 통지**. 정상이면 즉시 공개. (AI 비용 안전장치 경유, AI 실패·꺼짐 시엔 정상 글을 막지 않도록 통과) + 어드민에 '미검토(최신순)' 탭.
- **헌화 카운터**가 헌화할 때마다 "전체 합계 → 일부 합계"로 **숫자가 급감**해 보이던 것도 수정.

## 5. 내부 운영 — 워크스페이스·근태·급여·성과 (수정 완료)

- **휴가 일수가 주말·공휴일까지 포함**돼 연차가 실제보다 많이 깎이던 것 → 영업일만 차감.
- **승인된 휴가를 취소·복원할 방법이 없어** 안 쓴 연차가 영구 소멸하던 것 → 승인 취소(연차 복원) + 직원 본인 철회 경로 신설.
- **반차를 내도 그 날이 통째로 휴가 처리**돼 근무분이 사라지던 것 → 반차 별도 처리(근무시간 반영).
- **캘린더에서 일정 만드는 버튼이 죽어 있던 것** → 일정 생성/수정 모달 구현.
- **참석 응답(RSVP) 알림이 주최자에게 안 가고** 데이터가 두 곳에 분산되던 것 → 하나로 통일 + 주최자 알림.
- **멘션 알림 읽음 처리가 안 되던 것**, **알림 더보기가 작동 안 하던 것** 등 수정.
- **분기 성과급이 앞 달 급여명세에서 누락**되던 것 → 정산 확정 시 해당 분기 3개월 급여 재집계.
- 운영자가 못 들어가던 성과 분류 화면 권한, 근태 통계 과다 집계, 인센티브 캡 미반영 등 다수.

## 7. 콘텐츠·발송·재정 (수정 완료)

- **공개 캠페인·자료실 페이지에 헤더·푸터·후원 버튼이 아예 안 뜨던 것**(붙일 자리 이름 불일치) → 통일.
- **홈/마이페이지 큐레이션이 여러 개 등록해도 1개만 노출**되던 것 → 전부 정렬 노출.
- **응답폼 응답 삭제가 실패를 "성공"으로 위장**하던 것(임시 가짜 코드 잔존) → 실제 삭제 연결·실패 표시.
- **대량 발송 컴플라이언스(확정 정책)**: 수신거부·번호 미인증·탈퇴 회원을 **모든 대량발송에서 강제 제외**(문자·카카오·메일). ※이메일은 "전역 수신거부" 컬럼이 없어 탈퇴자만 제외 — 진짜 이메일 구독해지 강제는 §8 후속.
- **전표 번호 발번 동시성**(advisory lock 직렬화) + 통장↔전표 원자화로 중복·꼬임 방지.
- **에러 응답 표준화**(step·detail·stack) 전 영역 공통 적용 — 장애 시 원인 추적 쉬워짐.
- (메뉴 재귀삭제·반복전표 짧은달·롤업 통계 등 다수)

## 8. 권한 토글 도입 (P-1 정책 — 확정대로)

민감 작업(후원 확정·환불·정산 조회·AI 시스템프롬프트 변경·발간물 내보내기)에 **권한 키 5종**을 붙이고, **권한 정책 메뉴(admin-role-policy)에서 슈퍼어드민이 켜고 끌 수 있게** 했다.
