## BUG-1 (P0·fix 완료) — 설치 후 수집·목록·설정이 전부 실패

**증상**: 마이그(`migrate-org-news?run=1`) 호출로 표를 만든 뒤, '최신 재조사'·보고서 목록·설정 저장이 전부 500.

**원인**: 마이그가 만드는 표의 칸 이름·타입이 서버 코드가 실제로 쓰는 것과 전면 불일치. 마이그는 설계 §4 그대로(`source_count·sources·status`, `keywords` jsonb, reports에 `scopes/per_combo` 없음, settings에 `per_combo/cron_hour_kst/updated_by` 없음)인데, 서버(refresh·cron INSERT / list·get SELECT / settings)는 `collected_count·items·ai_status·scopes·per_combo·cron_hour_kst·updated_by`와 `keywords/scopes`를 `text[]`(raw 배열 바인딩)로 사용. 마이그가 500을 내서 A·B가 한 번도 같이 실행된 적 없어 미발견. (§6.1 필드·타입 일치 / §9.1.1 schema-DB 동기 위반)

**fix**: `netlify/functions/migrate-org-news.ts` 의 두 CREATE TABLE을 **서버가 실제 쓰는 컬럼·타입**으로 재작성.
- reports: `keywords text[]·scopes text[]·per_combo·collected_count·items jsonb·summary·keyword_cloud jsonb·sentiment jsonb·recommendations jsonb·diff_summary·ai_status·trigger_type·generated_by·created_at` + `created_at DESC` 인덱스.
- settings: `id·keywords text[]·scopes text[]·per_combo·auto_enabled·cron_hour_kst·updated_at·updated_by` + 싱글톤 CHECK + 시드(`keywords` raw 배열).
- 옛 마이그가 부분 실행된 경우 대비 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 방어 추가(멱등).

**fix 방향 근거**: 서버 5엔드포인트+cron은 동일 스키마로 **내부 일관**·면적 큼. 마이그(1파일)를 서버에 맞추는 게 최소 변경·무회귀. (서버 컬럼명은 응답 키 방향과 무관하므로 이 fix는 방향 독립적)

## BUG-2 (P1·fix 완료) — 서버가 동작해도 화면이 비어 보임

**증상(BUG-1 해소 후)**: 보고서가 있어도 "보고서가 없습니다", 수집 건수 0건, 생성시각·수집 소스 목록·히스토리 날짜 공백.

**원인**: 화면(A)이 설계 §6 계약 키·구조로 작성됐는데 서버(B)는 다르게 반환.
- **언래핑**: 화면은 `data.report`(중첩) 기대 → B get/refresh는 `data:{...필드 직접}`(평면) → 보고서 항상 null.
- **키 이름**: 화면 `sourceCount/generatedAt/sources/sourceType/pubDate` ↔ B `collectedCount/createdAt/items/scope/date` → 건수 0·시각/소스/날짜 공백.

**fix**: `public/js/admin-org-news.js` — **평면/중첩 양쪽 수용 + 서버 키 폴백 가산**(기존 계약 키 폴백 유지 → 어느 쪽이든 동작).
- `extractReport()` 헬퍼 신규(`data.report` 또는 평면 `data` 자동 판별) → get·refresh·history 상세 3곳 적용.
- 폴백 추가: `sources||items`·`sourceCount||collectedCount`·`generatedAt||createdAt`·소스 `sourceType||scope`·`pubDate||date`·히스토리 행 동일·수집기간은 값 있을 때만 표시.
- 캐시버스터 `?v=1 → ?v=2` (admin-org-news.html).
- **가산 변경이라 회귀 0**: 기존 키가 오면 그대로 동작, 서버 키가 오면 새 폴백이 잡음.

**변경 파일**: `netlify/functions/migrate-org-news.ts` · `public/js/admin-org-news.js` · `public/admin-org-news.html`(캐시버스터)

## 관찰 (비차단·참고)

- **설계 문서 §4/§6 stale**: 구현이 문서 계약과 어긋남. 본 fix는 화면이 양쪽 키를 모두 수용하므로, 메인이 ① 문서를 구현에 맞춰 동기화하거나 ② 문서 계약 우선이면 서버(B)를 계약 키로 정렬(이 경우 현재 화면도 그대로 동작) 중 택1. 권장=①(구현이 더 풍부·일관).
- **featureKey 표기차**: 등록은 됐으나 category가 설계 `admin_action` vs 구현 `cron_daily`, 이름도 다름. 기능 영향 0(fail-open)·AI 설정 화면 분류만 다름.
- **네이버 키 무효(401) 시**: 키가 "설정됐으나 틀린" 경우 개별 호출 실패를 catch → `ok:true·items:[]`(빈 보고서)로 표출. 키 미설정만 명시적 fail-closed. 운영 시 키 오류가 빈 보고서로 보일 수 있음(명시적 에러 아님) — 비차단이나 인지 필요.
- **옛 마이그 기 실행 시**: 가능성 낮음(pending Swain 액션)이나, 이미 옛 스키마로 실행됐다면 `keywords` 타입(jsonb)은 ADD COLUMN으로 못 고침 → 두 표 DROP 후 재실행 권장(보고서 데이터 없음·설정 시드만 재생성).
