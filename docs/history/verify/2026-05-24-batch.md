## BUG-1 (P1·fix 완료) — 어드민 출퇴근 정정이 같은 날 직원 동작으로 되돌아감

**증상(사용자 관점)**: 어드민이 **오늘 근무 중인 직원**의 출근(또는 퇴근) 시각을 바로잡아도, 그 직원이 평소처럼 퇴근하거나 재출근/셀프수정을 하면 **어드민이 고친 시각이 원래대로 되돌아간다**(조용한 데이터 손실).

**원인**: 이번 배치에서 하루 여러 출퇴근을 담는 **세션 배열(att_records.sessions)** 이 새로 도입됐고, 직원 출퇴근(att-checkin·att-checkout·att-session-edit)은 이 세션 배열로부터 요약 시각(출근·퇴근·근무시간)을 **재계산**한다. 그런데 어드민 시각 수정 경로 2곳이 요약 컬럼만 고치고 **세션 배열은 그대로 두어**, 다음 직원 동작 때 옛 세션 기준으로 재계산되며 어드민 수정을 덮어썼다.
- `admin-att-record-edit`(어드민이 기록을 직접 수정) — 가장 직접적, 오늘 근무 중 직원 정정 시 정상 퇴근만 해도 재발.
- `admin-att-correction-review`(직원 정정 요청 결재 승인) — 같은 근본 원인(주로 과거일이라 노출 낮으나 오늘 정정 시 동일).

CLAUDE.md §9.1.8(헬퍼·컬럼 도입 직후 모든 사용처 1회 검증) 누락 패턴. 화면 표시·통계·급여는 요약 컬럼만 읽어 영향 0이라 코드 정독으로만 드러남.

**재현**: ① 직원 09:10 출근(근무 중) → ② 어드민이 출근 09:10→09:00 정정 → ③ 직원 18:00 정상 퇴근 → ④ 세션[0].in=09:10(옛값) 기준 재계산 → 출근시각 09:10으로 환원(정정 소실).

**fix**: 어드민이 시각을 수정하면 **세션 배열을 요약 시각에 맞춰 정합화**.
- 신규 공용 헬퍼 `rebuildSingleSession(in, out, carry)` (`lib/att-session.ts`) — 위치·거점 정보 보존, 퇴근 전이면 진행 중(out=null) 단일 세션 유지(직원 정상 퇴근 가능), 다중 세션은 단일로 정규화(어드민 수정의 단일-span working_mins 재계산과 일치).
- `admin-att-record-edit`: old 조회에 sessions·위치 컬럼 추가 → 시각 수정 시 sessions 동기화 SET.
- `admin-att-correction-review`: UPSERT에 sessions 컬럼 추가(INSERT·DO UPDATE 양쪽).
- 시각 미수정(근무형태·메모만 수정)이면 sessions 미터치 — 불필요한 정규화 없음.

**fix 후 트레이스**: 위 재현 ③에서 세션[0].in=09:00(정정값) → 퇴근 재계산해도 출근 09:00 보존 ✅. tsc 0.

**변경 파일**: `lib/att-session.ts`(+27) · `netlify/functions/admin-att-record-edit.ts`(+15) · `netlify/functions/admin-att-correction-review.ts`(+17)
