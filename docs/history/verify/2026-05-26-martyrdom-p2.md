## 2. 발견·수정한 결함 (커밋 `70d22c2`)

### BUG-P2-1 [중대·개인정보] AI 비서 일반 검색이 순직 민감 자료를 노출
- **증상**: 운영자가 일반 AI 비서에게 질문하면, 답변 근거로 **다른 진행 중 순직 사건의 민감 자료**(고인 성명·의료/심리 기록·괴롭힘 정황 등)가 끌려나올 수 있었음. 순직 자료는 담당자 외 노출 금지(운영자 전용·사건 격리) 원칙(§1.2·§2.8·§P2.0 #10)에 위배.
- **원인**: 순직 자료가 공용 의미검색 테이블(`ai_rag_documents`)에 함께 색인되는데, **일반 AI 비서·RAG 진단 검색이 코퍼스 종류를 지정하지 않아 테이블 전체를 검색**(일반 안내 `qna`·`manual` + 순직 `martyr_*` 혼합). 순직 도입 전에는 테이블에 일반 자료뿐이라 드러나지 않던 잠복 결함이, 순직 색인이 쌓이면서 실제 노출로 전환.
- **영향**: 일반 AI 비서(`admin-ai-agent`)·RAG 검색 테스트(`admin-rag-status`). 순직 분석 측(전략·요건·코퍼스 검색)은 이미 `martyr_*`로 격리돼 있어 무관.
- **수정**: 두 일반 검색을 **`qna`·`manual`(일반 코퍼스)로만 한정** → 순직 자료가 AI 비서에 절대 섞이지 않음. 순직 분석은 그대로 `martyr_*`만. 양방향 코퍼스 분리 완성.

> 비고: P1 도입 시점부터 잠복하던 격리 누락이 P2 회귀 점검에서 표면화된 것. 순직 분석 기능 자체는 정상 작동했고, 노출 경로는 "일반 AI 비서"라는 별도 통로였음.

## 4. 핵심 점검 7종 (전부 PASS)

1. **추출 → 전략 자동 체인**: 추출 자동분석 완료 후 active 사건이면 전략 분석을 자동 트리거(요청 전송을 기다려 보장·P1 교훈 반영). ✅
2. **준비도 % 재현성**: 점수는 100% 규칙 계산(AI 미관여). 같은 입력 = 같은 값. AI 첨언은 정성 텍스트만. ✅
3. **정량 % 미노출**: 전략·요건 프롬프트가 확률·일치율 숫자 출력을 금지. 유사 사례는 정성 일치/차이로만 표기. 준비도 %는 "인정 확률 아님" 라벨. ✅
4. **코퍼스 검색 active 제외**: 운영자 코퍼스 검색·전략·요건 모두 과거사례+법령만. 진행 사건 자료 교차 노출 없음. ✅
5. **cron 중복 알림 방지 + 저장용량**: 알림 보낸 기한은 재알림 안 함(상태를 직접 '대기'로 되돌리면 재알림 가능하게 초기화). 용량 임계 알림 포함. ✅
6. **응답 키 1:1(A↔B)**: 대시보드(사건목록·요약·저장용량)·코퍼스(검색결과)·요건(목록/대조 항목·충족수)·준비도(구성·부족·첨언)·동의 — 전부 프론트 소비 키와 일치(프론트는 방어적 폴백까지 보유). ✅
7. **종결 → 학습**: 종결+결과 확정 시 인정 패턴 추출·과거사례 전환·검색 코퍼스 재태깅. 이중 실행 방지. ✅

추가: **백그라운드 4종(`*-background`) 전부 `config.path` 없음** — 자동 체인이 멈추던 P1 근본원인(백그라운드에 경로 지정 시 비동기 호출 미수신) 회피 확인. ✅

## 6. tsc·자산·라우팅

- `npx tsc --noEmit` **통과**(의존성 설치 후·수정 2건 반영).
- 캐시버스터: `admin-martyrdom.js?v=P2` ✅.
- 라우팅: P2 공개 함수 전부 `export const config = { path }` 보유. 백그라운드 4종은 `config` 부재(의도). cron은 `netlify.toml` + 함수 inline 스케줄 모두 등록(KST 08:00). 법령 문서는 번들 `included_files` 포함.
- 권한: 요건 master 쓰기·법령 파싱·사건 삭제 = super_admin. 나머지 운영자. 내부 백그라운드는 내부 시크릿(fail-closed).

## 7. 관찰 사항(결함 아님)

- **준비도 진입점 2개**: 전용 준비도 API와 생성 디스패처(type=readiness) 둘 다 규칙 계산·저장. 동작 동일·충돌 없음.
- **기한 알림 단일 플래그**: 임박 알림이 한 번 나가면 같은 기한의 경과 알림은 별도로 안 나감(중복 방지 플래그 공유). "중복 알림 방지" 설계 의도에 부합. 경과 후 별도 에스컬레이션이 필요하면 P3+에서 분리 검토 가능.
- **종결 학습의 코퍼스 재태깅**: active→과거사례 전환 시 임베딩 재사용(재색인 비용 0)·진행 코퍼스에서 제거. 효율적.

## 8. 남은 라이브 검증(Swain) 권고

1. **자동 체인**: active 사건에 자료 업로드 → 추출 후 전략이 자동 생성(검토 대기)되는지.
2. **분석 생성 3종**: 전략·요건 대조·골든타임 [생성] → 근거 인용·모순/타임라인/반론 카드 표시.
3. **준비도 재현성**: [준비도 계산] 두 번 눌러 같은 % 확인. AI 첨언에 숫자 없는지.
4. **AI 비서 격리(BUG 수정 검증)**: 일반 AI 비서에 순직 관련어로 질문 → **순직 사건 자료가 답변 근거로 안 나오는지**(수정 핵심).
5. **종결 학습**: 사건을 종결+결과 확정 → 과거 학습사례로 전환되고 다른 사건 분석에서 근거로 검색되는지.
6. **cron/용량**: 기한 임박 알림·저장 용량 임계 알림 수신.

> 결론: **P2 구현은 계약·스키마·라우팅·자동 체인·재현성·정량% 금지·코퍼스 격리(분석 측)가 모두 정합**. 회귀에서 발견한 AI 비서 민감자료 노출 누락 1건을 수정해 양방향 코퍼스 격리를 완성했다. 라이브에서 AI 생성 품질·종결 학습만 확인하면 P2 검증 통과 가능.
