## §2. 검증 시나리오 — Q1~Q10 (설계서 §11)

### Q1. cron-payroll-monthly 수동 트리거 → 회원 N명 payroll_slips DRAFT 생성·calculation_snapshot 정합

**검증 경로** (어드민 화면):
1. 슈퍼어드민 로그인 → cms-tbfa → 운영 관리 → 💰 급여관리 진입
2. 연·월 선택(예: 2026년 4월) → "재집계" 클릭 → confirm 통과
3. 응답: `{ candidateCount, created, updated, skipped, errors }` 토스트 표시
4. 일람에 회원별 행 표시 (status=DRAFT)
5. 상세 모달 → calculation_snapshot JSON 펴짐 — `att`·`leave`·`quarter`·`derived` 키 확인

**기대치**:
- candidateCount = members 중 status='active' AND (admin OR operatorActive) AND base_salary>0 인원수
- 각 회원에 대해 working_mins / overtime_mins / late_count / absent_count = att_records 집계
- paid/unpaid_leave_days = att_leave_requests + att_leave_types JOIN (APPROVED·해당 월 시작일)
- gross_pay = base_salary_month + overtime_pay - deduction_unpaid + performance_bonus + perfect_bonus

### Q2. 슈퍼어드민이 명세서 검토 → status=REVIEWED → APPROVED

**검증 경로**:
1. 상세 모달에서 "검토 완료" 버튼 → toast "검토 완료" → status REVIEWED 배지
2. "승인" 버튼 → confirm → toast "승인 완료" → status APPROVED 배지

### Q3. PDF 다운로드 → A4 1페이지·한글 정상·금액 정합

**검증 경로**:
1. 명세서 일람 또는 상세 모달의 "PDF" 버튼 클릭 → 새 탭에 PDF 표시
2. 머리말: "급여명세서 / YYYY년 MM월" + 발행일시
3. 직원 정보: 성명·이메일·직책
4. 근태 4행 2열·급여 5행 + 세전 총액(굵게)
5. NotoSansKR 한글 정상 표시 (깨짐 없음)

### Q4. 일괄 이메일 발송 → 10명씩 batch·500ms delay·Resend 응답 정상

**검증 경로**:
1. APPROVED 상태가 있는 월 → "📧 전체 발송" 클릭 → confirm
2. 응답: `{ total, sent, failed, details }` 토스트
3. **테스트 모드**: 환경변수 `RESEND_TEST_RECIPIENT` 설정 시 모든 메일이 그 주소로 redirect (lib/email.ts 기본 동작·라이브 도메인 검증 전 안전장치)
4. 11명 이상일 때 batch 사이 500ms 지연 — 응답 시간으로 추정 가능

### Q5. payroll_send_history 적재 + status=SENT 갱신

**검증 경로** (DB·또는 일람 새로고침):
1. 발송 후 일람 새로고침 → 해당 회원 status=SENT 배지
2. SQL 직접 확인 가능 (admin이 SQL 가능 시):
   ```sql
   SELECT slip_id, sent_to, status, resend_id, error_message
   FROM payroll_send_history ORDER BY id DESC LIMIT 20;
   ```

### Q6. 직원 마이페이지에서 본인 명세서 다운로드 (status≥SENT만)

**검증 경로**:
1. 일반 운영자(operator) 또는 admin 로그인 → workspace-attendance.html 진입
2. "💰 급여명세서" 탭 클릭 → 자동 init → 본인 SENT 명세서 일람
3. PDF 버튼 클릭 → 본인 명세서 PDF 표시
4. **부정 경로**: URL 직접 `/api/payroll-my-pdf?id=<다른사람>` → 403 "본인 명세서만 다운로드할 수 있습니다"

### Q7. CSV export (회계 시스템용) 컬럼 정합

**검증 경로**:
1. URL 직접: `https://tbfa.co.kr/api/admin-payroll-export?year=2026&month=4`
2. 다운로드: `payroll-2026-04.csv` (UTF-8 BOM)
3. 22개 컬럼: 회원UID·이름·이메일·연도·월·근무일수·총근무분·야근분·지각·결근·유급휴가·무급휴가·만근·월기본급·야근수당·무급차감·성과보너스·만근보너스·세전총액·상태·승인일·발송일

### Q8. 권한 매트릭스 — 일반 회원·미인증 차단·본인 외 다운로드 차단

**검증 경로**:
| 호출자 | 호출 | 기대 응답 |
|---|---|---|
| 비로그인 | GET `/api/admin-payroll?year=&month=` | 401 "관리자 로그인이 필요합니다" |
| 일반 회원(regular) | GET `/api/admin-payroll?year=&month=` | 401 |
| admin (super_admin 아님) | GET `/api/admin-payroll?year=&month=` | 403 "슈퍼어드민 전용" |
| super_admin | GET `/api/admin-payroll?year=&month=` | 200 |
| 비로그인 | GET `/api/payroll-my` | 401 |
| 일반 회원 | GET `/api/payroll-my` | 403 "운영자 권한이 필요합니다" |
| operator A | GET `/api/payroll-my-pdf?id=B의slipId` | 403 "본인 명세서만 다운로드할 수 있습니다" |
| operator A | GET `/api/payroll-my-pdf?id=A의 SENT 아닌 slip` | 403 "발송 완료된 명세서만…" |

### Q9. 회귀 — quarterly_settlements PAID·att_records 정상 반영

**검증 경로**:
1. 재집계 후 상세 모달 → calculation_snapshot.quarter.totalBonusPaid 값이 해당 월이 속한 분기의 PAID 결산 합계와 일치
2. att_records 정상 반영 여부 확인 — workingMins/overtimeMins/late_count 등이 attRecords의 직전 달 행과 정합

### Q10. 회원 baseSalary=0 → 명세서 미생성 (스킵)

**검증 경로**:
1. 운영자 중 base_salary=0 인 회원 1명 → 재집계 후 일람에 노출 안 됨 (calculatePayrollForMonth가 SELECT 단계에서 COALESCE(base_salary,0) > 0 필터)
2. 후보 카운트와 명세서 생성 수가 일치 — 스킵된 회원 없음(SELECT가 곧 작업 대상)

## §4. 알려진 회귀 위험 (설계서 §10 + 본 라운드)

| 위험 | 처리 |
|---|---|
| baseSalary=0 직원 | SELECT 단계 스킵 |
| workingMins NULL | SUM FILTER WHERE IS NOT NULL → 0 |
| 만근 보너스 미정의 | perfect_bonus=0 고정 |
| 이메일 발송 실패 | status 보존(재시도 가능) + history FAILED 적재 |
| REVIEWED/APPROVED/SENT 재집계 보호 | force=false 시 스킵 + skipped 카운트 |
| PDF 폰트 로딩 실패 | 폰트 캐시 + try/catch (admin/my-pdf 양쪽 jsonError) |
| Resend rate limit (100건/분) | 10명 batch + 500ms delay → 최대 ~1200건/분 미만 |
| RESEND_TEST_RECIPIENT 설정 시 | 모든 메일이 그 주소로 redirect (lib/email.ts 동작) |
