### 🚨 BUG-R10-1 (Major) — R10 프론트 MOCK 폴백 6종 잔존

**발견 위치**:
- `public/js/incident.js` Lines 462-463, 652-672 (voteComment), 738-744 (submitReport)
- `public/js/workspace-files.js` Lines 645-654, 662-663, 716-717, 729-733 (loadShareList, addShare, removeShare)

**현상**: A가 B 머지 전 MOCK 폴백으로 구현한 코드가 B 머지·라운드 10 완료 후에도 그대로 남음. R8·R9 동일 패턴. 백엔드 4xx/5xx 응답을 가짜 성공으로 둔갑.

**구체 영향**:
1. **댓글 투표**: 백엔드 에러 시 클라이언트 측 토글 시뮬레이션 → 새로고침 전까지 가짜 카운트 표시
2. **댓글 신고**: 백엔드 500 에러도 "신고되었습니다" 토스트 (실제 미저장)
3. **파일 공유 목록**: 에러 시 가짜 공유자(`sharedWith=5, permission=view`) 한 명 표시 — 실제 DB와 다름
4. **파일 공유 추가/취소**: 가짜 성공 토스트 → 새로고침 시 미반영 발각
5. **알림 설정 GET**: 백엔드 에러 시 mock 기본값 5종 표시 (실제 사용자 설정과 무관)
6. **알림 설정 PUT (일괄 저장)**: 백엔드 에러도 "알림 설정이 저장되었습니다" 토스트

**예상 동작**: 4xx/5xx 시 toast로 서버 에러 노출, 가짜 성공·가짜 데이터 금지
**실제 동작**: 모든 MOCK이 작동 → 사용자 혼동 + 운영 검증 곤란

**예상 fix**:
- MOCK_* 상수 6종 제거
- voteComment / submitReport / addShare / removeShare / loadShareList / saveAll / init 각 catch / `if (!result)` 분기에서 toast로 에러 노출 후 return
- BUG-R8-1, BUG-R9-1과 동일한 클린업 절차
