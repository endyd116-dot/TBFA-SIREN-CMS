## 5. 발견 BUG 1건 (자체 fix 완료)

### BUG-021 (Low) — `admin-finance-report.ts` 죽은 API의 옛 테이블 참조 잔재

**위치**: `netlify/functions/admin-finance-report.ts` (`/api/admin-finance-report`)

설계서 §4 "옛 테이블 코드 정리"는 정리 대상으로 `db/schema.ts` 정의·옛 지출 API 3개·`lib/ai-*`만 명시. **`admin-finance-report.ts`는 누락** — 이 파일이 `expenditures` 테이블을 raw SQL(`FROM expenditures`)로 4곳 참조 (line 81·90·105·133).

**영향 분석**:
- 전체 repo grep 결과 `/api/admin-finance-report`는 **어디서도 호출되지 않음** (프론트·다른 함수 모두). 22-B-R1에서 재정 화면이 cms-tbfa.html로 이전되며 호출처가 사라진 죽은 API.
- `expenditures` 테이블 자체는 DB에 보존(설계서 §9 — DROP 금지)이라 raw SQL은 현재 런타임 동작. tsc 영향도 없음(raw SQL).
- **시한폭탄**: 22-D-R2 이후 `expenditures` 테이블이 실제 DROP되면 이 API는 500. 죽은 API라 즉시 영향은 0이나 정리 누락.

**fix**: `netlify/functions/admin-finance-report.ts` 파일 삭제 (죽은 API + 폐기 테이블 참조 — 설계서 §4 옛 테이블 정리 정신에 부합). tsc 14건 유지(삭제 영향 0) 확인.
