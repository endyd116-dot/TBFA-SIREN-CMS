## §3 BUG-1 (Critical) — drizzle `sql.raw(q, params)` 파라미터 바인딩 미지원 — 7개 함수

### 재현
```
curl -b cookie 'https://tbfa.co.kr/api/admin-milestone-settlement?quarterId=1'
→ HTTP 500 / {"step":"select","detail":"there is no parameter $1"}
```

### 근거
- drizzle-orm의 `sql.raw(rawString)`는 단일 문자열 인자만 받음 — **두 번째 `params` 배열 인자는 무시됨**
- 따라서 raw SQL의 `$1`·`$2` placeholder가 PostgreSQL에 그대로 전달되어 "there is no parameter $1" 발생
- `sql.raw(q, params)` 패턴이 7개 함수에서 동일 사용 중

### 영향 함수 (7개)
| 함수 | 발현 경로 | 라이브 확인 |
|---|---|---|
| `admin-milestone-settlement.ts:45` | `?status=`·`?quarterId=`·`?memberId=` | HTTP 500 |
| `milestone-settlement.ts:35` | 항상 ($1=member_id 강제) | HTTP 500 |
| `admin-milestone-revenue.ts:52` | `?status=`·`?quarterId=`·운영자(`!super_admin`) | HTTP 500 |
| `admin-milestone-nonrevenue.ts:43` | `?status=`·`?quarterId=` | HTTP 500 |
| `milestone-definitions.ts:44` | `?role=`·`?category=`·운영자 | HTTP 500 |
| `milestone-revenue.ts:38` | 항상 + 추가 BUG (BUG-2) | HTTP 500 |
| `milestone-nonrevenue.ts:37` | 항상 | HTTP 500 |

### 심각도
**Critical** — 성과관리 매출/비매출/결산/정의 4개 도메인의 핵심 GET API가 모두 동작 불능. 어드민·운영자·일반 사용자 모두 영향. 이전 검증에서 누락된 이유는 무필터 GET만 호출했기 때문 (P1 Q8에서 `/api/admin-milestone-settlement` 단순 GET은 200 반환했음).

### 도입 시점
P2-B (d113b98) 이전부터 존재했을 가능성 — 패턴이 6개 다른 함수에 이미 있었음. P2-B의 `?memberId=` 필터 추가가 이번 검증에서 처음 드러나게 한 트리거.

### Fix (본 검증 동봉)
- 7개 함수 모두 `sql.raw(q, params)` → drizzle `sql\`\`` 템플릿 합성 패턴으로 정정
- 각 동적 조건(`if (xxx) baseSql = sql\`${baseSql} AND ...\``)을 안전 바인딩으로 변경
- 변경 위치: 위 7개 파일의 GET 블록만 (POST/PATCH/DELETE 영역은 미변경 — 별도 패턴)

## §4 BUG-2 (High) — `milestone-revenue.ts` `md.amount_unit` 컬럼 부재

### 재현
```
curl -b cookie 'https://tbfa.co.kr/api/milestone-revenue?quarterId=1'
→ HTTP 500 / detail: "column md.amount_unit does not exist"
```

### 근거
- `milestone_definitions` 테이블 schema(`db/schema.ts:3438~3457`)에 `amount_unit` 컬럼 **부재** — `threshold_unit`만 존재
- `amount_unit`은 `revenue_entries` 테이블에만 존재 (`db/schema.ts:3466`)
- `milestone-revenue.ts:30`의 SELECT `md.amount_unit as default_unit` → 컬럼 미존재 에러
- `default_unit` alias는 코드·프론트 어디서도 사용되지 않음 (dead code)

### Fix (본 검증 동봉)
- `md.amount_unit as default_unit` 줄 제거 (사용처 0)
- BUG-1 fix와 함께 같은 파일에서 처리
