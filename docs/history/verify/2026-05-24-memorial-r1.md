## 핵심 진단

백엔드(테이블·공개/어드민 API·AI 초안)와 화면 마크업·렌더 로직은 **정상**. 망가진 건 **프론트가 한 번도 백엔드와 같이 실행되지 못한 경계면 2곳**:

1. **공개 페이지가 가짜 데이터로 그려짐** — 갤러리·상세 화면이 화면 개발 단계의 하드코딩 샘플 4건을 그대로 그리고 있어, DB에 발행된 실제 이야기와 어드민 변경(추가·발행·삭제·조회수)이 **공개 사이트에 전혀 반영되지 않음.** 시드로 채운 DB와 운영자 도구가 무의미해지는 상태.
2. **어드민 "정보 가져오기" 버튼이 항상 실패** — 운영자가 유튜브 URL을 넣고 제목·썸네일을 미리 받아오려 눌러도, 서버가 그 요청을 목록 조회로만 처리해 매번 "정보를 가져올 수 없습니다" 안내가 뜸. (저장 시점엔 서버가 알아서 제목·썸네일을 채우므로 결과물은 정상 — 미리보기 단계만 깨짐.)

둘 다 고정된 경계라 **3파일(공개 JS 2 + 어드민 API 1) 가산 수정**으로 해소.

## BUG 상세

### BUG-1 — 공개 갤러리·상세가 가짜 데이터로 그려짐 (HIGH)

- **증상**: `/family-stories.html`·`/family-story.html`이 화면 개발 단계의 하드코딩 샘플 4건을 그림. 실제 발행 API(`/api/family-stories`)를 **호출하지 않음**. 운영자가 CMS에서 영상을 추가/발행/삭제해도, 조회수가 올라가도 공개 사이트엔 영원히 같은 4건만 노출.
- **원인**: 화면 JS 두 파일에 `var USE_MOCK = true;`가 남아 있어 API 분기를 타지 못함. (백엔드가 늦게 합류해 한 번도 같이 실행되지 못한 전형적 경계면 — CLAUDE.md §6.14 안티패턴.)
- **확인**: 라이브 `GET /api/family-stories`는 이미 발행 4건을 정상 반환(ok:true). 즉 백엔드는 멀쩡하고 화면만 단절.
- **fix**: `public/js/family-stories.js`·`public/js/family-story.js`에서 가짜 데이터 상수와 `USE_MOCK` 토글·해당 분기를 **완전 제거** → 화면이 곧장 실제 API를 호출. `family-stories.html`·`family-story.html` 캐시버스터 `?v=1`→`?v=2`.
- **영향**: 마침 가짜 데이터 4건과 시드 발행 4건의 내용이 동일하므로 화면 결과물은 그대로 보이되, **이제부터 운영자 변경이 즉시 반영**됨.

### BUG-2 — 어드민 "정보 가져오기" 버튼이 항상 실패 (MEDIUM)

- **증상**: 운영자가 영상 추가/수정 폼에서 유튜브 URL 입력 후 "정보 가져오기"를 눌러도 매번 "유튜브 정보를 가져올 수 없습니다 — 수동 입력" 안내만 뜸.
- **원인**: 프론트는 `GET /api/admin-family-stories?ytOembed=<URL>`로 요청하고 `data.oembed`(제목·썸네일)를 기대하는데, 서버 GET 핸들러가 `ytOembed` 파라미터를 무시하고 항상 전체 목록만 반환 → 프론트가 `oembed`를 못 찾아 실패 처리.
- **부분 완화**: 저장(POST/PATCH) 시점엔 서버가 URL에서 oEmbed를 직접 조회해 제목·썸네일을 채우므로 최종 결과물은 정상. 깨진 건 저장 전 미리보기 단계뿐.
- **fix**: `netlify/functions/admin-family-stories.ts` GET 핸들러에 `ytOembed` 분기 추가 — youtubeId 추출 후 기존 `fetchOembed` 헬퍼로 제목·썸네일 조회, `{ ok:true, data:{ oembed:{ title, thumbnailUrl } } }` 반환(썸네일은 oEmbed 실패해도 `i.ytimg.com/.../hqdefault.jpg` 폴백). 목록 GET 로직·인증 게이트 영향 없음.
