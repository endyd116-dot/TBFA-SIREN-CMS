## 3. 교차 확인 — 전표 ↔ 예산안 연결

**결과: ✅ PASS**

전표 작성 모달의 "예산 항목" 드롭다운(`admin-expenses-voucher.js:145-157`):
1. `GET /api/admin-budget-plan-list` → `data.plans[]`
2. `status === 'approved'` 필터 → `fiscalYear` 내림차순 정렬 → 최신 1개
3. `GET /api/admin-budget-plan-detail?id={latest.id}` → `data.lines[]`
4. 드롭다운 렌더 — `categoryName (편성: ₩X)` 형식

응답 키 호환 검증:
- `budget-plan-list` 응답: `fiscalYear` (camelCase) — JS `b.fiscal_year || b.fiscalYear` 다중 fallback ✅
- `budget-plan-detail` 응답: `lines[]` 각 항목 `categoryName`·`plannedAmount` — JS `l.category_name || l.categoryName`, `l.planned_amount || l.plannedAmount` ✅
- budget_lines 없을 때: "예산 미연결" 빈 옵션 + "예산안 승인 후 항목이 표시됩니다" 안내 ✅

## 4. 발견 BUG 2건 (전부 자체 fix 완료)

### BUG-019 (Medium) — finance TOOL_GROUPS에 신규 도구·키워드 누락

**위치**: `netlify/functions/admin-ai-agent.ts:293-311` finance TOOL_GROUPS

설계서 §3·§7.1은 `lib/ai-agent-tools.ts`(도구 선언·핸들러)·`lib/ai-agent-config.ts`(매핑 표) 갱신만 명시. **`admin-ai-agent.ts`의 `selectRelevantTools` TOOL_GROUPS는 또 누락** — 22-B-R1 BUG-016과 동일 메타 패턴 (selectRelevantTools 갱신 누락 3회째: BUG-009·BUG-016·BUG-019).

- `tools` 배열: `budget_plan_list/create/approve` + `voucher_list/create/approve` + `account_codes_list` 7개 누락
- `keywords`: "전표"·"예산안"·"계정과목" 누락 ("예산"·"회계"는 있음)

**영향**:
- "예산안 만들어줘" → "예산" 키워드 매칭 → finance 그룹으로 좁혀짐 → 그러나 `budget_plan_create`가 tools 배열에 없음 → AI에 미전달 → **호출 불가** (Q11 R2 FAIL)
- "전표 보여줘" → "전표" 키워드 없음 → 매칭 0 → `null` → 전체 도구(안전망)로 동작. 단 "예산" 포함 메시지는 finance 그룹 매칭 → voucher_* 누락

**fix**: finance 그룹 `tools`에 7개 도구 추가 + `keywords`에 "예산안·편성·차년도·내년 예산·전년 실적·전표·계정과목·계정·세목·증빙·임차료·공과금·통신비" 추가.

### BUG-020 (Medium) — 전표 승인 버튼 action 누락

**위치**: `public/js/admin-expenses-voucher.js:580` `approveVoucher()`

```javascript
const res = await api('POST', '/api/admin-voucher-approve', { id });  // action 누락
```

`admin-voucher-approve.ts:34-38`은 `{ id, action }` 필수 (approve.ts가 action으로 approve/reject 분기). action 없으면 400 "id, action 필수" → 전표 승인 버튼 클릭 시 항상 실패.

(반려는 `admin-voucher-reject.ts` 별도 엔드포인트 — `{ id, rejectionReason }`만 받으므로 프론트 `doRejectVoucher` 정상)

**fix**: `approveVoucher`의 호출에 `action: 'approve'` 추가.
