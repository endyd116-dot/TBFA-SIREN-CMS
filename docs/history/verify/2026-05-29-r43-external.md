## 3. BUG / 관찰사항

### BUG-1 (P2 — 설계서 vs 코드 의도 불일치, 의도된 결정 가능성 큼)

**위치**: `lib/martyrdom-external.ts:184`
**증상**: 외부 검색이 `internalBulk: true` 옵션으로 호출 → `lib/ai-gemini.ts:513` `checkFeatureBeforeCall(featureKey, { skipSurge: true })`에서 5분 surge 카운터 면제. 결과: 운영자가 분당 5회+ 검색해도 surge cooldown 미발동.
**재현**:
1. 짧은 시간 내 `POST /api/admin-martyrdom-external-search` 5회+ 호출
2. martyrdom_ai 전체 surge 카운터에 합산 X
3. 월cap·featureKey OFF는 정상 동작 (이 부분은 fail-closed 살아있음)
**설계 의도**: 설계서 §2.2 "5분 surge 카운터에 `internalBulk:false`로 합산"이지만 실제 코드는 `internalBulk:true` (line 184 주석 "일괄 수집 — surge 면제 (월cap·토글은 그대로)").
**판정**: 트리거 체크박스 Q12의 "분당 5회+ surge cooldown 발동"이 코드 의도와 다름. 다만 **월cap·토글은 정상**이므로 절대적 비용 안전장치는 살아있음 — 결정자 판단.
