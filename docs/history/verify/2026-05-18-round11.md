### 🚨 BUG-R11-2 (Critical) — 신고 검토 PATCH 요청에 `status` 필드 누락

**발견 위치**: `public/js/admin-comment-reports.js:178-183`

**현상**:
```javascript
// 프론트 발송 (status 없음)
body: { reportId: parseInt(currentReportId, 10), action: action }

// 백엔드 검증 (admin-comment-report-review.ts:49, 55-56)
const status = String(body?.status || "").trim();  // → ""
if (!VALID_STATUS.has(status)) return 400 "유효하지 않은 status"
```
- 백엔드가 `status` 필드를 필수로 요구하지만 프론트에서 전혀 보내지 않음
- 모든 검토 요청이 400으로 실패 → `comment_reports.status` / `incident_comments.is_hidden` 갱신 안 됨
- BUG-R11-1 MOCK이 이 400 오류를 가려 UI는 성공처럼 보임

**연쇄 영향**:
- Q3 신고 처리 불가 (DB 미갱신)
- Q4 알림 미발송 (PATCH 400 실패 → dispatch 코드 미도달)
- 어드민이 "처리되었습니다" 토스트를 보고 완료로 오해

**예상 fix**: `submitReview()`에서 action→status 매핑 추가 후 PATCH body에 포함
```javascript
var status = action === 'none' ? 'dismissed' :
             action === 'hide_comment' ? 'reviewed' : 'resolved';
body: { reportId: parseInt(currentReportId, 10), action, status }
```

### 🚨 BUG-R11-3 (Major) — 리마인더 cron이 참조하는 `remindAt` 필드가 저장되지 않음

**발견 위치**:
- cron 조건: `cron-workspace-task-reminder.ts:34-35`
- PATCH 저장: `admin-workspace-task-reminder.ts:56`
- 프론트 전송: `workspace-kanban.js:2128-2132`

**현상**:
- cron 쿼리 조건: `reminder_config->>'remindAt' IS NOT NULL AND remindAt::timestamptz <= NOW()+2분`
- PATCH API: 프론트가 보낸 `reminderConfig` 객체를 그대로 JSONB 저장 (`remindAt` 계산 없음)
- 프론트 `saveReminder()`: `{ enabled, minutesBefore, channels }` 전달 (`remindAt` 없음)
- 결과: 모든 `workspace_tasks.reminder_config`에 `remindAt` 키가 존재하지 않음 → cron `IS NOT NULL` 조건 미충족 → 어떤 태스크도 선택 안 됨 → 알림 발송 불가

**예상 fix**: PATCH API에서 task.dueDate 조회 후 `remindAt = dueDate - minutesBefore * 60초`를 계산해 reminderConfig에 병합
```typescript
const remindAt = new Date(task.dueDate.getTime() - minutesBefore * 60 * 1000).toISOString();
const config = { ...reminderConfig, remindAt, firedAt: null };
await db.update(workspaceTasks).set({ reminderConfig: config, ... })
```
