## 4. 발견된 버그

### 🚨 BUG-R9-1 (Major) — 워크스페이스 R9 MOCK 폴백 5종 잔존

**발견 위치**: `public/js/workspace-kanban.js` Lines 1955-1970, 2026-2029, 2076-2088, 2118-2130, 2168-2176, 2204-2211

**현상**: A가 B 머지 전 mock 폴백으로 구현했으나, B 머지·schema 활성화 후에도 mock 상수와 try/catch 폴백이 그대로 남음. 라운드 8의 BUG-R8-1과 동일 패턴.

```js
// 예 — addSubtask()
try {
  res = await api('/api/admin-workspace-subtask-create', {...});
} catch (err) {
  if (String(err.message).includes('1단계')) { toast(...); return; }
  res = MOCK_SUBTASK_CREATE;     // ← 다른 모든 HTTP 에러를 가짜 성공으로 둔갑
}
```

본 파일의 `api()` 헬퍼(line 1996-2008)는 `res.ok=false` 시 throw하므로, 백엔드 4xx/5xx가 전부 catch로 흘러 mock 폴백 발동 → 사용자에게 거짓 성공 표시.

**영향 범위 5종**:
- `MOCK_SUBTASKS` (서브태스크 목록 — 가짜 2건 표시)
- `MOCK_SUBTASK_CREATE` (서브태스크 생성 — 가짜 id=103 반환)
- `MOCK_CHECKLIST` (체크리스트 저장 — 토스트 성공이지만 DB 미반영)
- `MOCK_REMINDER` (리마인더 저장 — 토스트 성공이지만 DB 미반영)
- `MOCK_RECURRING` (반복 작업 생성 — 가짜 id=202 반환, 실제 작업 미생성)

**예상 동작**: 4xx/5xx 에러 시 toast로 서버 에러 노출, 가짜 성공 금지
**실제 동작**: 가짜 성공 → 새로고침해야 사용자가 미반영 인지

**예상 fix**: BUG-R8-1과 동일 — MOCK_* 상수 5개 제거, `try { res = await api(...) } catch (err) { toast(err.message); return; }` 패턴으로 통합.

### 🚨 BUG-R9-2 (Major) — 채팅 검색 프론트 미구현

**발견 위치**: `public/js/chat-user.js` (검색 호출 0건)

**현상**: 백엔드 `chat-search` API는 완성·라우팅 정상이나, 사용자 채팅 UI에 검색창·검색 버튼·단축키 등 진입점이 전혀 없음.

`grep -rn "chat-search\|chatSearch" public/` → 0건.

**예상 동작**: 채팅방 상단 또는 본문에 검색 입력 + 결과 패널 노출
**실제 동작**: 사용자가 검색 기능에 접근할 수단 없음 — 백엔드 자원 사실상 사용 불가

**예상 fix**: 채팅 모달 헤더에 🔍 버튼 + 검색 패널 추가. 결과 클릭 시 해당 메시지로 스크롤. 별도 라운드 또는 R9 후속 패치 권장.

### ⚠ BUG-R9-3 (Minor) — 반복 작업 `[반복]` 접두사 누락

**발견 위치**: `netlify/functions/admin-workspace-task-recurring.ts:56` + `public/js/workspace-kanban.js:2204`

**현상**: 트리거 Q4 명세 "새 작업 제목에 `[반복]` 접두사 표시"가 미구현. 사용자가 직접 새 제목 칸에 적지 않는 한, 원본 제목 그대로 복제됨.

**예상 동작**: `title = "[반복] " + parent.title` 자동 prefix
**실제 동작**: `title = body.title || parent.title` — prefix 없음

**예상 fix**: 백엔드에서 `const title = (body?.title || \`[반복] \${parent.title}\`)` 한 줄 추가.
