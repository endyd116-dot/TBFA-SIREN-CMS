## §2. 헬퍼 파일 (메인 작업)

### lib/admin-role.ts (신규)

```typescript
// 역할 우선순위
const ROLE_RANK: Record<string, number> = {
  super_admin: 3,
  admin: 2,
  operator: 1,
};

export type AdminRole = "super_admin" | "admin" | "operator";

// 최소 역할 체크 — requireAdmin 통과 후 추가 호출
export function requireRole(
  member: { role?: string | null },
  minRole: AdminRole
): boolean {
  const rank = ROLE_RANK[member.role ?? ""] ?? 0;
  return rank >= ROLE_RANK[minRole];
}

// forbidden 응답 생성 헬퍼
export function roleForbidden(requiredRole: AdminRole): Response {
  return new Response(
    JSON.stringify({
      ok: false,
      error: `${requiredRole} 권한이 필요합니다`,
      step: "role_check",
    }),
    { status: 403, headers: { "Content-Type": "application/json" } }
  );
}
```

**사용 패턴:**
```typescript
// 기존 인라인 코드 교체
if (auth.ctx.member.role !== "super_admin") return forbidden(...)

// 신규 헬퍼 사용
if (!requireRole(auth.ctx.member, "super_admin")) return roleForbidden("super_admin")
```

## §3. API 수정 (B 작업)

### 3.1 [버그 수정] admin-login.ts — role 하드코딩 제거

```typescript
// AS-IS (버그: 모든 로그인을 super_admin으로 반환)
role: "super_admin"

// TO-BE (DB 실제 role 반환)
role: member.role ?? "operator"
```

**영향**: 로그인 후 JWT에 실제 role이 담겨 권한 구분 활성화

### 3.2 [수정] super_admin 체크 함수들 — requireRole 헬퍼로 교체

아래 파일의 `member.role !== "super_admin"` 패턴을 `requireRole + roleForbidden`으로 교체:
- `admin-audit.ts`
- `admin-budget-plan-approve.ts`
- `admin-expense-approve.ts`
- `admin-expense-category-create.ts`
- `admin-grades.ts`
- `admin-donation-policy.ts`
- `admin-ai-config.ts`

**기능 변경 없음** — 패턴 통일만.

### 3.3 [신규] admin 등급 필수 체크 추가

아래 기능에 `requireRole(member, "admin")` 체크 추가 (현재 운영자 누구나 접근 가능):
- `admin-members-blacklist.ts` — 블랙 처리/해제
- `admin-eligibility-force-change.ts` — 자격 강제 변경
- `admin-billing-key-reactivate.ts` — 빌링키 재활성화
- `admin-donate-refund.ts` — 환불

### 3.4 [수정] admin-operators.ts — role 선택에 'admin' 추가

```typescript
// AS-IS
const newRole = body.role === "super_admin" ? "super_admin" : "operator";

// TO-BE
const VALID_ROLES = ["super_admin", "admin", "operator"];
const newRole = VALID_ROLES.includes(body.role) ? body.role : "admin";
```
