## 2. A (프론트) 작업 분담

### 2-A-1. workspace-attendance.html
- **재택보고서** 탭 추가 (data-tab="report")
- 탭 패널: AI 초안 버튼 + 보고서 본문 편집 영역 + 제출 버튼
- 오늘 WBS 카드 목록 표시 섹션

### 2-A-2. workspace-attendance.js
- `loadReport()` - 오늘 보고서 조회 (`GET /api/att/remote-report?date=YYYY-MM-DD`)
- `renderReport()` - 보고서 상태별 렌더링 (DRAFT/SUBMITTED/없음)
- `generateAIDraft()` - Gemini 초안 생성 (`POST /api/att/ai-draft`)
- `submitReport()` - 보고서 제출 (`PUT /api/att/remote-report`)
- `saveReportDraft()` - 임시 저장 (`POST /api/att/remote-report`)
- 탭 전환 시 report 탭 → `loadReport()` 호출

### 2-A-3. admin-attendance-settings.html
- **재택보고서 모니터링** 탭 (data-tab="remotereports")
  - 직원 목록 (좌) + 선택 직원 보고서 목록 (우)
  - 필터: 기간·제출 상태
  - 별표 표시·댓글 버튼
- **근무형태 관리** 탭 (data-tab="workmodes")
  - 직원 목록 + 선택 직원 스케줄 조회·편집
  - 기본 근무형태 선택 (OFFICE/REMOTE/FIELD/HYBRID)
  - HYBRID: 요일별 설정

### 2-A-4. admin-attendance-settings.js
- `loadRemoteReports()` - 슈퍼어드민 보고서 목록 (`GET /api/admin/att/remote-reports`)
- `renderReportDetail()` - 보고서 상세 + 댓글 입력
- `starReport()` / `addComment()` - 별표·댓글 API 호출
- `loadWorkModes()` - 직원별 스케줄 조회 (`GET /api/admin/att/work-mode?memberUid=X`)
- `saveWorkMode()` - 스케줄 저장 (`POST /api/admin/att/work-mode`)

## 4. API 계약 (Request/Response)

### att-remote-report.ts
```
GET  /api/att/remote-report?date=YYYY-MM-DD
→ { ok, data: { id, date, content, aiDraft, qualityScore, status, submittedAt, wbsCardIds } | null }

POST /api/att/remote-report
body: { date, content?, wbsCardIds? }
→ { ok, data: { id } }  // DRAFT 저장

PUT  /api/att/remote-report
body: { date, content }  // SUBMITTED 제출
→ { ok, message: "제출 완료" }
```

### att-ai-draft.ts
```
POST /api/att/ai-draft
body: { date }  // 오늘 날짜
→ { ok, draft: "AI 생성 500자 이내 보고서 초안" }
// 내부에서 오늘 WBS 카드(할당된 카드들) + 이전 3일 보고서 수집 → Gemini 호출
```

### att-ai-insight.ts
```
POST /api/att/ai-insight
body: { memberUid, type: "remote"|"all", startDate, endDate }
→ { ok, data: { summary, categories: [{name, pct}], qualityAvg } }
```

### admin-att-remote-reports.ts
```
GET /api/admin/att/remote-reports?memberUid=X&startDate=Y&endDate=Z&status=SUBMITTED
→ { ok, data: [{ id, date, memberName, content, qualityScore, isStarred, submittedAt }] }

PUT /api/admin/att/remote-reports
body: { id, supervisorNote?, isStarred? }
→ { ok }
```

### admin-att-work-mode.ts
```
GET  /api/admin/att/work-mode?memberUid=X
→ { ok, data: { schedules: [...], overrides: [...] } }

POST /api/admin/att/work-mode
body: { memberUid, workMode, recurringRule?, startDate, endDate?, workplaceId? }
→ { ok, data: { id } }

DELETE /api/admin/att/work-mode
body: { id, type: "schedule"|"override" }
→ { ok }
```

### att-export.ts
```
GET /api/att/export?memberUid=X&year=2026&month=5&format=csv
→ CSV 파일 (Content-Disposition: attachment)
// 컬럼: 날짜, 근무형태, 출근시각, 퇴근시각, 근무시간, 상태, 지각여부
```

## 5. Gemini 프롬프트 전략

### 재택보고서 초안 (att-ai-draft.ts)
```
시스템: "당신은 재택근무 일일 보고서 초안 작성 도우미입니다. 500자 이내 한국어로 작성하세요."
사용자: "오늘({date}) 작업한 카드 목록:\n{카드목록}\n이전 보고서 맥락:\n{이전보고서}\n4개 섹션으로 보고서 초안을 작성해주세요: 오늘 한 일, 진행률, 내일 계획, 이슈"
```

### AI 흐름파악 (att-ai-insight.ts)
```
시스템: "NPO 직원 업무 흐름을 분석하는 HR 어시스턴트입니다."
사용자: "{기간} 재택근무 보고서 및 WBS 데이터를 분석해 작업 카테고리 분류, 진행 패턴, 이상 신호, 종합 평가(500자)를 JSON으로 반환해주세요."
응답 형식: { categories: [{name, pct}], pattern: string, anomalies: string[], summary: string }
```

## 6. Cron 로직 요약

### cron-att-morning.ts (09:30 KST)
- 오늘 출근 예정인 직원(스케줄 있음) 중 출근 기록 없는 직원 조회
- 해당 직원에게 인앱 알림 발송 ("출근 체크가 완료되지 않았습니다")
- 슈퍼어드민에게 요약 알림

### cron-att-evening.ts (자정 KST)
- 퇴근 기록 없는 직원(출근은 했으나 퇴근 null) → 퇴근 기록 자동 생성(EARLY_LEAVE)
- 재택근무자 중 보고서 미제출 → 알림
- 주간 누적 근무 48시간 도달 직원 → 임박 알림
- 주간 누적 근무 52시간 초과 직원 → 슈퍼어드민 즉시 알림

### cron-att-ai-daily.ts (18:00 KST)
- 전 직원 오늘 근태 요약 데이터 수집
- Gemini로 "오늘 전직원 근태 요약 + 이상 신호" 생성
- 슈퍼어드민에게 인앱 알림 발송

### cron-att-leave-auto.ts (매월 1일)
- 전월 만근 직원(결근 0건) 조회 → 유급휴가 +1일 부여
- 입사 1년 도래 직원 → 연차 15일 일괄 부여
- 연차 소진 D-30 직원 → 사용 촉진 알림
