## 2. 데이터 모델

### 기존 활용 (추가 테이블 불필요)
- `donations` — 후원 수입 (토스·효성·계좌이체·일시)
- `billing_logs` — 정기 결제 로그
- `members` — 후원자 연결

### 신규 테이블 (Phase 6에서 마이그레이션)

#### `budget_categories` — 사업 분류 마스터
```sql
CREATE TABLE budget_categories (
  id          SERIAL PRIMARY KEY,
  name        VARCHAR(50) NOT NULL,          -- '심리상담', '법률지원', '장학사업', '운영비', '홍보'
  code        VARCHAR(20) UNIQUE NOT NULL,   -- 'psych', 'legal', 'scholar', 'ops', 'pr'
  description TEXT,
  is_active   BOOLEAN DEFAULT TRUE,
  created_at  TIMESTAMP DEFAULT NOW()
);
```

#### `budgets` — 연간 예산 계획
```sql
CREATE TABLE budgets (
  id              SERIAL PRIMARY KEY,
  fiscal_year     INTEGER NOT NULL,           -- 2025, 2026 등
  category_id     INTEGER REFERENCES budget_categories(id),
  planned_amount  NUMERIC(12,0) NOT NULL,     -- 계획 금액
  note            TEXT,
  created_by      INTEGER REFERENCES admins(id),
  created_at      TIMESTAMP DEFAULT NOW(),
  UNIQUE(fiscal_year, category_id)
);
```

#### `expenditures` — 지출 기록
```sql
CREATE TABLE expenditures (
  id            SERIAL PRIMARY KEY,
  category_id   INTEGER REFERENCES budget_categories(id),
  amount        NUMERIC(12,0) NOT NULL,
  spent_at      DATE NOT NULL,                -- 집행일
  description   VARCHAR(500) NOT NULL,        -- 지출 내용
  payee         VARCHAR(100),                 -- 지급처
  status        VARCHAR(20) DEFAULT 'draft',  -- 'draft' | 'approved' | 'rejected'
  receipt_url   TEXT,                         -- R2 영수증 첨부
  created_by    INTEGER REFERENCES admins(id),
  approved_by   INTEGER REFERENCES admins(id),
  approved_at   TIMESTAMP,
  note          TEXT,
  created_at    TIMESTAMP DEFAULT NOW()
);
```

## 4. Phase 6 — 지출 기록 + 예산 배분 (4~6h)

### 4.1 마이그레이션

`migrate-finance-tables.ts` — `budget_categories` + `budgets` + `expenditures` 3개 테이블 생성 + 초기 카테고리 5개 INSERT (`심리상담`, `법률지원`, `장학사업`, `운영비`, `홍보`).

### 4.2 신규 페이지·화면

**`public/admin-finance-budget.html`** + **`public/js/admin-finance-budget.js`**

```
[예산 관리]
┌─ 연도 선택 ──────────────────────────────────┐
│                                               │
│  [사업별 예산 테이블]                           │
│  사업명   | 계획(원) | 집행(원) | 잔액 | 집행률  │
│  심리상담  | 5,000만  | 3,200만 | 1,800만 | 64% │
│  법률지원  | 2,000만  | 1,100만 |   900만 | 55% │
│  장학사업  | 3,000만  | 2,800만 |   200만 | 93% │
│  운영비    | 1,500만  |   900만 |   600만 | 60% │
│  [예산 편성 버튼] (새 연도 계획 입력)           │
│                                               │
│  [최근 지출 목록 — 최신 20건]                   │
│  날짜 | 사업 | 내용 | 금액 | 상태 | 영수증       │
└───────────────────────────────────────────────┘
```

**지출 입력 모달**
```
날짜 / 사업 분류 / 금액 / 지급처 / 내용 / 영수증 첨부(R2)
[기안 저장] → status='draft' → 어드민 승인 대기
```

**승인 흐름**
- 기안자(어드민) → 저장 → status='draft'
- 승인자(슈퍼어드민 또는 어드민) → 승인 → status='approved' / 반려 → status='rejected'
- 승인 시 이메일 알림 (Resend)

### 4.3 신규 API

| 엔드포인트 | 메서드 | 설명 |
|---|---|---|
| `/api/admin-finance-budget-list` | GET | 연도별 예산 + 집행 집계 |
| `/api/admin-finance-budget-upsert` | POST | 예산 편성/수정 |
| `/api/admin-finance-expenditure-list` | GET | 지출 목록 (필터: status·category·날짜) |
| `/api/admin-finance-expenditure-create` | POST | 지출 기안 생성 |
| `/api/admin-finance-expenditure-approve` | PATCH | 승인/반려 |
