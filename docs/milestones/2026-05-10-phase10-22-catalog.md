# Phase 10~22 — 잔여 마일스톤 카탈로그 (스펙 합의 완료)

> **합의 시점**: 2026-05-10 / 메인 채팅 (설계 세션)
> **목적**: Phase 8·9 이후 잔여 13묶음 카탈로그를 9개 작업으로 통합 + Phase 10~20에 분배
> **상태**: ✅ 카탈로그·분배 합의 / ⏸ 개별 설계서는 각 Phase 도달 시점에 작성

---

## 1. 합의 결정 사항

| 결정 | 내용 |
|---|---|
| 13개 후보 → 9개 작업 통합 | 중복·인접 후보를 한 묶음으로 합쳐 작업량 절감 |

## 2. Phase 10~20 작업 분배

### Phase 10 — 통합 발송 시스템 (정기·일회성 + AI 트리거)

**무엇**: 후원자·회원·운영자에게 정기 뉴스레터·일회성 공지·AI 감지 트리거 알림을 한 시스템에서 발송.

**단계**:
1. 발송 템플릿 빌더 (디자인 + 변수 치환)
2. 수신자 그룹 선택 (전체·정기 후원자·일시·등급별·태그·캠페인 대상)
3. 발송 예약 큐 + 즉시 발송
4. 열람률·클릭률 추적
5. AI 트리거 자동 발송 (이탈 위험 ↑ 회원에게 재참여 메시지·캠페인 부진 시 운영자 알림 등)

**기존 활용**: 이메일 발송 인프라(템플릿 23종) + 회원 그룹 필터 + 이탈 위험 점수·캠페인 부진 자동 감지(이미 매일 운영).

**의존**: Phase 8 알림 채널 통합 인프라 위에서 작동.

---

### Phase 11 — 멘션·구독 (게시판·채팅)

**무엇**: 게시판·채팅에서 @멘션 시 알림, 게시글 구독 시 새 댓글 알림, 채팅 안 읽음 푸시.

**단계**:
1. @멘션 텍스트 파싱 (게시글·댓글·채팅 메시지)
2. 게시글·게시판 구독 관리 UI
3. 새 댓글·답변·멘션 발생 시 구독자에 알림 발송
4. 채팅 안 읽음 메시지 누적 시 일정 시간 후 푸시

**기존 활용**: 게시판·댓글·채팅 본체 모두 운영 중. 멘션·구독 로직만 신규.

**의존**: Phase 8 알림 인프라.

---

### Phase 12 — 신고 진행 상황 공개 + 익명 신고 강화

**무엇**: 사용자가 본인 신고가 어느 단계(접수·검토·진행·종결)에 있는지 마이페이지에서 확인 + 어드민이 익명도에 따라 신원을 단계적으로 식별.

**단계**:
1. 사용자 마이페이지 — 본인 신고 목록 + 단계 타임라인 시각화
2. 단계 변경 시 사용자에 자동 안내 (Phase 10 발송 시스템 활용)
3. 어드민 화면 — 익명 단계별 신원 식별 (0단계 익명 / 1단계 기본 정보 / 2단계 모든 정보)
4. 익명 보호 감사 로그 (어느 어드민이 어느 단계로 식별했는지)

**기존 활용**: 신고 단계 필드·익명 플래그·신원 분리 저장 모두 운영 중. UI만 추가.

---

### Phase 13 — 신고 통계 대시보드

**무엇**: 어드민이 월별 사건 유형·해결률·평균 처리일·심각도 분포를 시각화.

**단계**:
1. 월별·분기별·연간 집계
2. 사건 유형(괴롭힘·법률·일반 사건) 분포 차트
3. 처리 단계별 적체 현황
4. AI 분석 결과(심각도·요약) 통계
5. 정기 보고서 자동 생성 옵션 (Phase 4 대표 보고서 패턴 재사용)

**기존 활용**: 신고 데이터·기간 필터·AI 분석 결과 모두 있음. 집계 + 차트 화면만 추가.

---

### Phase 14 — 외부 기관 인계 (신규)

**무엇**: 경찰·교육청·법률 기관에 신고 사건을 표준 양식으로 인계 + 인계 이력·결과 추적.

**단계**:
1. 외부 기관 마스터 (이름·연락처·관할 영역)
2. 기관별 표준 양식 템플릿 (PDF 자동 채움)
3. 인계 흐름 (어드민이 기관·양식 선택 → PDF 생성 → 발송 기록)
4. 인계 후 결과 추적 (회신·진행 상태 갱신)

**기존 인프라 거의 없음** — 신규 데이터·UI 필요.

---

### Phase 15 — 전문가 매칭 고도화

**무엇**: 변호사·심리상담사 1:1 매칭에 분야·언어·가용 시간·과거 매칭 만족도 기반 자동 추천 점수.

**단계**:
1. 전문가 프로필 확장 (전문 분야·언어·요일·시간대)
2. 사용자 요청 시 적합도 점수 계산 (분야 일치 + 언어 + 가용 + 과거 만족도)
3. 어드민 화면에 추천 순위 표시 + 수동 배정 가능
4. 매칭 만족도 피드백 수집 → 다음 추천에 반영

**기존 활용**: 1:1 매칭 본체 + 자동 추천 함수 운영 중. 점수 계산·필터링만 고도화.

---

### Phase 16 — 통합 분석 대시보드 (이탈·라이프사이클·KPI·이사회 보고)

**무엇**: 협회 운영자가 후원·회원·서비스 흐름을 한 화면에서 보고 의사결정 + 이사회·총회 자동 보고서.

**단계**:
1. 이탈 위험 회원 목록 + 우선순위 큐 + 선제 대응(재참여 메시지) 버튼
2. 회원 라이프사이클 코호트 분석 (신규 → 첫 후원 → 정기 → 이탈 단계별 전환율·평균 기간)
3. 경영 KPI 위젯 (월간 수입·신규 회원·SIREN 처리·서비스 만족도 등)
4. 이사회·총회용 분기·연간 자동 보고서 — Phase 4 대표 보고서 인계 기능 보강 (분기·연간 단위 자동 발송)

**기존 활용**: 이탈 위험 점수 매일 자동 계산·캠페인 부진 감지·후원/재정/신고/회원 개별 대시보드·Phase 4 대표 보고서 모두 운영 중. 통합 화면 + 인계 기능만 추가.

---

### Phase 17 — 보안·감사 강화

**무엇**: 관리자 행동·세션·민감 정보 추적·보호 강화.

**단계**:
1. 감사 로그 강화 (세션 ID·접속 IP·관리자 행동 상세)
2. 민감 정보(결제 키·개인정보) 자동 마스킹 필터
3. 세션 만료 정책 + 관리자 자동 로그아웃
4. 권한 변경·중요 작업(승인·환불·삭제) 별도 알림

**기존 활용**: 감사 로그 본체 + 기본 추적 있음. 강화·확장만.

---

### Phase 18 — 성능 최적화

**무엇**: 큰 테이블(후원·로그·메시지) 조회 속도 ↑ + 시스템 응답 시간 ↓.

**단계**:
1. 느린 부분 식별 (운영 로그 분석)
2. 인덱스 추가·복합 조회 최적화
3. 자주 쓰는 결과 캐시
4. 페이지네이션·지연 로딩 보강

**기존 활용**: 큰 테이블 인덱스 일부 있음. 식별 + 추가만.

---

### Phase 19 — 자동 테스트 보강

**무엇**: 핵심 비즈니스 흐름(후원·매칭·결제·신고)에 자동 단위·통합 테스트 추가.

**단계**:
1. 테스트 도구·구조 도입
2. 핵심 흐름 우선 (후원 결제·매칭 알고리즘·인증·권한)
3. 회귀 방지 — 신규 기능 PR 시 테스트 통과 의무
4. CI 자동 실행

**기존 인프라 거의 없음** — 테스트 0%. 큰 묶음.

---

### Phase 20 — 운영 안정성 (모니터링 + 백업·복구)

**무엇**: 운영 중 발생하는 에러를 실시간 추적·통계 + 정기 백업·복구 절차 검증.

**단계**:
1. 외부 에러 모니터링 도구 선정·연동
2. 알림 규칙 (Critical 에러 → 어드민 즉시 알림)
3. 백업 정책 정리 (DB·파일함)
4. 정기 복구 시뮬레이션 + 재해 복구 시간 측정

**기존 인프라**: DB는 호스팅 자동 백업, 그 외 없음. 신규 도입 위주.

---

### Phase 21·22 — 여유 슬롯 (보류)

미래 기능 합의 시 채움. 후보:
- 글로벌화(다국어) — 영문 안내·해외 후원자 대응
- 모바일 앱 — 현재 웹 반응형만, 네이티브 앱 검토
- 블록체인 기부 영수증 — 투명성 강화 (실험적)
- 협회 간 연동 — 다른 NPO·재단과 데이터 공유

Phase 19~20 진행 중 또는 완료 후 합의.

## 3. Phase 4 보강 (ANL-D 흡수)

Phase 4 대표 보고서가 분기·연간 자동 생성 + PDF 출력을 이미 운영 중. ANL-D 이사회 보고서는 별도 묶음이 아니라 Phase 4의 인계 기능 보강으로 처리:

- 분기·연간 단위 자동 발송 옵션 (이사회 멤버 그룹 지정 → Phase 10 발송 시스템 경유)
- 이사회·총회용 템플릿 추가 (현재 일반 보고서 템플릿 + 회의 안건 형식)
- 결재·열람 기록 추적

Phase 16 통합 분석 대시보드 작업 시 같이 처리.

## 4. 일정 윤곽 (추정)

```
Phase 8·9     →  현재 진행 (알림 인프라 + 외부 API)
Phase 10·11   →  발송 시스템 + 멘션·구독         (Phase 8·9 완료 직후)
Phase 12·13   →  신고 상태 공개·통계             (사용자 가치 큰 묶음)
Phase 14·15   →  외부 기관 인계·매칭 고도화       (SIREN 마무리)
Phase 16      →  통합 분석 대시보드 (큰 묶음)
Phase 17·18   →  보안·성능
Phase 19      →  자동 테스트 (큰 묶음, 단독)
Phase 20      →  운영 안정성
Phase 21·22   →  여유 — 미래 기능 합의용
```

각 Phase 시작 시점에 별도 설계서 작성. 본 카탈로그는 큰 그림 합의용.
