## 3. 정직한 평가

다이어트는 **비약적이지 않음**. 1M 중 1.3% 절약. 한 작업 끝날 때 도구 호출로 더 많이 차오름.

체감 부담의 진짜 원인은 **누적**과 **중복 정독**. 다이어트보다 효과 큰 3가지 행동 변화:

1. **세션 분할 명확화** — "한 작업(머지·검증·설계 중 하나) 끝나면 새 채팅". 1M 다 안 쓰고 50% 시점에 새 채팅.
2. **Subagent 활용** — 정독·조사는 Explore agent에 위임. 메인은 결과 요약만 받음.
3. **Read 정책** — 큰 문서는 limit/offset 발췌, 같은 파일 두 번 Read 금지.

→ **시나리오 B + 위 3가지 행동 변화를 CLAUDE.md 단일 섹션으로 신설**

### 행동 변화 정책 (CLAUDE.md §14에 영구 등록)
- §14.1 정독 정책: PROJECT_STATE만 정독, HANDOFF는 발췌, archive 자발적 안 읽음

### 정직한 평가 (재확인)

다이어트 자체 효과는 **약 11K 토큰 = 1M의 1.1%**. 한 작업 끝날 때 도구 호출로 그만큼 차오르므로 절대값은 작음.

진짜 효과는 **§14 행동 변화**. 이번 세션처럼 PROJECT_STATE 587줄을 정독하지 않고 87줄만 정독하면 그 자체로 ~7K 토큰 절감 + 메인 컨텍스트에 원본 텍스트 누적 안 됨. 누적이 안 되면 자동 압축(/compact) 도달까지 시간이 더 길어짐.

**다이어트 + 행동 변화 결합 시** 중간 규모 작업 1개를 더 처리할 여유가 생김 (한 세션에서 머지 2건 + 검증 안내 + 다음 단계 설계까지 가능 추정).
