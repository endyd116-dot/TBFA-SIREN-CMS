# BUG-AI-AGENT-PHASE1-05 — 인자 자동 추출 부정확 (현재 날짜·자동 부풀림)

**발견일**: 2026-05-14 (V4 라운드)
**검증자**: C 채팅 (curl)
**심각도**: Medium (도구 호출은 됨, 인자 값만 부정확)

표준 §17.2 "인자 자동 추출 정확도 ~70%" 한계의 라이브 검증 사례 3건.

## 사례 1 — 현재 날짜 모름 (E1)

**입력**: `이번 주 일정 보여줘` (시스템 현재일 2026-05-14)
**호출**: `events_list(fromDate=2023-11-20, toDate=2023-11-26)`
**문제**: AI가 2023년 11월로 인식 → 시스템 프롬프트에 현재 날짜 주입 안 됨

## 사례 2 — 미래 날짜 부정확 (E2)

**입력**: `내일 오후 3시에 박두용 미팅 1시간 잡아줘`
**호출**: `event_create(startAt=2024-05-17T15:00, endAt=2024-05-17T16:00)`
**문제**: "내일"을 2024-05-17로 인식 (실제 2026-05-15여야 정상). 시간(15:00·16:00)은 정확.

## 사례 3 — 단일 명령 자동 부풀림 (T1)

**입력**: `Phase 1 검증용 더미 작업 만들어줘`
**호출**: AI가 task_create를 **3번** 호출:
1. "Phase 1 기능 검증 - 회원 검색 및 상세 조회" (priority: high)
2. "Phase 1 기능 검증 - 후원 통계 및 내역 확인" (priority: medium)
3. "Phase 1 기능 검증 - 알림 발송 및 템플릿 확인" (priority: low)

**문제**: 단일 작업 1개 요청을 우선순위별 3개로 자동 부풀림. 사용자 의도 왜곡.

## 메인 채팅 fix 권장

1. **시스템 프롬프트에 현재 날짜·시간 주입** (예: `${new Date().toISOString()}` 동적 삽입). cron 갱신 또는 호출 시점 동적.
2. **도구 description에 "사용자 명시 인자만 사용, 자동 부풀림 금지" 명시**. 또는 LLM 출력에서 동일 도구 N개 중복 호출 시 첫 1개만 채택.
3. 도구 인자 description에 매핑 힌트: `startAt: "사용자가 명시한 정확한 ISO 시각, 추정 금지"`
