# C. SIREN 신고 메뉴얼 (사건·괴롭힘·법률 상담)

> **대상**: 피해자·유족·내부고발자·일반 시민 (익명 가능)
> **핵심 원칙**: **익명 보호 우선** — 신고자 신원은 별도 권한 + 감사 로그 없이는 운영자도 볼 수 없습니다.

---

## C-0. 신고 3종 구분

SIREN은 성격이 다른 3종 신고를 별도 트랙으로 운영합니다.

| 종류 | 무엇 | 진입 | 예시 |
|---|---|---|---|
| **사건 신고** | 학교·교실 내 사건, 학대, 자살·사망 사건 등 | `/incidents.html` 또는 메인 [신고] | 학생 사망 사건, 학교 폭력 |
| **악성민원·괴롭힘** | 학부모·관리자 등으로부터의 괴롭힘 | `/report-harassment.html` | 학부모 악성 민원, 동료 따돌림 |
| **법률 상담** | 법적 분쟁·소송·자문이 필요한 사안 | `/legal-support.html` 안내 → 신청 | 명예훼손 대응, 손해배상 |

세 종류 모두 **본인 동의 시에만 공개**되고, 기본은 비공개·익명입니다.

---

## C-1. 사건 신고 (Incident)

### 신청 흐름
1. 메인 또는 `/incidents.html` → [신고하기]
2. **익명 / 실명** 선택 (익명 선택 시 본인 식별 정보 저장 안 됨, 단 연락 회신은 불가)
3. 사건 분류 선택 (학생/교사/학부모 관련 등)
4. 발생일·발생 장소·간략한 경위 입력
5. **증빙 파일** 업로드 (R2 Pre-signed URL, 최대 100MB)
6. 연락 가능 채널(이메일/휴대폰) — 익명 신고면 생략
7. [제출] → **AI 자동 분석** 즉시 실행 (분류 정합성·시급도 점수)

### 제출 후
- 회원은 `/my-reports.html` 또는 마이페이지 > 내 신고 탭에서 진행 상태 확인
- 운영자가 검토 후 다음 중 처리:
  - **공식 접수(확정)** → 사이렌이 정식 사건으로 등록, 필요 시 공개 사건 목록(`/incidents.html`)에 익명화 노출
  - **보완 요청** → 추가 정보 요청 메시지 발송
  - **반려** → 사유 회신

### 상태값
| 상태 | 의미 |
|---|---|
| 접수 대기 | 운영자 검토 전 |
| 검토 중 | 운영자 배정·심사 진행 |
| 보완 요청 | 추가 정보 요청 — 회원이 답변 필요 |
| 확정 | 공식 사건으로 접수 |
| 종결 | 처리 완료 |
| 반려 | 접수 불가 — 사유 표시 |

---

## C-2. 악성민원·괴롭힘 신고 (Harassment)

### 신청 흐름 (`/report-harassment.html`)
1. 익명/실명 선택
2. **가해자 유형**: 학부모 / 관리자 / 동료 / 학생 / 기타
3. **반복 여부**: 단발 / 반복(횟수)
4. **피해 유형**: 언어폭력·문자/통화·SNS·대면·서면 등 (복수 선택)
5. 피해 기간·내용 서술
6. 증빙(녹취·문자 캡처·이메일) 업로드
7. **희망 지원**: 심리 상담 / 법률 자문 / 학교 행정 개입 / 단순 기록
8. 제출 → AI가 피해 강도·법률 검토 필요성 분석

### 후속 처리
- 운영자가 분류·우선순위 부여
- **법률 검토 필요** 판정 → 자동으로 법률 상담 트랙으로 연결 제안
- **유사 사건 매칭**: 운영자가 AI 도구(`admin-ai-similar-cases`)로 과거 유사 사건 참조

---

## C-3. 법률 상담 (Legal Consultation)

### 신청 흐름
1. `/legal-support.html`에서 지원 범위·전문 변호사 풀 확인
2. [법률 상담 신청] → 폼 진입
3. 분쟁 유형(민사·형사·행정·노동), 사실관계, 희망 결과 입력
4. 관련 문서 업로드
5. 제출 → **AI 전문가 매칭**(`admin-ai-expert-recommend`)이 변호사 풀에서 후보 추천
6. 운영자가 확정 → 변호사·법률 지원금 매칭

### 진행 단계
| 단계 | 내용 |
|---|---|
| 신청 | 회원 제출 완료 |
| 검토 | 운영자가 사안 분류 |
| 매칭 | 변호사 후보 추천 |
| 진행 | 변호사 배정 + 상담 진행 |
| 종결 | 합의·승소·반려 결과 기록 |

---

## C-4. 유족 지원 (별도 트랙, 신고와 구분)

`/support.html` 진입 → 지원금·심리상담·장학금 등 신청.
- 서류 업로드/다운로드, 보충 서류 요청·답변 가능
- 어드민 > 유족 지원 심사에서 운영자가 심사
- 자세한 내용은 어드민 메뉴얼 2차에서 다룸

---

## C-5. 익명 보호 (핵심 정책)

### 회원 관점
- 익명 신고 시 신원 정보는 DB에 저장하지 않거나 분리 저장
- 운영자도 기본 화면에서 신원을 볼 수 없음 (마스킹)
- 신원 공개가 필요한 사안은 **회원 동의 후**에만 공개

### 운영자 관점
- 신원 식별은 별도 페이지 `/admin-anon-reveal.html`에서만 가능
- 식별 시 **반드시 사유 입력** → 감사 로그(`/admin-anon-audit.html`)에 영구 기록
- 부적절한 식별은 사후 감사 대상

### 시스템 보장
- 익명 ID 토큰으로 신고-회원 연결 (직접 회원 ID 노출 X)
- 첨부파일은 별도 권한 + 만료 URL
- 모든 신원 조회 = 감사 로그 기록 (`audit_logs` 테이블)

---

## C-6. 운영자 처리 화면 (참고)

신고 3종은 어드민에서 각각 별도 페이지로 관리되지만, 공통 흐름:
1. **목록**: 상태·날짜·우선순위·AI 점수로 필터
2. **상세**: 신고 내용·첨부·이력·AI 분석 결과·유사 사건
3. **AI 답변 초안**: [AI 회신 초안 생성] → 운영자가 검토·편집 후 발송
4. **상태 변경**: 검토 중 → 보완 요청 / 확정 / 반려
5. **메시지 발송**: 회원에게 진행 안내 (회원이 익명이면 발송 불가, 시스템 메모로만)

상세 어드민 절차는 2차 메뉴얼(E-SIREN 운영)에서 다룸.

## C-7. 자주 묻는 질문

| 질문 | 답 |
|---|---|
| 신고하면 학교/회사에 알려지나? | 운영자가 회원 동의 없이 외부에 전달하지 않음. 법적 의무 보고(아동학대 등)는 예외. |
| 익명으로 했는데 연락받을 수 있나? | 익명은 회신 불가. 연락 원하면 실명/연락처 입력 필요. |
| 신고 내용을 다른 사람이 볼 수 있나? | 기본 비공개. 공개 사건 목록은 운영자가 익명화 후 게시. |
| 증빙 파일이 너무 큰데? | 단일 100MB까지. 더 크면 분할 업로드. |
| 신고를 취소하고 싶다 | `/my-reports.html`에서 [취소] — 단, 운영자 검토가 시작되면 제한될 수 있음. |
| AI가 내 신고를 분석한다는데 외부 전송되나? | Google Gemini API로 전송됨. 개인 식별 정보는 사전 제거 후 전송. 동의 시에만 분석. |
| 법률 상담 비용은? | 협의회 지원금 범위 내 무료. 범위 초과 시 사전 안내. |
