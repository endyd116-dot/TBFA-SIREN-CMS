## E-결제-1. 후원 결제 수단 운영

### 1) KICC·이지페이 (카드 — 일시·정기 빌키)
- **테스트 / 라이브 전환**: 환경변수 `KICC_MODE=test|live`
- 일시 결제 → 거래 등록(`donate-kicc-*`) → KICC 결제창 → 승인(returnUrl 복귀)
- 정기: 카드 등록창 → 빌키 발급(`billing-register`/`billing-approve`) → 매월 `cron-kicc-billing`이 자동 청구
- 실패 시 재시도 + 회원·운영자 알림
- 운영자 작업: 실패 회원 목록 확인 → 회원에게 안내 메시지 발송

### 2) 효성 CMS+ (계좌 자동이체)
- 회원이 계좌·생년월일·인감 등록 → 효성 본인인증
- 매월 지정일 출금 (효성 파일 송수신 — `hyosung-parser.ts`)
- 운영자가 효성 결과 파일 업로드 → 시스템이 회원별 매칭·상태 갱신
- 출금 실패는 회원에게 자동 안내

### 3) 계좌이체 (무통장)
- 회원이 입금 → 운영자가 회원 상세에서 [입금 확인] → 영수증 발급
- 자동 매칭은 없음 (수기 작업)

## E-결제-2. 후원 처리 운영 흐름

### A. 일시 후원 운영
1. 발생: 회원/비회원이 `/donate.html`에서 결제
2. 시스템: 결제 성공 시 자동 영수증 발급 + 회원 누적 후원금 갱신
3. 운영자 액션:
   - **이상 결제 확인**: 대시보드에서 거액·반복 결제 확인
   - **환불 요청 처리**: 회원 문의 시 KICC(이지페이) 콘솔에서 수동 환불 → 시스템에 메모

### B. 정기 후원 운영
1. **신규 등록 모니터링**: 매일 신규 빌링 등록 수 확인
2. **결제 실패 처리**: 실패 회원 목록 → 사유별(잔액·카드정지·만료)
3. **카드 만료 알림**: `cron-billing-card-expiry`가 만료 2개월 전 자동 알림. 응답 없는 회원에게 추가 안내
4. **해지 처리**: 회원 직접 해지 시 즉시 반영. 운영자가 강제 해지(블랙·환불 등)도 가능

### C. 후원자 보상·소통
- **기념일 알림**: `cron-anniversary-check`가 기일·후원 시작일 등 체크 → 자동 메시지 트리거
- **이탈 예측**: `cron-churn-predictor`가 후원 패턴 이상(2회 연속 실패·6개월 결제 없음) 감지 → 운영자 회복 캠페인 제안
- **계약서 생성**: 회원 목록 > 선택 > [계약서 일괄 생성](`admin-members-contract-export`) → 협의회 양식 PDF 일괄 다운로드

## E-결제-3. 영수증·기부금 영수증

### 일반 영수증
- 결제 직후 회원 이메일로 PDF 자동 발송 (`pdf-lib` + NotoSansKR)
- 회원 마이페이지에서 상시 다운로드
- 운영자가 회원 상세에서 재발송 가능

### 연말 기부금 영수증
- 매년 1~2월 일괄 발급 (수동 트리거)
- 주민번호 미등록 회원 → 사전 입력 캠페인 발송
- 발급 후 회원 마이페이지에 비치 + 이메일 일괄 발송

## E-결제-4. 발송 (후원 운영 관련)

후원 운영자가 자주 쓰는 발송:
- **결제 실패 안내**
- **카드 만료 알림**
- **연말 기부금 영수증 안내**
- **신규 캠페인 후원 요청**
- **기념일 메시지**

### 발송 작업 만들기
1. `/admin-send-job-create.html`
2. **수신자 그룹** 선택 (`/admin-recipient-groups.html`에서 사전 정의 — 등급·후원 상태·기간 필터)
3. **템플릿** 선택 (`/admin-templates.html`) 또는 새로 작성
4. 발송 시각 예약 또는 즉시
5. `[발송 시작]` → `cron-communication-send-dispatcher`가 배치 처리
6. **결과 분석**: `/admin-send-analytics.html` — 발송·열람·클릭·반송
7. **개별 회원 발송 이력**: `/admin-member-send-history.html`

### 자동 트리거 (AI 자동 발송)
- `/admin-auto-triggers.html`에서 조건 정의:
  - "정기 후원 등록 7일 후 환영 메시지"
  - "최근 3개월 후원 없으면 회복 캠페인"
- AI가 메시지 톤·내용을 자동 생성 (운영자 검수 가능)
- `cron-auto-trigger-evaluator`가 매일 평가

## E-결제-5. 통계·KPI

### 대시보드 (`admin-stats`)
- 일별·월별·연도별 후원금
- 결제수단별 비중
- 신규/재후원/이탈
- 캠페인별 모금 현황

### 가입 채널 KPI
- `/admin.html` 회원 목록 > [채널별 KPI](`admin-members-source-kpi`)
- URL 파라미터(`?src=캠페인A`)로 유입된 회원의 후원 전환·LTV 분석
- 마케팅 채널 ROI 판단 근거

## E-결제-6. 회원 관련 후원 작업 (간략)

### 회원 상세 모달에서
- **후원 내역 탭**: 전체 후원 + 영수증 재발송
- **정기 후원 탭**: 빌링 상태 + 강제 해지
- **포인트 탭**: 수동 적립·차감 (사유 입력 필수)
- **블랙 처리**: 후원·신고·게시판 등 접근 차단
- **메모**: 운영자 간 공유 메모

### 회원 검색·필터
- 이메일·이름·휴대폰·등급·후원 누적·가입 채널
- Excel 일괄 다운로드(`admin-members-export`)

## E-결제-7. 비상·예외 처리

| 상황 | 대응 |
|---|---|
| KICC 결제 다운 | KICC(이지페이) 콘솔에서 상태 확인, 회원에게 점검 안내 발송 |
| 효성 출금 파일 오류 | 효성 측 응답 파일 재요청, 매칭 실패 회원 수기 처리 |
| 영수증 PDF 미수신 회원 | 회원 상세 > [영수증 재발송] |
| 환불 요청 | KICC 콘솔에서 환불 + 시스템 메모 + 후원 내역 상태 변경 |
| 카드 정보 유출 의심 | 즉시 빌링키 폐기 + 회원에게 재등록 안내 |
| 대량 결제 실패 발생 | KICC/효성 상태 확인 후, 발송 자동화 일시 정지 |

## E-결제-8. 자주 묻는 질문 (운영자 관점)

| 질문 | 답 |
|---|---|
| 회원이 영수증을 못 받았다고 한다 | 회원 상세 > [영수증 재발송]. 이메일 오타·스팸함 확인 권유. |
| 정기 후원이 두 번 결제됐다 | 결제 로그·KICC 콘솔 확인 → 중복 결제면 KICC에서 환불 + 시스템 메모. |
| 환불은 시스템에서 되나? | 자동 환불 기능 없음. KICC/효성 콘솔에서 수동 + 시스템 메모. |
| 카드 만료된 회원 한 번에 알리려면? | 수신자 그룹 [카드 만료 임박] → 발송 작업 생성. 또는 자동 트리거가 이미 처리 중일 수 있음. |
| 후원자 명단을 엑셀로 받고 싶다 | 회원 목록 필터 적용 → [Excel 내보내기](`admin-members-export`). |
| 연말 기부금 영수증 발급 절차는? | 1~2월 별도 일괄 발급 트리거 + 안내 발송. 주민번호 미등록자 사전 캠페인. |
| 신규 후원 채널 성과를 보고 싶다 | 회원 목록 > 채널별 KPI. URL `?src=채널명`으로 유입되어야 분류됨. |
