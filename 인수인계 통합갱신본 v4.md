## 💬 7. 채팅 시스템 (★ Phase H+I 강화)

### 아키텍처

\```
사용자 마이페이지 → "1:1 상담" 탭 → 카테고리 선택 → 채팅방 생성
    ↓
텍스트/이미지 메시지 송수신 (5초 폴링)
    ↓ (★ H-1: 이미지가 실제로 표시됨, 라이트박스+다운로드)
관리자 백오피스 → "채팅 관리" 메뉴 → 좌우 분할 레이아웃
    ↓
회원 정보 + 메모 + 답변 + 종료/보관/복구/블랙
\```

### 주요 사양

* **폴링**: 5초 간격
* **카테고리**: 후원/홈페이지/가입/기타
* **메시지 보존**: 영구
* **이미지 보존**: 1년 (★ I-2: 자동정리, active방은 보존)
* **이미지 압축**: 클라이언트 Canvas API (max 1200px, JPEG 75%)
* **이미지 표시**: ★ H-1 — 인라인 `<img>` + 라이트박스 + 다운로드
* **상태 전환**: active ↔ closed ↔ archived (★ I-3 채팅 아카이브)
* **블랙리스트**: 관리자 등록/해제 + ★ I-3 회원 모달에 표시
* **채팅 메모**: 관리자 메모 → ★ I-3 회원 모달 목록 표시
* **알림**: 헤더 뱃지 (30초 폴링) + 사이드바 뱃지

## 📄 8. PDF 영수증 시스템 (★ Phase H-2 완전체)

### 핵심 흐름

\```
1. 회원이 후원 → donate.ts → DB 저장 + 감사 메일 (★ H-3)
2. 마이페이지 → 후원 내역 → "📄 발급" 버튼
   ↓
3. /api/donation-receipt?id=N (본인 또는 관리자)
   ↓
4. issueReceiptNumber() — 영수증번호 발급 (TBFA-2026-000042)
   ↓
5. generateReceiptPDF() — DB(receipt_settings) 우선 → 환경변수 → 샘플 폴백
   ↓
6. pdf-lib + NotoSansKR 폰트 임베딩 (subset: false, 한글 안정)
   ↓
7. A4 PDF 바이너리 응답 (인라인 또는 다운로드)
\```

### 영수증 설정 관리 (★ H-2d 양방향 동기화)

\```
[admin.html "📄 영수증 설정" 메뉴]  ←→  [cms-tbfa.html "📄 영수증 설정"]
              ↓                              ↓
          GET/PATCH /api/admin/receipt-settings (단일 진실 원천)
                          ↓
                  receipt_settings 테이블 (id=1)
                          ↓
                  PDF 생성 시 자동 반영
\```

### 한글 깨짐 핫픽스 (중요)

\```ts
// lib/pdf-receipt.ts
let _fontCache: ArrayBuffer | null = null;

function loadKoreanFont(): Uint8Array {
  if (!_fontCache) {
    const buf = readFileSync(fontPath);
    _fontCache = buf.buffer.slice(buf.byteOffset, buf.byteOffset + buf.byteLength);
  }
  return new Uint8Array(_fontCache.slice(0));  // 매번 새 Uint8Array
}

const font = await pdfDoc.embedFont(fontBytes, { subset: false });  // ← 핵심
\```

* `subset: true`는 큰 한글 폰트(6MB)에서 CMAP 누락 → 한글 깨짐
* 폰트 cache는 ArrayBuffer로 보관 + 매번 새 Uint8Array (pdf-lib mutate 방어)
* PDF 크기 ~6MB로 증가하지만 한글 100% 안정
