## 🔵 Block 1: 인증 + 회원 (3 케이스)

### TC-1.1 사용자 가입/로그인 (기본 동작)

**시나리오**:

1. `/index.html` → "회원가입" 클릭
2. 일반 후원 회원으로 가입 (이메일/이름/비번/연락처)
3. 회원가입 모달의 약관 링크 2개 클릭

**합격 기준**:

* [ ] 약관 링크 → privacy.html / terms.html 새 탭에서 열림
* [ ] 가입 완료 후 자동 로그인 → 헤더에 이름 표시
* [ ] 마이페이지 접근 가능

### TC-1.2 관리자 로그인

**시나리오**:

1. `/admin.html` 접속
2. `admin / admin1234`로 로그인
3. 사이드바 메뉴 9개 확인 (대시보드/회원/기부/지원/운영자/채팅/AI/콘텐츠/**영수증설정**/시스템)

**합격 기준**:

* [ ] **📄 영수증 설정** 메뉴 표시
* [ ] **💬 채팅 관리** 메뉴 표시
* [ ] 각 메뉴 클릭 시 해당 페이지 정상 로드

### TC-1.3 교유협 CMS 진입

**시나리오**:

1. 관리자 로그인 → admin-hub → 교유협 CMS 클릭
2. 사이드바에 **📄 영수증 설정** 메뉴 확인

**합격 기준**:

* [ ] 좌측 사이드바 8개 메뉴 (대시보드/통합회원/외부등록/이관/태그/알림/추출/**영수증설정**)

## 💝 Block 2: 후원 + PDF 영수증 (8 케이스 — 핵심)

### TC-2.1 회원 후원 + 감사 메일 (★ H-3)

**시나리오**:

1. 사용자 로그인 → 메인 → "🎗 후원 동참" 클릭
2. 정기 후원 30,000원 / 신용카드 / **본인 이메일 입력**
3. 후원 모달의 "개인정보 수집" 링크 클릭 → privacy.html 열림 확인
4. 본인 인증 후 결제

**합격 기준**:

* [ ] "후원이 완료되었습니다" 메시지 + D-0000XXX 번호 표시
* [ ] **본인 이메일 수신함**에 감사 메일 도착 (1\~2분 내)
* [ ] 메일 제목: `[SIREN] ○○○님, 따뜻한 후원에 감사드립니다 🎗`
* [ ] 메일에 후원 정보 표 + **"📄 영수증 발급하러 가기" 버튼**
* [ ] 후원금 사용 안내 4개 항목 표시

### TC-2.2 비회원 후원 + 비회원용 메일

**시나리오**:

1. **로그아웃 상태**에서 후원 버튼 클릭
2. 비회원으로 후원 + 이메일 입력

**합격 기준**:

* [ ] 메일이 정상 도착
* [ ] 영수증 영역이 **"회원가입 후 발급"** 안내로 표시 (회원과 다름)
* [ ] "회원가입 →" 버튼 표시

### TC-2.3 PDF 영수증 발급 (★ H-2c)

**시나리오**:

1. TC-2.1 메일의 "영수증 발급하러 가기" 클릭 → 마이페이지 이동
2. "내 후원 내역" 탭 → 완료된 후원 행의 **📄 발급** 클릭

**합격 기준**:

* [ ] 새 탭에서 PDF 인라인 표시
* [ ] **한글이 깨지지 않고 정상 출력** (제목/협회정보/금액 등 모두)
* [ ] 영수증번호 형식 확인 (`TBFA-2026-000XXX`)
* [ ] 한글 금액 변환 (예: "₩30,000 (금 삼만 원정)")
* [ ] 직인 표시 (붉은 원형)

### TC-2.4 PDF 다운로드 (한글 파일명)

**시나리오**:

1. PDF 미리보기 화면에서 브라우저 다운로드 버튼

**합격 기준**:

* [ ] 파일명이 **`기부금영수증_TBFA-2026-000XXX.pdf`** 형식 (한글 깨짐 없음)

### TC-2.5 영수증 설정 — 사이렌 백오피스 (★ H-2d-3)

**시나리오**:

1. 관리자 → 📄 영수증 설정 메뉴
2. 폼이 자동으로 DB 값으로 채워지는지 확인
3. 단체명을 "테스트단체A"로 변경 → 💾 저장
4. 👀 PDF 미리보기 클릭

**합격 기준**:

* [ ] "영수증 설정이 저장되었습니다" 토스트
* [ ] 마지막 수정자/시간 갱신
* [ ] 미리보기 PDF에 **"테스트단체A"** 반영

### TC-2.6 영수증 설정 — 양방향 동기화 (★ H-2d-4) — **가장 중요**

**시나리오**:

1. 사이렌에서 단체명 "동기화테스트1"로 변경 → 저장
2. 새 탭에서 교유협 CMS → 📄 영수증 설정 메뉴
3. **"동기화테스트1"이 그대로 보이는지** 확인 ⭐
4. 교유협에서 단체명 "동기화테스트2"로 변경 → 저장
5. 사이렌 탭으로 돌아가서 🔄 다시 불러오기

**합격 기준**:

* [ ] 양방향 동기화 100% 동작
* [ ] 어느 쪽에서 변경하든 다른 쪽에서 즉시 보임 (refresh 필요)

### TC-2.7 하단 안내문 동적 추가/삭제

**시나리오**:

1. 영수증 설정 → "하단 안내문" 영역
2. **+ 안내문 추가** 클릭 → 빈 input 추가됨
3. 텍스트 입력 → 💾 저장
4. ✕ 버튼 클릭 → 행 삭제됨

**합격 기준**:

* [ ] 최대 10개까지 추가 가능 (11번째 시도 시 토스트 알림)
* [ ] 모두 삭제하면 빈 input 1개 자동 보장
* [ ] 미리보기 PDF에 추가/삭제된 안내문이 반영됨

### TC-2.8 영수증 PDF 연속 발급 (한글 깨짐 핫픽스 검증)

**시나리오**:

1. 동일한 후원 ID로 **PDF 미리보기 4회 연속** 발급 (탭 닫고 → 다시 열기 반복)
2. 마이페이지에서 다른 후원 ID도 발급

**합격 기준**:

* [ ] **모든 PDF에서 한글이 100% 정상** ⭐ (subset:false + ArrayBuffer 캐싱 효과)
* [ ] 첫 번째만 정상이고 두 번째부터 깨지면 → 핫픽스 실패 → 즉시 보고

## 🤝 Block 3: 지원 신청 + 메일 (6 케이스)

### TC-3.1 일반 지원 신청 + 일반 메일 (★ H-4)

**시나리오**:

1. 사용자 로그인 → 메인 → "🤝 지원 신청"
2. 심리상담 / 제목: "상담 받고 싶어요" / 내용: "도움이 필요합니다"
3. 지원 모달의 약관 링크 → privacy.html 열림

**합격 기준**:

* [ ] 신청 완료 + 본인 이메일에 **접수 확인 메일**
* [ ] 메일에 **🔵 파란 안내 박스** ("영업일 1\~3일 이내 답변")
* [ ] **채팅 안내 없음** (긴급이 아니므로)
* [ ] 4단계 처리 절차 안내 표시

### TC-3.2 긴급 지원 신청 + 긴급 메일

**시나리오**:

1. 동일 흐름이지만 내용에 **긴급 키워드** 포함 (예: "지금 자살 충동이 있어요. 긴급히 도와주세요")
2. AI가 `urgent`로 분류해야 함

**합격 기준**:

* [ ] 메일 제목에 **"🔴 긴급 -"** 접두사
* [ ] 메일 본문에 **🔴 빨간 경고 박스**
* [ ] **⚡ 1:1 채팅 상담 시작 버튼** 표시
* [ ] 관리자 페이지 → 지원 관리 → **🔴 긴급 라벨** 표시

### TC-3.3 관리자 답변 → 신청자 메일

**시나리오**:

1. 관리자 → 지원 관리 → TC-3.1 신청 행의 "📝 상세/답변" 클릭
2. 답변 작성 + **"체크박스: 메일 발송"** 체크
3. 저장

**합격 기준**:

* [ ] 신청자 이메일에 답변 알림 메일 도착
* [ ] 메일 본문에 **답변 내용은 노출 안 됨** (보안 — "마이페이지 로그인 후 열람" 안내만)

### TC-3.4 AI 4가지 기능 동작

**시나리오**:

1. 관리자 → 지원 상세 모달 열기

**합격 기준**:

* [ ] AI 우선순위 라벨 표시 (🔴/🟡/🟢)
* [ ] **"✍️ AI 답변 초안 생성"** 버튼 표시 → 클릭 시 동작
* [ ] **🤝 AI 추천 전문가** 자동 로드 (3-5초)
* [ ] **🔍 AI 유사 사례** 자동 로드

### TC-3.5 운영자 다중 메일 (긴급)

**시나리오**:

1. 운영자 관리 → 회원 1명을 운영자로 승급 + 알림수신 ON
2. TC-3.2 긴급 신청 발생

**합격 기준**:

* [ ] **모든 활성 운영자**에게 메일 발송 (notifyOnSupport와 무관)
* [ ] 일반 신청은 notifyOnSupport=true 운영자만

### TC-3.6 회원 정보 팝업 (★ I-3 통합)

**시나리오**:

1. 관리자 → 회원 관리 또는 지원 관리 → 회원 이름 클릭
2. 회원 정보 모달 열림

**합격 기준**:

* [ ] 가입일, 후원 누계, 지원 신청 카운트 표시
* [ ] **블랙리스트면 빨간 박스 + 🔓 블랙 해지 버튼** (★ I-3)
* [ ] **채팅 관리자 메모 목록** 표시 (있는 경우)

## 💬 Block 4: 채팅 시스템 (9 케이스)

### TC-4.1 사용자 채팅 시작

**시나리오**:

1. 사용자 마이페이지 → "💬 1:1 상담" 탭
2. "+ 새 상담 시작" → 카테고리 선택 → 채팅창 열림
3. 텍스트 메시지 전송

**합격 기준**:

* [ ] 메시지 즉시 표시 (낙관적 UI)
* [ ] 5초 이내 관리자측 채팅에도 표시

### TC-4.2 채팅 이미지 업로드 + 표시 (★ H-1)

**시나리오**:

1. 사용자 채팅창에서 📷 버튼 클릭 → 이미지 선택
2. 클라이언트 압축 + 업로드 + 메시지 전송

**합격 기준**:

* [ ] 이미지가 **인라인 썸네일로 표시** (240x240px max) ⭐
* [ ] `[이미지]` 텍스트만 보이지 않음
* [ ] 이미지 클릭 → **풀스크린 라이트박스** 열림
* [ ] 라이트박스에 **💾 다운로드** + **✕ 닫기** 버튼
* [ ] ESC 키 + 배경 클릭으로 닫힘

### TC-4.3 다운로드 한글 파일명 보존

**시나리오**:

1. 라이트박스의 💾 클릭

**합격 기준**:

* [ ] 원본 한글 파일명 그대로 다운로드됨

### TC-4.4 관리자측 채팅 (5초 반복 버그 픽스 검증 ★ I-1)

**시나리오**:

1. 관리자 → 채팅 관리 → 좌측 목록에서 활성 채팅방 선택
2. **30초 이상 가만히** 두기 (폴링 6회 발생)

**합격 기준**:

* [ ] **마지막 메시지가 반복 표시되지 않아야 함** ⭐
* [ ] 새 메시지만 추가됨

### TC-4.5 관리자 채팅 텍스트 정렬 (★ I-1b)

**시나리오**:

1. 관리자가 긴 텍스트 메시지 전송 (예: "안녕하세요. 문의 주신 내용을 확인했습니다.")

**합격 기준**:

* [ ] 메시지가 **가로로 정상 표시** ⭐
* [ ] 한 글자씩 세로로 보이지 않아야 함
* [ ] 시간 표시 정상

### TC-4.6 채팅 아카이브 (★ I-3 — 이 대화)

**시나리오**:

1. 활성 채팅방의 🔒 종료 클릭 → 종료됨
2. 사이드바 필터 \*\*"종료"\*\*로 변경 → 종료된 방 선택
3. 헤더에 **🗄 보관 / 🔓 재개** 버튼 표시 확인
4. 🗄 보관 클릭 → 보관함 이동
5. 사이드바 필터 \*\*"보관"\*\*으로 변경 → 보관된 방 선택
6. 🔓 복구 클릭 → 종료 상태로 복귀

**합격 기준**:

* [ ] active → closed → archived → closed 전환 자유
* [ ] 보관함에 있는 동안은 **opacity 0.7** + 메시지 입력창 비활성화
* [ ] KPI 카운트 정상 갱신

### TC-4.7 블랙리스트 등록 + 양방향 (★ I-3 — 다른 대화)

**시나리오**:

1. 관리자 → 채팅 관리 → 활성 방 선택 → ⛔ 블랙 클릭 → 사유 입력
2. 채팅창의 헤더에 블랙 정보 표시 + 🔓 블랙 해지 버튼 보임
3. 회원 관리 페이지 → 그 회원 이름 클릭
4. 회원 모달에 **빨간 블랙 박스 + 🔓 해지 버튼** 표시
5. 모달에서 🔓 해지 → 채팅창에도 즉시 반영

**합격 기준**:

* [ ] 블랙 등록 시 해당 회원의 active 채팅방 자동 closed
* [ ] 회원 모달과 채팅창 양쪽에서 **블랙 정보 표시 + 해지 가능**
* [ ] 한쪽에서 해지하면 다른 쪽도 갱신

### TC-4.8 미읽음 카운트

**시나리오**:

1. 사용자가 메시지 보냄
2. 관리자가 그 방을 **읽지 않은 채로 다른 방 선택**
3. 사이드바의 **🔴 빨간 뱃지** + 카운트 확인

**합격 기준**:

* [ ] 안 읽은 메시지 수가 뱃지로 표시
* [ ] 방 선택 후 자동 읽음 처리 (PATCH 호출)
