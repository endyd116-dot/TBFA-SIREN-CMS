## 🗂 8. R2 저장소 시스템 (★ M-2.5에서 도입)

### 왜 도입했나

* Netlify Functions(AWS Lambda)는 **요청 본문 6MB 하드 리밋**
* 이미지 20MB / 파일 100MB 지원 위해 R2 직접 업로드로 우회

### 흐름

```
[클라이언트 업로드 — Pre-signed URL]
1. POST /api/blob-presign → {id, uploadUrl}
2. PUT uploadUrl (R2 직접) — 6MB 한도 무관
3. POST /api/blob-confirm → upload_status='completed'

[서버 직접 업로드 — M-14에서 추가]
lib/r2-server.ts의 uploadToR2() / downloadFromR2()
- PDF 영수증 (M-14)
- 직인 이미지 (M-14)
- 향후 M-19 활동보고서 PDF 등
```

### 저장 정책

* 이미지: 클라이언트 자동 압축 (1600px, JPEG 85%)
* 파일 한도: 이미지 20MB / 일반 100MB
* blob\_uploads 테이블에 `storage_provider='r2'` 기록
* 미참조 파일 7일 후 자동 정리 (cron 예정)
* **영수증 PDF**: 만료 없음 (M-14, 영구 보관 + 캐시 활용)

* 타이밍 공격 방어: `DUMMY_BCRYPT_HASH` (lib/auth.ts)
* Enumeration 방지 (비번 재설정에서 동일 응답)
* HttpOnly 쿠키만 (응답 본문에 token 없음)
* 토스 webhook HMAC + crypto.timingSafeEqual
* SameSite=Lax CSRF 방어
* escapeHtml 전수 적용
* 401 자동 처리 (common.js fetch monkey-patch)
* 모든 중요 액션 audit\_logs 자동 기록
