✅ 확정 공식 사건으로 등록 상태 → "확정" · 공개 목록 게시(회원 동의 시)
⛔ 반려 접수 불가 · 사유 입력 필수 상태 → "반려" · 회원에게 사유 알림

AI 회신 초안 사용 팁

  • 초안은 그대로 보내지 말고 반드시 운영자가 검토·편집 후 발송
  • AI가 사실관계를 추측한 부분(예: "○○씨께서는...")이 있으면 삭제
  • 익명 신고에는 회신 발송이 불가능하므로 초안은 운영자 메모용으로 활용
  • AI 회신은 별도 비용으로 청구되니 AI 비용 관리에서 월 한도 확인

유사 사건 매칭 자세히

운영자가 [🔍 유사 사건 보기]를 누르면 AI가 같은 분류·유사 키워드의 과거 사건을 자동 검색합니다.

  • 점수 기준: 분류 일치도 + 키워드 유사도 + 발생 시기 가중
  • 표시 정보: 사건 번호·요약·처리 결과(확정/반려)·처리 기간
  • 활용 방법: 비슷한 사건이 반려됐다면 사유 참고. 확정 사건의 회신 톤도 참조

유족 지원 운영 (별도 트랙)

유족 지원금·심리상담·장학금은 신고 트랙과 구분되며 /admin-hub.html의 유족 지원 심사 화면에서 처리합니다.

  • 서류 검토 · 보충 서류 요청 · 지급 결정
  • 지급 결정 시 회원 알림 + 회계 시스템 연동
  • 익명 보호 정책은 신고와 동일하게 적용

🔐 익명 신원 식별 페이지 🛡️ 슈퍼어드민

익명 신고자의 신원을 봐야 할 정당한 사유(법적 의무 보고, 회원 본인 요청 등)가 있을 때만 사용. 진입 /admin-anon-reveal.html

⚠️ 절대 금지 행위
호기심 · 개인적 친분 · 비공식 요청으로 익명 신원을 식별하는 것. 시스템은 막지 않지만 행위가 감사 로그에 영구 기록되고, 최고관리자가 정기 점검합니다. 부적절한 식별은 사후 추궁됩니다.

정당한 식별 사유 (예시)

  • 법적 의무 보고가 필요한 사안(아동학대 등) — 외부 기관 통보 전 신원 확인
  • 회원 본인이 "내가 신고한 건이 맞는지 확인해 달라"고 요청한 경우
  • 이중·중복 신고 여부 확인 (같은 사안을 여러 번 신고)
  • 법원·수사기관의 정식 영장·요청

📜 감사 로그 페이지 🛡️ 슈퍼어드민

모든 익명 신원 식별 행위가 영구 기록되는 곳. 진입 /admin-anon-audit.html

로그에 기록되는 정보

📌 식별 정보

  • 식별 일시 (분 단위까지)
  • 식별한 운영자 ID·이름
  • 대상 신고 번호
  • 입력한 식별 사유 전문

🔍 점검 도구

  • 기간·운영자·신고 번호 필터
  • 의심스러운 식별 패턴 알림 (단시간 다건 등)
  • CSV 내보내기
  • 자동 삭제 없음 (영구 보관)

정기 감사 운영

  • 최고관리자가 매월 1회 감사 로그 점검
  • 이상 패턴 발견 시 해당 운영자에게 사유 재확인
  • 중대 위반 발생 시 운영자 권한 회수 + 감사 위원회 회부
💡 운영자도 자기 행위 이력을 볼 수 있나?
본인이 한 식별 이력은 본인이 조회할 수 있습니다. 다만 로그 자체는 수정·삭제 불가이며, 본인이 자기 이력을 봐도 그 조회 또한 별도 로그에 기록됩니다.

E-1. 결제 수단 3종 운영

회원이 쓰는 결제는 토스(카드)·효성(계좌)·계좌이체(무통장) 3가지. 운영자가 신경 써야 할 포인트가 각기 다릅니다.

💳 토스페이먼츠

카드 결제 · 일시 + 정기

  • 설정: 환경변수 TOSS_MODE=test|live
  • 일시: 즉시 결제 + 즉시 영수증
  • 정기: 빌링키 발급 → 매월 자동 청구
  • 실패 처리: 자동 재시도 + 회원 안내
  • 운영자 액션: 토스 콘솔에서 수동 환불 + 시스템 메모

🏦 효성 CMS+

계좌 자동이체 · 일시 + 정기

  • 등록: 회원이 계좌·생년월일·인감 등록
  • 본인인증: 효성 측 확인 절차
  • 출금: 매월 지정일 효성 파일 송수신
  • 운영자 액션: 효성 결과 파일 업로드 → 회원별 매칭
  • 실패 회원: 자동 안내 발송 + 운영자 확인

📮 계좌이체

무통장 입금 · 일시만

  • 흐름: 회원이 직접 협의회 계좌로 입금

E-9. AI 운영 · 비용 관리 🛡️

AI 비서·자동 트리거·회신 초안 등 모든 AI 호출은 비용을 발생시킵니다. 운영자가 월 한도와 도구 권한을 통제합니다.

💰 AI 비용 관리 /admin-ai-cost.html

  • 기능별 월 한도 설정 (자동 트리거·AI 회신·작업 요약 등)
  • 한도 초과 시 자동 차단 (추가 과금 0)
  • 일·월 누적 비용 그래프
  • 가장 비용이 큰 기능 Top 5

🛡️ AI 비서 설정 /admin-ai-config.html

  • 시스템 프롬프트 편집 (슈퍼어드민)
  • 84개 도구별 활성/비활성 토글
  • 도구별 필요 권한 등급 설정 (위험 도구는 슈퍼어드민만)
  • 60초 캐시 후 자동 반영

비용 절감 팁

  • 자주 호출되는 도구는 저가 모델(Gemini Flash)로 우선 매핑
  • 자동 트리거 빈도 조정 (예: 매일 → 매주)
  • AI 회신 초안은 운영자 검수 시점에만 호출 (자동 생성 OFF)
  • 월 한도를 보수적으로 설정 → 초과 차단 발생 시 한도 재검토

AI 호출 로그

/admin-ai-logs-list.html에서 모든 AI 호출(누가·어떤 도구·어떤 결과)을 시간순 조회. 비서 대화 단위로 보고 싶으면 /admin-ai-conversations-list.html.

E-10. 비상 · 예외 처리

상황대응
토스 결제 다운토스 콘솔에서 상태 확인 → 회원에게 점검 안내 발송 → 자동 트리거 일시 정지
효성 출금 파일 오류효성 측 응답 파일 재요청 → 매칭 실패 회원 수기 처리
영수증 PDF 미수신 회원회원 상세 → [영수증 재발송] · 스팸함 안내
환불 요청토스/효성 콘솔에서 수동 환불 → 시스템 메모 + 후원 내역 상태 변경
카드 정보 유출 의심즉시 빌링키 폐기 → 회원에게 재등록 안내
대량 결제 실패 발생토스/효성 상태 확인 → 자동 트리거 일시 정지 → 원인 파악 후 재개
발송 작업 잘못 송출/admin-send-jobs.html 즉시 [일시 정지] → 정정 안내 후속 발송
AI 비용 폭증/admin-ai-cost.html에서 해당 기능 즉시 차단 → 호출 패턴 조사
익명 식별 이상 패턴/admin-anon-audit.html 즉시 점검 → 해당 운영자 사유 재확인

질문답변

환불은 시스템에서 되나?자동 환불 없음. 토스/효성 콘솔에서 수동 + 시스템 메모.
카드 만료 회원 한 번에 알리려면?수신자 그룹 [카드 만료 임박] → 발송 작업 생성. 자동 트리거가 이미 처리 중일 수 있음.
후원자 명단을 엑셀로 받고 싶다회원 목록 필터 적용 → [Excel 내보내기].
연말 기부금 영수증 절차는?1~2월 별도 일괄 발급 트리거. 주민번호 미등록자 11~12월 사전 캠페인.
신규 후원 채널 성과를 보고 싶다회원 목록 > 채널별 KPI. URL ?src=채널명으로 유입되어야 분류됨.
급여 명세서를 잘못 발급했다/admin-payroll.html에서 재발급. 직원에게 재발급 알림 자동 발송.
AI 비용이 갑자기 늘었다/admin-ai-cost.html Top 5 기능 확인 → 해당 트리거·기능 점검 또는 일시 차단.
